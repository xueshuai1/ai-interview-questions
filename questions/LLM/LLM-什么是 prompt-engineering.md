## 题目描述
Prompt Engineering（提示工程）是有效使用大语言模型的关键技能。请解释 Prompt Engineering 的定义、核心技巧及最佳实践。

**标签：** Prompt Engineering, 提示工程，Few-Shot, 思维链，最佳实践

## 参考答案
### 定义

**Prompt Engineering** 是通过设计和优化输入提示（Prompt），引导大语言模型生成更准确、更有用输出的技术。

### 核心技巧

**1. 零样本提示（Zero-Shot）**
```
直接给出任务："将以下文本翻译成法语：Hello"
```

**2. 少样本提示（Few-Shot）**
```
提供示例：
English: Good morning → French: Bonjour
English: Thank you → French: Merci
English: Hello → French: ?
```

**3. 思维链（Chain-of-Thought）**
```
"让我们一步步思考：1... 2... 3... 答案：..."
```

**4. 角色设定**
```
"你是一位经验丰富的 Python 工程师，请帮我 review 代码..."
```

**5. 结构化输出**
```
"请用 JSON 格式回答，包含字段：summary, key_points"
```

### 最佳实践

**1. 清晰明确** - 避免模糊指令
**2. 提供上下文** - 给出充分背景信息
**3. 指定约束** - 长度、格式、风格等
**4. 迭代优化** - 根据输出调整提示
**5. 使用分隔符** - 清晰区分不同部分

### 常见错误

| 错误 | 改进 |
|------|------|
| 提示太模糊 | 添加具体要求和约束 |
| 缺少示例 | 添加 Few-Shot 示例 |
| 任务太复杂 | 分解为多个子任务 |
| 忽略上下文 | 提供充分背景信息 |

## 延伸追问

### 追问 1: Few-Shot 示例数量（5分）
**问题：** 提供多少个示例最合适？

**答案：** 通常 2-3 个最佳。简单任务 Zero-Shot，复杂任务 4-8 个。

### 追问 2: 思维链的适用场景（5分）
**问题：** 什么情况下应该使用思维链？

**答案：** 数学计算、逻辑推理、多步决策问题。简单事实查询不需要。

### 追问 3: Prompt 注入攻击（5分）
**问题：** 什么是 Prompt 注入？如何防范？

**答案：** 用户通过特殊输入绕过系统限制。防范：分隔符、输入过滤、输出审核。

### 追问 4: Prompt 优化流程（5分）
**问题：** 如何系统地优化 Prompt？

**答案：** 定义目标→基线测试→添加技巧→A/B 测试→收集反馈→迭代优化。

## 深入理解
Prompt Engineering 是人与 AI 高效协作的关键技能。掌握核心技巧和最佳实践，可以显著提升大模型的使用效果。
