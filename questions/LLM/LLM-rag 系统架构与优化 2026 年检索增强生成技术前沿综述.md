## 题目描述
RAG（Retrieval-Augmented Generation，检索增强生成）是解决大模型知识更新和幻觉问题的重要技术。请说明 RAG 的系统架构、核心组件及优化方法。

**标签：** RAG, 检索增强生成，向量检索，知识更新，幻觉减少

## 参考答案
### 系统架构

RAG 系统由三个核心阶段组成：

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   检索阶段   │ -> │   增强阶段   │ -> │   生成阶段   │
│  Retrieval  │    │ Augmentation│    │  Generation │
└─────────────┘    └─────────────┘    └─────────────┘
```

### 核心组件

**1. 检索器（Retriever）**
- **功能：** 从知识库中检索与查询相关的文档
- **类型：**
  - 稠密检索（Dense Retrieval）：使用向量相似度
  - 稀疏检索（Sparse Retrieval）：使用 BM25 等传统方法
  - 混合检索（Hybrid）：结合两者优势

**2. 知识库（Knowledge Base）**
- **文档存储：** 原始文本分块存储
- **向量索引：** 使用向量数据库（如 FAISS、Milvus、Pinecone）
- **更新机制：** 支持增量更新，无需重新训练

**3. 生成器（Generator）**
- **基础模型：** LLM（如 GPT、Claude、Qwen 等）
- **输入格式：** `[检索结果] + [用户查询] -> [生成回答]`
- **上下文窗口：** 需要足够大以容纳检索结果

### 工作流程

```python
# RAG 伪代码示例
def rag_query(user_query, knowledge_base, llm, top_k=5):
    # 1. 查询编码
    query_embedding = encoder(user_query)
    
    # 2. 向量检索
    relevant_docs = knowledge_base.search(
        query_embedding, 
        top_k=top_k,
        threshold=0.7
    )
    
    # 3. 结果重排序（可选）
    reranked_docs = reranker(user_query, relevant_docs)
    
    # 4. 构建提示
    context = "\n\n".join([doc.content for doc in reranked_docs])
    prompt = f"""基于以下信息回答问题：

{context}

问题：{user_query}
回答："""
    
    # 5. 生成回答
    response = llm.generate(prompt)
    
    # 6. 引用标注（可选）
    response_with_citations = add_citations(response, reranked_docs)
    
    return response_with_citations
```

### 优化方法

**1. 检索优化**
| 方法 | 说明 | 效果 |
|------|------|------|
| 查询改写 | 扩展/重写查询，提高召回率 | +15-20% |
| 混合检索 | 稠密 + 稀疏检索融合 | +10-15% |
| 多向量索引 | ColBERT 等细粒度检索 | +5-10% |
| 查询路由 | 根据查询类型选择检索策略 | 降低成本 |

**2. 上下文优化**
- **分块策略：** 重叠分块（overlap=100-200 tokens）
- **上下文压缩：** 只保留最相关信息（LLMLingua 等）
- **窗口扩展：** 包含前后文（parent document retrieval）

**3. 生成优化**
- **提示工程：** 明确指示模型基于检索结果回答
- **引用标注：** 标注信息来源，提高可信度
- **答案验证：** 检查答案与检索结果的一致性

## 延伸追问

### 追问 1: RAG vs Fine-tuning（5分）（5分）
**问题：** RAG 与微调（Fine-tuning）有什么区别？如何选择？

**答案：** | 维度 | RAG | Fine-tuning |
|------|-----|-------------|
| 知识更新 | 实时更新知识库 | 需要重新训练 |
| 幻觉问题 | 显著减少 | 仍存在 |
| 成本 | 低（无需训练） | 高（训练成本） |
| 私有数据 | 支持 | 支持 |
| 推理延迟 | 略高（需检索） | 低 |
| 适用场景 | 知识密集型、频繁更新 | 风格适配、任务特定 |

**选择建议：** 知识更新频繁→RAG；风格/任务特定→微调；最佳→RAG+ 微调结合

### 追问 2: 向量检索原理（5分）（5分）
**问题：** 稠密检索（向量检索）的原理是什么？常用哪些相似度度量？

**答案：** **原理：**
1. 使用编码器（如 BERT、bge）将文本映射为向量
2. 计算查询向量与文档向量的相似度
3. 返回相似度最高的 top-k 文档

**相似度度量：**
- **余弦相似度：** cos(θ) = (A·B)/(||A||×||B||)，最常用
- **内积（点积）：** A·B，用于归一化向量
- **欧氏距离：** ||A-B||，距离越小越相似

### 追问 3: RAG 评估指标（5分）（5分）
**问题：** 如何评估 RAG 系统的性能？

**答案：** **检索阶段指标：**
- **Recall@k：** 相关文档在前 k 个中的比例
- **MRR：** 第一个相关文档的排名倒数均值
- **NDCG：** 考虑排名位置的增益

**生成阶段指标：**
- **答案准确性：** 与标准答案对比（EM、F1）
- **忠实度：** 答案是否基于检索内容
- **相关性：** 答案是否切题

### 追问 4: RAG 最新进展（5分）（5分）
**问题：** RAG 技术有哪些最新进展（2025-2026）？

**答案：** 1. **Agentic RAG：** Agent 自主决定检索策略、迭代检索
2. **Graph RAG：** 结合知识图谱，提升推理能力
3. **Multimodal RAG：** 支持图像、表格等多模态检索
4. **Corrective RAG：** 自动纠正检索错误，迭代优化
