## 🔴 死循环识别规则

### 规则 1：同一动作最多 2 次
- 同一个工具调用（如 exec grep 同样的内容）→ **最多 2 次**
- 第 2 次失败 → **立即换方案，不重试**
- 第 2 次结果和第 1 次相同 → **停止，换方向**

### 规则 2：输出重复检测
- 如果连续 2 次输出的内容相似度 > 80% → **立即停止**
- 例如：连续 2 次都在"分析历史数据"但没写代码 → 停

### 规则 3：进度检查
- 执行 3 分钟后自问：**"我产出了什么实际东西？"**
- 如果答案只是"我看了很多文件/跑了很多命令" → **立即停止分析，开始产出**

## 🚪 强制跳出机制

### 触发条件（满足任一即跳出）

| 条件 | 动作 |
|------|------|
| 同一工具调用 ≥ 2 次 | 换方案或停止 |
| 连续 2 次输出相似 | 停止当前任务 |
| 3 分钟无实际产出 | 跳到执行阶段 |
| 5 分钟未完成 | 汇报进度 + 停止 |
| 同一问题反复出现 ≥ 3 次 | 记录到 EVOLUTION.md，换方案 |

### 跳出后做什么

```
检测到循环 → 
  1. 记录到 EVOLUTION.md（什么问题、循环了几次）
  2. 换方案（如果有的话）
  3. 或直接停止 + 汇报（如果没有可行方案）
  4. 绝不重复第 3 次
```

---

## 📋 执行前检查清单

每次 cron 启动时自问：
1. 读 EVOLUTION.md 的"当前待解决问题"（不是历史记录！）
2. 上次执行留下了什么未完成的事？
3. **我有没有在做和上次同样的事？** → 如果是，换方向

  （最多 3 次尝试，绝不循环）
