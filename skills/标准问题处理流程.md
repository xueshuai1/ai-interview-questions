## 🎯 完整流程示例

### 示例：新增文章不显示

#### 1. 接收问题
```
用户反馈：新增了文章，但在网站上看不到
预期：文章应该显示在分类列表和详情页
实际：找不到新增文章
```

#### 2. 复现问题
```bash
# 启动本地服务器
npm run dev

# 访问分类列表
http://localhost:3000/knowledge/LLM
# ❌ 找不到新增文章

# 访问文章详情
http://localhost:3000/knowledge/LLM/001_Transformer_示例
# ❌ 404 或文章不存在
```

#### 3. 分析排查
```bash
# 检查文章文件
ls content/knowledge/LLM/
# ✅ 文件存在

# 检查文章元数据
head -20 content/knowledge/LLM/001_Transformer_示例.md
# ✅ 元数据正确

# 检查代码
cat src/app/knowledge/[category]/page.tsx
# ❌ 发现是客户端组件，使用 useEffect 调用 API
# ❌ API 在构建时生成缓存，不会自动更新
```

#### 4. 解决问题
```bash
# 将分类列表页改为服务器端渲染
vim src/app/knowledge/[category]/page.tsx

# 添加 ISR 配置
export const revalidate = 3600;

# 直接读取 Markdown 文件
const articles = getArticlesByCategory(category);
```

#### 5. 本地验证
```bash
# 启动本地服务器
npm run dev

# 访问分类列表
http://localhost:3000/knowledge/LLM
# ✅ 能看到新增文章

# 访问文章详情
http://localhost:3000/knowledge/LLM/001_Transformer_示例
# ✅ 文章内容正常显示
```

#### 6. 完整测试
```bash
# 执行完整构建
npm run build
# ✅ 构建成功

# 测试其他页面
# ✅ 首页正常
# ✅ 其他分类正常
# ✅ 文章详情正常
```

#### 7. 提交部署
```bash
# 预提交检查
git add -A
git commit
# ✅ 文章质量检查通过
# ✅ 本地构建测试通过
# ✅ 新增文章验证通过

# 推送
git push origin main

# Vercel 部署
# ✅ 部署成功

# 验证生产环境
# ✅ 问题已解决
```
