# 技术文档质量评估标准

**版本：** 1.0  
**创建时间：** 2026-03-29  
**用途：** 评估收集到的技术文档/面试题质量

---

## 📊 评分标准（100 分制）

### 1️⃣ 问题定义（10 分）

| 分值 | 标准 |
|------|------|
| 9-10 | 清晰说明技术背景、应用场景、解决的核心痛点 |
| 7-8 | 说明了问题和场景，但不够深入 |
| 5-6 | 仅简单提及问题，缺乏背景 |
| <5 | 未说明解决的问题 |

**关键要素：**
- ✅ 技术出现的背景
- ✅ 解决的痛点问题
- ✅ 适用场景说明
- ✅ 与旧方案的对比

---

### 2️⃣ 核心概念（10 分）

| 分值 | 标准 |
|------|------|
| 9-10 | 概念准确，原理清晰，无模糊或错误表述 |
| 7-8 | 概念基本准确，个别地方不够清晰 |
| 5-6 | 概念有模糊之处，需要读者自行理解 |
| <5 | 概念混乱或有明显错误 |

**关键要素：**
- ✅ 术语定义准确
- ✅ 原理讲解清晰
- ✅ 有类比或图示辅助理解
- ✅ 无知识性错误

---

### 3️⃣ 使用示例（15 分）

| 分值 | 标准 |
|------|------|
| 13-15 | 提供完整可运行代码，有输入输出示例，有注释 |
| 10-12 | 有代码示例，但缺少部分细节 |
| 7-9 | 仅有伪代码或代码片段 |
| <7 | 无代码示例 |

**关键要素：**
- ✅ 完整的代码示例
- ✅ 输入输出展示
- ✅ 关键步骤注释
- ✅ 环境依赖说明
- ✅ 可复制运行

**示例：**
```python
# ✅ 好的示例
from sklearn.ensemble import RandomForestClassifier

# 准备数据
X_train, X_test = train_test_split(X, y, test_size=0.2)

# 创建模型
model = RandomForestClassifier(n_estimators=100, max_depth=5)

# 训练
model.fit(X_train, y_train)

# 预测
predictions = model.predict(X_test)
```

```python
# ❌ 差的示例
model = RandomForest()  # 缺少参数
model.fit(X, y)  # 缺少数据划分
```

### 4️⃣ 实践指南（15 分）

| 分值 | 标准 |
|------|------|
| 13-15 | 包含最佳实践、常见陷阱、调优技巧、性能考量 |
| 10-12 | 有实践建议，但不够全面 |
| 7-9 | 仅有基本使用方法 |
| <7 | 无实践指导 |

**关键要素：**
- ✅ 最佳实践建议
- ✅ 常见错误/陷阱
- ✅ 参数调优技巧
- ✅ 性能优化建议
- ✅ 扩展阅读资源

---

### 5️⃣ 对比分析（10 分）

| 分值 | 标准 |
|------|------|
| 9-10 | 与 3+ 种类似技术对比，有优缺点表格 |
| 7-8 | 与 1-2 种技术对比，有简要分析 |
| 5-6 | 仅提及有其他方案 |
| <5 | 无对比分析 |

**关键要素：**
- ✅ 与类似技术对比
- ✅ 优缺点对比表
- ✅ 适用场景分析
- ✅ 选择建议

**示例对比表：**
| 特性 | Random Forest | XGBoost | LightGBM |
|------|--------------|---------|----------|
| 训练速度 | 中 | 慢 | 快 |
| 预测精度 | 高 | 很高 | 高 |
| 内存占用 | 高 | 中 | 低 |
| 适用场景 | 通用 | 比赛 | 大数据 |

---

### 6️⃣ 深度适中（10 分）

| 分值 | 标准 |
|------|------|
| 9-10 | 深入浅出，既有理论又有实践，适合目标读者 |
| 7-8 | 深度基本合适，个别地方过深或过浅 |
| 5-6 | 过于表面或过于深入，不适合目标读者 |
| <5 | 深度严重不合适 |

**关键要素：**
- ✅ 目标读者明确
- ✅ 理论与实践平衡
- ✅ 数学公式适量（有解释）
- ✅ 由浅入深的结构

---

### 7️⃣ 结构清晰（10 分）

| 分值 | 标准 |
|------|------|

**关键要素：**
- ✅ 清晰的目录结构

- ✅ 有关键词索引

- ✅ 架构图/流程图

### 9️⃣ 时效性（5 分）

| 分值 | 标准 |
|------|------|

**关键要素：**
- ✅ 提及最新版本号

| 分值 | 标准 |
|------|------|

| 4 | 来源基本可靠 |

**关键要素：**
- ✅ 作者背景介绍
- ✅ 引用权威论文/文档
- ✅ 有参考文献列表
- ✅ 来自知名平台（GitHub、ArXiv、官方文档等）

## 📋 评估流程

### 步骤 1：快速浏览（2 分钟）
- 看标题和摘要
- 看目录结构
- 看是否有代码和图表

### 步骤 2：详细评估（5 分钟）
- 逐项打分（10 个维度）
- 记录优点和不足
- 计算总分

### 步骤 3：决策
- **90-100 分**：优秀，直接采用
- **70-89 分**：良好，补充后采用
- **50-69 分**：一般，需要大量补充
- **<50 分**：较差，不建议采用

## 📌 注意事项

1. **不要过分追求完美**：70 分以上的文档就可以采用
2. **注重实用性**：代码示例比理论更重要
3. **关注时效性**：优先选择近 2 年的内容
4. **交叉验证**：重要知识点从多个来源验证
5. **持续更新**：技术文档需要定期更新
