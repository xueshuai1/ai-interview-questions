# 术语命名规范 📝

> 所有技术文档标题必须遵循的命名规范

---

## 核心原则

**缩写 + 中文全称**

如果标题包含技术术语缩写，必须同时包含**英文缩写**和**中文全称**。

---

## 格式规范

```
[缩写]（[中文全称]）+ 主题
```

**示例：**

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| `CV 技术详解` | `CV（计算机视觉）技术详解` |
| `RAG 实战指南` | `RAG（检索增强生成）实战指南` |
| `LLM 应用开发` | `LLM（大语言模型）应用开发` |
| `NLP 入门` | `NLP（自然语言处理）入门` |
| `Transformer 详解` | `Transformer（变换器）详解` |

## 应用场景

### 1. 文件命名

```
content/knowledge/LLM/006_Prompt（提示）Engineering（工程）实战指南.mdx
```

### 2. 文章标题

```mdx
# LLM（大语言模型）位置编码详解
```

### 3. Frontmatter

```mdx
---
title: "RAG（检索增强生成）实战指南"
category: "LLM"
---
```

### 4. 摘要

```mdx
> **分类**: LLM（大语言模型） | **编号**: LLM-003
>
> `RAG（检索增强生成）` `LLM（大语言模型）`
```

---

## 例外情况

以下情况可以只用缩写：

1. **广为人知的术语**
   - AI（人工智能）- 可以直接用 AI
   - API（应用程序接口）- 可以直接用 API
   - GPU（图形处理器）- 可以直接用 GPU

2. **没有通用中文译名的术语**
   - Attention - 可以直接用 Attention
   - Embedding - 可以直接用 Embedding

3. **首次出现时已说明**
   ```
   摘要中：本文介绍 LLM（大语言模型）技术...
   标题中：LLM 技术详解 ✅
   ```
