# 双轨内容体系设计文档

## 概述

本网站采用**双轨内容体系**，将内容分为两套独立但关联的体系：

1. **知识库** - 系统化学习，按难度分级
2. **面试题库** - 面试准备，按岗位/公司分类

两套体系通过知识点关联，支持互相跳转，形成完整的学习 → 面试闭环。

---

## 一、知识库体系（学习导向）

## 三、两套体系的关联

### 3.1 知识点关联

- **面试题目 → 知识库文章**：每道面试题关联对应的知识点文章
- **知识库文章 → 面试题目**：学习知识点时可查看相关面试题

### 3.2 学习路径

```
基础知识（知识库⭐） 
    ↓
深入理解（知识库⭐⭐）
    ↓
高级应用（知识库⭐⭐⭐）
    ↓
面试准备（面试题库）
```

### 3.3 互相跳转

**在知识库页面：**
- 显示"相关面试题"模块
- 点击可跳转到面试题库

**在面试题库页面：**
- 显示"相关知识点"模块
- 点击可跳转到知识库文章

## 五、实现方案

### 5.1 路由设计

```
/                          # 首页
/knowledge                 # 知识库首页
/knowledge/[category]      # 按分类浏览
/knowledge/[category]/[id] # 知识库文章详情

/interview                 # 面试题库首页
/interview/[position]      # 按岗位浏览
/interview/[company]       # 按公司浏览
/interview/[id]            # 面试题目详情
```

### 5.2 数据模型

```typescript
// 知识库文章
interface KnowledgeArticle {
  id: string;
  title: string;
  type: 'knowledge';
  category: string;
  difficulty: 1 | 2 | 3 | 4;
  content: string;
  relatedQuestions: string[];
  tags: string[];
}

// 面试题目
interface InterviewQuestion {
  id: string;
  title: string;
  type: 'interview';
  position: string;
  company?: string;
  round?: string;
  category: string;
  difficulty: 1 | 2 | 3 | 4;
  content: string;
  relatedKnowledge: string[];
  tags: string[];
}
```

### 5.3 关联查询

通过 `relatedQuestions` 和 `relatedKnowledge` 字段实现双向关联：

- 知识库文章 → 面试题目：`article.relatedQuestions`
- 面试题目 → 知识库文章：`question.relatedKnowledge`
