# 离线强化学习面试题（5 道）

## 题目 1：核心挑战
**问题：** 离线强化学习与在线 RL 的核心区别是什么？主要挑战有哪些？

**参考答案：**
**核心区别**：

| 方面 | 在线 RL | 离线 RL |
|------|---------|---------|
| **数据收集** | 在线交互 | 固定数据集 |
| **探索** | 主动探索 | 无法探索 |
| **安全性** | 可能危险 | 安全 |
| **成本** | 高（交互） | 低（仅计算） |
| **适用** | 仿真环境 | 真实应用 |

**主要挑战**：

**1. 分布外（OOD）问题**：
- 学习策略可能选择数据集中未见的动作
- Q 函数对 OOD 动作估计不可靠
- 导致性能下降

**2. 外推误差**：
```
Q(s, a_OOD) 估计不准确
→ 策略选择 a_OOD（因为 Q 值高估）
→ 实际性能差
```

**3. 数据覆盖有限**：
- 行为策略β可能不是最优
- 数据只覆盖状态空间一部分
- 无法学习未覆盖区域的最优策略

**4. 无法验证**：
- 不能在线测试策略
- 只能依赖离线评估
- 评估本身也有挑战

**5. 分布偏移**：
- 学习策略π与行为策略β不同
- 状态分布随策略变化
- Bellman 方程假设被违反

**解决方案方向**：
- 约束策略（BCQ）
- 保守 Q 学习（CQL）
- 期望回归（IQL）
- 不确定性估计

**评分要点：** 说出 OOD 问题得基本分，详细分析挑战得高分

## 题目 2：CQL 原理
**问题：** CQL（Conservative Q-Learning）的核心思想是什么？如何实现保守估计？

**参考答案：**
**核心思想**：
学习保守的 Q 函数，系统性地低估分布外动作的价值，避免策略选择不可靠的动作。

**CQL 损失**：
```
L_CQL(θ) = L_TD(θ) + α · (E_a∼π[Q_θ(s,a)] - E_a∼D[Q_θ(s,a)])
```

**各项含义**：
- `L_TD`：标准 TD 损失
- `E_a∼π[Q(s,a)]`：当前策略动作的 Q 值期望
- `E_a∼D[Q(s,a)]`：数据集中动作的 Q 值期望
- `α`：保守系数

**保守机制**：

**1. 最小化策略动作 Q 值**：
```
min_θ E_a∼π[Q_θ(s,a)]
```
- 降低所有可能动作的 Q 值
- 特别是 OOD 动作

**2. 最大化数据动作 Q 值**：
```
max_θ E_a∼D[Q_θ(s,a)]
```
- 保持数据集中动作的 Q 值准确
- 相对提高数据动作的价值

**3. 效果**：
```
Q_保守(s, a_OOD) < Q_真实(s, a_OOD)
Q_保守(s, a_数据) ≈ Q_真实(s, a_数据)
```

**实现方式**：

**log-sum-exp 近似**：
```python
# 随机采样动作
random_actions = uniform(-1, 1)

# log-sum-exp 近似期望
cql_loss = logsumexp(Q(s, random_actions), dim=1).mean() - Q(s, data_actions).mean()
```

**为什么有效**：
- OOD 动作 Q 值被压低
- 策略不会选择 OOD 动作
- 只在数据覆盖区域优化

**评分要点：** 说出保守估计得基本分，写出损失函数得高分

## 题目 3：BCQ 算法
**问题：** BCQ 如何解决分布外问题？它的核心机制是什么？

**参考答案：**
**BCQ（Batch-Constrained Q-learning）**：

**核心思想**：
约束策略只能选择与数据集中相似的动作，避免分布外动作。

**核心机制**：

**1. 生成模型**：
```
G(s) → 动作分布 p(a|s)
```
- 学习数据集中动作的条件分布
- 通常用 VAE 或 GAN

**2. 扰动机制**：
```
a = G(s) + ξ,  ξ ∈ [-Φ, Φ]
```
- 在生成动作基础上加小扰动
- Φ控制扰动范围
- 确保动作接近数据分布

**3. Q 学习约束**：
```
π(s) = argmax_{a∈[G(s)-Φ, G(s)+Φ]} Q(s,a)
```
- 只在约束区域内最大化 Q
- 避免选择 OOD 动作

**算法流程**：
```
1. 训练生成模型 G(s) 拟合数据动作分布
2. 训练 Q 函数：
   Q_target = r + γ max_{a'∈G(s')} Q(s',a')
3. 策略：π(s) = argmax_{a≈G(s)} Q(s,a)
```

**优点**：
- 显式约束动作选择
- 理论保证
- 简单有效

**缺点**：
- 需要训练生成模型
- 扰动范围Φ需要调
- 可能过于保守

**与 CQL 对比**：
- BCQ：显式约束动作
- CQL：隐式约束（通过 Q 值）

**评分要点：** 说出生成模型 + 扰 động得基本分，对比 CQL 得高分

## 题目 4：离线评估
**问题：** 如何评估离线 RL 算法的性能？有哪些评估方法？

**参考答案：**
**评估挑战**：
- 不能在线测试
- 真实环境交互成本高/危险
- 需要可靠的离线评估

**评估方法**：

**1. 行为策略评估（Behavior Policy Evaluation）**：
```
在行为策略β上评估
```
- 简单但不准确
- 不能反映学习策略π的性能

**2. 重要性采样（Importance Sampling）**：
```
η(π) ≈ (1/N) Σ ρ_i · R_i
ρ_i = Π_t π(a_t|s_t) / β(a_t|s_t)
```
- 无偏估计
- 方差可能很大
- 长序列问题

**3. 加权重要性采样（WIS）**：
```
η(π) ≈ Σ w_i R_i / Σ w_i
w_i = ρ_i
```
- 有偏但方差小
- 实践中更稳定

**4. 模型-based 评估**：
```
学习动力学模型 P(s'|s,a)
在模型中仿真策略π
```
- 可以评估任意策略
- 模型误差影响评估

**5. FQE（Fitted Q Evaluation）**：
```
用数据拟合 Q^π
η(π) = E_s∼D,a∼π[Q^π(s,a)]
```
- 常用方法
- 相对稳定

**6. 保守评估**：
```
评估下界而非期望
```
- 更安全
- 可能过于保守

**实践建议**：
- 多种方法结合
- 报告置信区间
- 与行为策略对比
- 敏感性分析

**评分要点：** 说出 3 种以上方法得基本分，详细分析得高分

## 题目 5：应用场景
**问题：** 离线 RL 适合哪些应用场景？为什么这些场景需要离线学习？

**参考答案：**
**适合的应用场景**：

**1. 机器人学习**：
- **原因**：
  - 真实机器人交互成本高
  - 探索可能损坏硬件
  - 安全约束严格
- **数据来源**：
  - 遥操作演示
  - 历史控制数据
  - 仿真数据

**2. 医疗决策**：
- **原因**：
  - 不能对患者随机试验
  - 伦理约束
  - 个体差异大
- **数据来源**：
  - 电子病历
  - 临床试验数据
  - 医生决策记录

**3. 推荐系统**：
- **原因**：
  - 用户不能随机试验
  - 在线 A/B 测试成本高
  - 用户体验重要
- **数据来源**：
  - 用户行为日志
  - 历史推荐记录
  - 点击/购买数据

**4. 自动驾驶**：
- **原因**：
  - 安全关键
  - 真实道路测试危险
  - 法规限制
- **数据来源**：
  - 人类驾驶记录
  - 仿真数据
  - 测试车队数据

**5. 金融交易**：
- **原因**：
  - 真金白银风险
  - 市场不可重复
  - 监管要求
- **数据来源**：
  - 历史市场数据
  - 交易记录
  - 模拟交易

**6. 教育个性化**：
- **原因**：
  - 学生不能随机试验
  - 学习效果长期
  - 伦理考虑
- **数据来源**：
  - 学生学习记录
  - 测试成绩
  - 互动数据

**共同特点**：
1. **交互成本高**：时间、金钱、资源
2. **安全约束**：不能随意探索
3. **伦理考虑**：涉及人/动物
4. **数据可用**：有历史数据积累

**离线 RL 优势**：
- 安全：无危险探索
- 高效：利用现有数据
- 可部署：直接学习可部署策略

**评分要点：** 说出 3 个以上场景得基本分，详细分析原因得高分
