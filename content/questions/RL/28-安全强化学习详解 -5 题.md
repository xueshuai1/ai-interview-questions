# 安全强化学习详解面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是安全强化学习？为什么它对真实世界应用至关重要？

**参考答案：**
**安全强化学习（Safe RL）**：
在优化回报的同时确保系统满足安全约束，避免危险行为的强化学习方法。

**核心目标**：
```
max E[回报]
s.t. 安全约束满足
```

**为什么至关重要**：

**1. 安全风险**：
```
自动驾驶：碰撞可能致命
机器人：可能损坏设备或伤人
医疗：患者生命安全
金融：重大经济损失
```

**2. 法规要求**：
```
许多应用有安全法规
必须合规才能部署
```

**3. 用户信任**：
```
安全是用户接受的前提
一次事故可能摧毁信任
```

**4. 探索风险**：
```
RL 需要探索
探索可能危险
需要安全探索
```

**安全约束类型**：

**硬约束**：
```
绝对不能违反
如：不碰撞、不超限
```

**软约束**：
```
尽量不违反
如：舒适度、能耗
```

**概率约束**：
```
违反概率<阈值
如：P(碰撞)<0.01
```

**安全 RL vs 标准 RL**：

| 方面 | 标准 RL | 安全 RL |
|------|---------|---------|
| **目标** | 最大化回报 | 回报 + 安全 |
| **探索** | 自由探索 | 安全探索 |
| **约束** | 无 | 有 |
| **部署** | 仿真为主 | 真实世界 |

**评分要点：** 说出安全约束得基本分，解释真实世界重要性得高分

## 题目 2：CMDP
**问题：** 什么是 CMDP？如何用拉格朗日方法求解？

**参考答案：**
**CMDP（Constrained MDP）**：
带约束的马尔可夫决策过程，在优化回报的同时满足成本约束。

**数学定义**：
```
max_π E[Σ γ^t r(s_t, a_t)]
s.t. E[Σ γ^t c(s_t, a_t)] ≤ d
```
其中：
- r：回报
- c：成本
- d：成本阈值

**拉格朗日方法**：

**增广拉格朗日函数**：
```
L(π, λ) = E[回报] - λ(E[成本] - d)
```

**优化过程**：
```
1. 固定λ，优化π最大化 L
2. 固定π，更新λ最小化 L
3. 交替迭代
```

**λ更新规则**：
```
λ ← max(0, λ + α(E[成本] - d))
```
- 成本超支→λ增加→更重视约束
- 成本有余→λ减少→更重视回报

**实现代码**：
```python
# 策略损失（带约束）
policy_loss = -Q.mean() + lambda_ * (cost.mean() - threshold)

# 更新λ
cost_violation = cost.mean() - threshold
lambda_ = max(0, lambda_ + lambda_lr * cost_violation)
```

**优点**：
- 简单实现
- 可处理软约束
- 广泛使用

**缺点**：
- 不保证硬约束
- λ调参困难
- 可能振荡

**改进方法**：
1. **CPO**：约束策略优化
2. **PDO**：投影优化
3. **RCPO**：风险约束

**实践建议**：
- 从λ=0 开始
- 监控约束违反
- 调整λ学习率

**评分要点：** 写出 CMDP 公式得基本分，解释λ更新得高分

## 题目 3：控制屏障函数
**问题：** 控制屏障函数（CBF）如何保证安全？如何实现？

**参考答案：**
**控制屏障函数（CBF）**：
用数学方法保证系统状态始终在安全集内。

**核心思想**：
```
定义安全集：C = {s | h(s) ≥ 0}
保证：h(s(t)) ≥ 0, ∀t
```

**安全条件**：
```
ḣ(s) + α(h(s)) ≥ 0
```
其中α是扩展 K 类函数（如α(x)=kx）。

**离散时间**：
```
h(s_{t+1}) - h(s_t) + α·h(s_t) ≥ 0
```

**实现方式**：

**1. 安全过滤**：
```python
def project_action(state, desired_action):
    # 检查是否安全
    if h(state) >= 0:
        return desired_action
    
    # 投影到安全集
    safe_action = solve_QP(desired_action, CBF_constraint)
    return safe_action
```

**2. 二次规划**：
```
min ||a - a_desired||²
s.t. ∇h(s)·f(s,a) + α(h(s)) ≥ 0
```

**优点**：
- 理论保证安全
- 实时计算
- 可与任何策略结合

**缺点**：
- 需要系统模型
- 可能过于保守
- 高维难计算

**应用场景**：
```
无人机：避障
机器人：关节限位
自动驾驶：安全距离
```

**实践要点**：
1. 设计合适的 h(s)
2. 选择α参数
3. 高效求解 QP
4. 处理模型误差

**评分要点：** 写出安全条件得基本分，解释实现方式得高分

## 题目 4：安全探索
**问题：** 如何在保证安全的前提下进行探索？有哪些安全探索方法？

**参考答案：**
**安全探索挑战**：
```
探索需要尝试新动作
新动作可能危险
如何平衡？
```

**安全探索方法**：

**1. 安全集内探索**：
```
定义安全集 S_safe
只在 S_safe 内探索
```

**实现**：
```python
if next_state in S_safe:
    execute(action)
else:
    choose_safe_action()
```

**2. 保守探索**：
```
选择不确定性低且安全的动作
避免高风险探索
```

**3. 恢复策略**：
```
探索前准备恢复动作
危险时立即恢复
```

**4. 渐进式探索**：
```
从安全区域开始
逐步扩大探索范围
```

**5. 模型预测安全**：
```
用模型预测下一状态
预测不安全则不执行
```

**实现框架**：
```python
class SafeExplorer:
    def select_action(self, state):
        # 候选动作
        candidates = self.get_candidate_actions()
        
        # 安全过滤
        safe_actions = []
        for action in candidates:
            if self.is_safe(state, action):
                safe_actions.append(action)
        
        # 从安全动作中选择
        if safe_actions:
            return self.select_from_safe(safe_actions)
        else:
            return self.emergency_action()
    
    def is_safe(self, state, action):
        # 预测下一状态
        next_state = self.model.predict(state, action)
        
        # 检查安全
        return self.safety_check(next_state)
```

**挑战**：
1. **安全集定义**：
   - 太大：不安全
   - 太小：无法学习

2. **模型误差**：
   - 预测不准
   - 可能误判安全

3. **计算成本**：
   - 实时性要求
   - 复杂约束难求解

**实践建议**：
- 定义保守安全集
- 使用冗余安全检查
- 准备紧急停止
- 持续监控

**评分要点：** 说出安全集探索得基本分，列举多种方法得高分
