# 约束强化学习面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是约束强化学习？它与安全 RL 有什么区别？

**参考答案：**
**约束强化学习（Constrained RL）**：
在优化回报的同时满足约束条件的强化学习方法。

**数学形式（CMDP）**：
```
max_π E[Σ γ^t r(s_t, a_t)]
s.t. E[Σ γ^t c_i(s_t, a_t)] ≤ d_i, ∀i
```

**与安全 RL 的区别**：

| 方面 | 约束 RL | 安全 RL |
|------|---------|---------|
| **约束类型** | 通用约束 | 安全约束 |
| **目标** | 回报 + 约束 | 安全优先 |
| **违反后果** | 性能下降 | 危险/损坏 |
| **应用** | 资源、能耗 | 自动驾驶、机器人 |
| **严格程度** | 可软可硬 | 通常硬约束 |

**约束类型**：

**1. 资源约束**：
```
CPU、内存、带宽
预算限制
```

**2. 性能约束**：
```
延迟上限
吞吐量下限
SLA 保证
```

**3. 安全约束**：
```
不碰撞
不超限
安全操作
```

**4. 法规约束**：
```
合规要求
政策限制
```

**为什么需要约束 RL**：
```
真实世界问题：
- 资源有限
- 性能要求
- 安全关键
- 法规合规

标准 RL 无法处理约束
```

**求解挑战**：
1. 约束满足与回报权衡
2. 可行策略可能存在
3. 约束可能冲突
4. 需要理论保证

**评分要点：** 说出 CMDP 公式得基本分，对比安全 RL 得高分

## 题目 2：拉格朗日方法
**问题：** 拉格朗日方法如何求解约束 RL？乘子如何更新？

**参考答案：**
**拉格朗日方法**：
将约束优化转化为无约束优化。

**增广拉格朗日函数**：
```
L(π, λ) = E[回报] - Σ λ_i (E[成本_i] - d_i)
```

**优化过程**：
```
1. 固定λ，优化π最大化 L
2. 固定π，优化λ最小化 L
3. 交替迭代
```

**乘子更新规则**：
```
λ_i ← max(0, λ_i + α(E[c_i] - d_i))
```

**解释**：
- 成本超支 (E[c_i] > d_i)：λ_i 增加
- 成本有余 (E[c_i] < d_i)：λ_i 减少
- λ_i ≥ 0（非负）

**直观理解**：
```
λ_i 是约束的"价格"
违反约束→价格上涨→更重视约束
满足约束→价格下降→更重视回报
```

**实现代码**：
```python
# 策略损失
policy_loss = -value.mean()
for i in range(n_constraints):
    policy_loss += lambdas[i] * (cost_i.mean() - threshold_i)

# 乘子更新
for i in range(n_constraints):
    violation = cost_i.mean() - threshold_i
    lambdas[i] = max(0, lambdas[i] + lambda_lr * violation)
```

**优点**：
- 简单实现
- 广泛使用
- 可处理多约束

**缺点**：
- 不保证硬约束
- 可能振荡
- λ调参困难

**改进方法**：
1. 自适应λ学习率
2. 投影梯度
3. 原始 - 对偶方法

**评分要点：** 写出拉格朗日函数得基本分，解释乘子更新得高分

## 题目 3：CPO 算法
**问题：** CPO（约束策略优化）的核心思想是什么？如何保证约束满足？

**参考答案：**
**CPO（Constrained Policy Optimization）**：
信任域方法扩展到约束 RL，保证单调改进和约束满足。

**优化问题**：
```
max_θ E[A_π_θ(a|s)]
s.t. KL(π_θ_old || π_θ) ≤ δ
     E[cost] ≤ d
```

**核心思想**：
```
1. 信任域约束：策略变化不大
2. 成本约束：满足约束
3. 近似求解：二阶近似
```

**如何保证约束满足**：

**1. 线性近似**：
```
成本约束线性化
g^T Δθ ≤ d - J_cost(π_old)
```

**2. 投影**：
```
在可行域内更新
投影到约束边界
```

**3. 回溯线搜索**：
```
检查约束满足
不满足则回溯
```

**求解步骤**：
```
1. 计算梯度 g 和 Hessian H
2. 求解无约束方向：Δθ_unc = H^{-1} g
3. 检查约束：
   - 满足：接受
   - 不满足：投影到约束
4. 线搜索确保改进
```

**优点**：
- 理论保证
- 几乎满足约束
- 单调改进

**缺点**：
- 实现复杂
- 计算成本高
- 二阶近似误差

**与拉格朗日对比**：

| 方面 | 拉格朗日 | CPO |
|------|----------|-----|
| **约束保证** | 软保证 | 几乎硬 |
| **实现** | 简单 | 复杂 |
| **计算** | 低 | 高 |
| **理论** | 弱 | 强 |

**实践建议**：
- 简单应用：拉格朗日
- 安全关键：CPO
- 资源有限：拉格朗日

**评分要点：** 说出 CPO 优化问题得基本分，解释约束保证得高分

## 题目 4：原始 - 对偶方法
**问题：** 原始 - 对偶方法如何求解约束 RL？与拉格朗日有什么区别？

**参考答案：**
**原始 - 对偶方法**：
同时优化原始变量（策略）和对偶变量（乘子）。

**拉格朗日函数**：
```
L(π, λ) = E[回报] - Σ λ_i (E[成本_i] - d_i)
```

**原始 - 对偶更新**：
```
策略（原始）：θ ← θ + α_θ ∇_θ L  （最大化）
乘子（对偶）：λ ← λ - α_λ ∇_λ L  （最小化）
```

**具体更新**：
```
θ ← θ + α_θ ∇_θ (E[r] - Σ λ_i E[c_i])
λ_i ← max(0, λ_i - α_λ (E[c_i] - d_i))
```

**与拉格朗日方法的区别**：

| 方面 | 拉格朗日 | 原始 - 对偶 |
|------|----------|-------------|
| **更新方式** | 交替 | 同时 |
| **策略更新** | 固定λ | λ同时变 |
| **收敛** | 可能慢 | 通常快 |
| **稳定性** | 较好 | 可能振荡 |
| **实现** | 简单 | 稍复杂 |

**拉格朗日方法**：
```
for iteration in range(N):
    # 固定λ，更新θ
    for _ in range(k):
        θ ← θ + α ∇_θ L(θ, λ_fixed)
    
    # 固定θ，更新λ
    λ ← max(0, λ + β ∇_λ L(θ_fixed, λ))
```

**原始 - 对偶方法**：
```
for iteration in range(N):
    # 同时更新
    θ ← θ + α_θ ∇_θ L(θ, λ)
    λ ← max(0, λ - α_λ ∇_λ L(θ, λ))
```

**优点**：
- 同时优化
- 通常收敛快
- 适合在线学习

**缺点**：
- 需要调两个学习率
- 可能不稳定
- 理论复杂

**实践建议**：
- 小问题：拉格朗日
- 大问题：原始 - 对偶
- 在线学习：原始 - 对偶

**评分要点：** 说出同时更新得基本分，对比拉格朗日得高分

## 题目 5：实际应用
**问题：** 约束 RL 在实际应用中有什么最佳实践？如何处理约束冲突？

**参考答案：**
**最佳实践**：

**1. 约束优先级**：
```
硬约束：必须满足（安全）
软约束：尽量满足（性能）
优先级处理
```

**2. 约束松弛**：
```
初期：宽松约束
后期：收紧约束
渐进满足
```

**3. 监控指标**：
```
约束违反率
平均成本
拉格朗日乘子
```

**4. 可行初始化**：
```
从可行策略开始
避免初始违反
```

**5. 约束归一化**：
```
不同量纲约束
归一化到相同尺度
```

**处理约束冲突**：

**1. 优先级排序**：
```
约束 1 > 约束 2 > ...
先满足高优先级
```

**2. 加权组合**：
```
Σ w_i (E[c_i] - d_i)
权重反映重要性
```

**3. 分层优化**：
```
先优化高优先级
再优化低优先级
```

**4. 可行性检查**：
```
检查是否存在可行解
无解时放松约束
```

**5. 交互式调整**：
```
用户调整约束
实时反馈
```

**实践流程**：
```
1. 识别所有约束
2. 分类（硬/软）
3. 设定阈值
4. 训练并监控
5. 调整冲突约束
6. 部署验证
```

**应用案例**：

**数据中心冷却**：
```
目标：最小化能耗
约束：
- 温度 < 25°C（硬）
- 湿度 40-60%（软）
- 噪音 < 50dB（软）
```

**自动驾驶**：
```
目标：最小化时间
约束：
- 不碰撞（硬）
- 加速度 < 3m/s²（软）
- 车道保持（硬）
```

**监控告警**：
```
约束违反>阈值→告警
乘子过大→检查约束
性能下降→调整权重
```

**评分要点：** 说出优先级处理得基本分，详细处理冲突得高分
