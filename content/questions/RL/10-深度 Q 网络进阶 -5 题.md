## 题目 1：经验回放
**问题：** 为什么 DQN 需要经验回放？它解决了什么问题？有什么优缺点？

**参考答案：**
**经验回放（Experience Replay）**：
将智能体的经验 (s, a, r, s', done) 存储到缓冲区，训练时随机采样。

**解决的问题**：

**1. 打破时间相关性**：
- 连续的经验高度相关
- 直接训练导致梯度下降不稳定
- 随机采样打破相关性，满足 IID 假设

**2. 提高样本效率**：
- 每个经验可多次使用
- 对于昂贵/危险的经验尤其重要
- 从过去经验反复学习

**3. 平滑数据分布**：
- 避免遗忘早期经验
- 学习更稳定的策略

**优点**：
- 稳定训练
- 提高样本效率
- 允许离线学习

**缺点**：
- 内存消耗大
- 可能学习过时策略
- 不适合非平稳环境

**变体**：
- 均匀采样（标准）
- 优先级采样（Prioritized Replay）
-  hindsight 经验回放（HER）

**评分要点：** 说出打破相关性得基本分，分析优缺点得高分

## 题目 2：目标网络
**问题：** DQN 中目标网络的作用是什么？为什么需要定期更新而不是每步更新？

**参考答案：**
**目标网络的作用**：

**问题**：标准 Q-Learning 用同一网络计算目标和预测
```
y = r + γ max Q(s', a; θ)
```
- 目标随θ变化
- 导致"移动目标"问题
- 训练不稳定，可能发散

**解决方案**：使用独立的目标网络θ^-
```
y = r + γ max Q(s', a; θ^-)
```
- θ^- 定期从θ复制
- 短期内目标固定
- 训练更稳定

**为什么定期更新**：

**每步更新的后果**：
- 目标变化太快
- 类似"追自己尾巴"
- 梯度震荡，难以收敛

**定期更新的好处**：
- 目标在短期内稳定
- 类似监督学习（固定标签）
- 允许梯度下降收敛

**更新频率选择**：
- 太小（如 10 步）：目标仍不稳定
- 太大（如 10000 步）：目标过时
- 经验值：1000-10000 步

**软更新（DDPG）**：
```
θ^- ← τθ + (1-τ)θ^-, τ << 1
```
- 每步缓慢更新
- 更平滑的目标变化

**评分要点：** 说出"移动目标"问题得基本分，解释软更新得高分

## 题目 3：损失函数
**问题：** DQN 使用什么损失函数？为什么有时用 Huber 损失代替 MSE？

**参考答案：**
**标准损失（MSE）**：
```
L(θ) = E[(y - Q(s,a; θ))²]
```
其中 y = r + γ max Q'(s',·) 是目标。

**Huber 损失**：
```
L_δ(x) = {
    0.5x²           如果 |x| < δ
    δ(|x| - 0.5δ)   否则
}
```
通常δ=1。

**为什么用 Huber 损失**：

**MSE 的问题**：
- 对异常值敏感
- 大 TD 误差导致大梯度
- 可能破坏已学习的知识

**Huber 的优势**：
1. **鲁棒性**：
   - 小误差：二次（类似 MSE）
   - 大误差：线性（类似 MAE）
   - 减少异常值影响

2. **梯度稳定**：
   ```
   ∇L_δ(x) = {
       x       如果 |x| < δ
       δ·sign(x) 否则
   }
   ```
   - 大误差时梯度有界
   - 防止梯度爆炸

3. **可导**：
   - 在 x=±δ处连续可导
   - 适合梯度下降

**实践建议**：
- 初期：Huber（稳定）
- 后期：MSE（精确）
- 或全程 Huber（鲁棒）

**评分要点：** 写出 Huber 公式得基本分，解释梯度稳定得高分

## 题目 4：代码实现
**问题：** 请实现 DQN 的核心训练循环，包括经验回放和目标网络更新。

**参考答案：**
```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import numpy as np
from collections import deque

class DQNAgent:
    def __init__(self, state_dim, action_dim, gamma=0.99, lr=1e-4,
                 buffer_size=10000, batch_size=32, target_update=1000):
        self.gamma = gamma
        self.batch_size = batch_size
        self.target_update = target_update
        self.steps = 0
        
        # 网络
        self.q_net = DQN(state_dim, action_dim)
        self.q_target = DQN(state_dim, action_dim)
        self.q_target.load_state_dict(self.q_net.state_dict())
        
        self.optimizer = torch.optim.Adam(self.q_net.parameters(), lr=lr)
        self.buffer = deque(maxlen=buffer_size)
    
    def select_action(self, state, epsilon=0.1):
        if np.random.random() < epsilon:
            return np.random.randint(self.q_net.net[-1].out_features)
        with torch.no_grad():
            state = torch.FloatTensor(state).unsqueeze(0)
            return torch.argmax(self.q_net(state)).item()
    
    def store_transition(self, state, action, reward, next_state, done):
        self.buffer.append((state, action, reward, next_state, done))
    
    def update(self):
        if len(self.buffer) < self.batch_size:
            return None
        
        # 随机采样
        batch = np.random.choice(len(self.buffer), self.batch_size, replace=False)
        transitions = [self.buffer[i] for i in batch]
        states, actions, rewards, next_states, dones = zip(*transitions)
        
        # 转 tensor
        states = torch.FloatTensor(states)
        actions = torch.LongTensor(actions).unsqueeze(1)
        rewards = torch.FloatTensor(rewards)
        next_states = torch.FloatTensor(next_states)
        dones = torch.FloatTensor(dones)
        
        # 计算目标（关键：用目标网络）
        with torch.no_grad():
            next_q = self.q_target(next_states).max(dim=1)[0]
            targets = rewards + self.gamma * next_q * (1 - dones)
        
        # 计算当前 Q
        current_q = self.q_net(states).gather(1, actions).squeeze()
        
        # 损失
        loss = F.mse_loss(current_q, targets)
        
        # 更新
        self.optimizer.zero_grad()
        loss.backward()
        nn.utils.clip_grad_norm_(self.q_net.parameters(), 10)
        self.optimizer.step()
        
        # 定期更新目标网络
        self.steps += 1
        if self.steps % self.target_update == 0:
            self.q_target.load_state_dict(self.q_net.state_dict())
        
        return loss.item()
    
    def train_episode(self, env, epsilon=0.1):
        state = env.reset()
        total_reward = 0
        
        while True:
            action = self.select_action(state, epsilon)
            next_state, reward, done, _ = env.step(action)
            
            self.store_transition(state, action, reward, next_state, done)
            self.update()
            
            total_reward += reward
            state = next_state
            
            if done:
                break
        
        return total_reward
```

**关键点**：
1. 经验存储到 deque
2. 随机采样 batch
3. 目标网络计算 target
4. 定期更新目标网络

**评分要点：** 代码正确得基本分，处理目标网络得高分

## 题目 5：训练稳定性
**问题：** DQN 训练不稳定的原因有哪些？如何提高稳定性？

**参考答案：**
**不稳定原因**：

**1. 移动目标问题**：
- Q 网络同时用于预测和计算目标
- 目标随参数变化
- 解决：目标网络

**2. 数据相关性**：
- 连续经验相关
- 梯度估计有偏
- 解决：经验回放

**3. Q 值高估**：
- max 操作导致系统性高估
- 误差累积
- 解决：Double DQN

**4. 梯度爆炸**：
- 大 TD 误差→大梯度
- 破坏网络权重
- 解决：梯度裁剪

**5. 探索不足**：
- 陷入局部最优
- 解决：ε调度、Noisy Nets

**提高稳定性的方法**：

| 技术 | 作用 | 实现 |
|------|------|------|
| **目标网络** | 固定目标 | 定期复制θ→θ^- |
| **经验回放** | 打破相关 | 随机采样 batch |
| **梯度裁剪** | 防止爆炸 | clip_grad_norm |
| **Huber 损失** | 鲁棒学习 | 替代 MSE |
| **奖励缩放** | 限制范围 | sign(r)*min(|r|,1) |
| **学习率衰减** | 精细调整 | 逐步降低 lr |
| **Double DQN** | 减少高估 | 解耦选择评估 |

**实践建议**：
1. 必须：目标网络 + 经验回放
2. 推荐：梯度裁剪 + Huber 损失
3. 进阶：Double DQN + Dueling

**调试技巧**：
- 监控平均 TD 误差
- 检查 Q 值分布
- 可视化学习曲线

**评分要点：** 说出 3 个以上原因得基本分，给出完整方案得高分
