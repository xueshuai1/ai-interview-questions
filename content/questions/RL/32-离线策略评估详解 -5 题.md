# 离线策略评估详解面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是离线策略评估？为什么它在实际应用中如此重要？

**参考答案：**
**离线策略评估（Off-Policy Evaluation, OPE）**：
使用历史数据评估新策略的性能，无需在线部署。

**核心问题**：
```
给定历史数据 D（由行为策略 b 收集）
评估目标策略π的性能η(π)
无需在线执行π
```

**为什么重要**：

**1. 安全考虑**：
```
医疗：不能拿患者测试新策略
自动驾驶：不能冒险测试
金融：避免重大损失
```

**2. 成本考虑**：
```
在线测试成本高
流量/资源有限
OPE 成本低
```

**3. 速度考虑**：
```
在线测试需要时间
OPE 快速评估
快速迭代
```

**4. 合规考虑**：
```
某些领域禁止在线测试
需要离线验证
法规要求
```

**评估方法分类**：

**1. 重要性采样**：
```
无偏估计
高方差
```

**2. 直接方法**：
```
有偏估计
低方差
```

**3. 双重鲁棒**：
```
偏差 - 方差权衡
实践中常用
```

**挑战**：
1. 分布偏移（b≠π）
2. 高方差（长序列）
3. 支持覆盖（π(a|s)>0 需要 b(a|s)>0）

**评分要点：** 说出无需在线部署得基本分，解释重要性得高分

## 题目 2：重要性采样
**问题：** 重要性采样如何评估离线策略？有什么优缺点？

**参考答案：**
**重要性采样（Importance Sampling）**：
用重要性权重校正分布偏移。

**核心公式**：
```
η(π) = E_b[ρ · G]
ρ = Π_t π(a_t|s_t) / b(a_t|s_t)
```

**估计器**：
```
η_IS(π) = (1/n) Σ_{i=1}^n ρ_i · G_i
```

**工作原理**：
```
1. 计算每个轨迹的重要性权重ρ
2. 用ρ加权回报 G
3. 平均得到估计
```

**优点**：

**1. 无偏估计**：
```
E[η_IS] = η(π)
理论上准确
```

**2. 简单实现**：
```
只需策略概率
无需模型
```

**3. 一致性**：
```
样本增多→收敛到真值
```

**缺点**：

**1. 高方差**：
```
权重可能很大
尤其是长序列
方差随序列长度指数增长
```

**2. 需要支持覆盖**：
```
π(a|s)>0 需要 b(a|s)>0
否则权重无穷大
```

**3. 权重退化**：
```
长序列权重趋近 0 或∞
数值不稳定
```

**改进方法**：

**1. 加权 IS（WIS）**：
```
η_WIS = Σ w_i G_i / Σ w_i
有偏但方差低
```

**2. 截断 IS**：
```
权重截断到阈值
减少方差
```

**3. 每步决策 IS**：
```
每步独立权重
方差更低
```

**实践建议**：
- 短序列：普通 IS
- 长序列：WIS 或 PDIS
- 监控权重分布

**评分要点：** 写出权重公式得基本分，解释方差问题得高分

## 题目 3：双重鲁棒
**问题：** 双重鲁棒估计器如何工作？为什么它被称为"双重鲁棒"？

**参考答案：**
**双重鲁棒（Doubly Robust, DR）**：
结合重要性采样和直接方法，只要一个正确就能得到准确估计。

**DR 估计器**：
```
η_DR = (1/n) Σ [
    Q^π(s_0, π(s_0)) +
    Σ_t ρ_{0:t} (r_t + Q^π(s_{t+1}, π(s_{t+1})) - Q^π(s_t, a_t))
]
```

**工作原理**：
```
1. 直接方法估计：Q^π
2. 用 IS 校正残差
3. 结合两者
```

**为什么"双重鲁棒"**：

**情况 1：Q 函数准确**：
```
即使重要性权重不准
DR 仍准确（因为 Q 准）
```

**情况 2：重要性权重准确**：
```
即使 Q 函数不准
DR 仍准确（因为 IS 准）
```

**只要一个准确→DR 准确**：
```
故名"双重鲁棒"
```

**优点**：

**1. 偏差 - 方差权衡**：
```
比 IS 方差低
比 DM 偏差低
```

**2. 鲁棒性**：
```
一个方法失效另一个补偿
更可靠
```

**3. 实践有效**：
```
 empirically best
广泛使用
```

**缺点**：

**1. 需要 Q 函数**：
```
需要训练价值函数
增加复杂度
```

**2. 实现复杂**：
```
比 IS 复杂
需要仔细实现
```

**3. 仍需要支持覆盖**：
```
b(a|s)>0 要求
```

**实践建议**：
- 首选 DR
- 准备好 Q 函数
- 监控两个组件

**评分要点：** 说出结合 IS 和 DM 得基本分，解释双重鲁棒含义得高分

## 题目 4：置信区间
**问题：** 如何计算 OPE 估计的置信区间？为什么它重要？

**参考答案：**
**置信区间**：
量化估计的不确定性。

**为什么重要**：

**1. 决策支持**：
```
点估计不够
需要知道不确定性
```

**2. 统计显著性**：
```
新策略是否真的更好？
需要统计检验
```

**3. 风险评估**：
```
最坏情况性能
下界估计
```

**计算方法**：

**1. Bootstrap**：
```
重采样数据
计算多个估计
取分位数
```

**实现**：
```python
def bootstrap_ci(estimates, n_bootstrap=1000, alpha=0.05):
    bootstrap_means = []
    for _ in range(n_bootstrap):
        sample = np.random.choice(estimates, len(estimates), replace=True)
        bootstrap_means.append(np.mean(sample))
    
    lower = np.percentile(bootstrap_means, alpha/2 * 100)
    upper = np.percentile(bootstrap_means, (1-alpha/2) * 100)
    return lower, upper
```

**2. 正态近似**：
```
CI = mean ± z · std/√n
```

**3. 学生 t 分布**：
```
小样本用 t 分布
更准确
```

**解释置信区间**：
```
95% CI: [0.8, 1.2]
含义：
- 真值有 95% 概率在区间内
- 或重复实验 95% 区间包含真值
```

**决策规则**：
```
新策略 CI 下限 > 当前策略→部署
否则→不部署或更多数据
```

**实践建议**：
- 始终报告 CI
- 用 Bootstrap
- 考虑最坏情况

**评分要点：** 说出 Bootstrap 得基本分，解释决策意义得高分

## 题目 5：实际应用
**问题：** OPE 在实际应用中有什么最佳实践？如何选择合适的评估方法？

**参考答案：**
**最佳实践**：

**1. 多方法对比**：
```
同时用 IS、DM、DR
对比结果
一致性高→可信
```

**2. 敏感性分析**：
```
改变假设
看结果变化
鲁棒性检查
```

**3. 分层分析**：
```
按状态/用户分层
识别问题区域
```

**4. 置信区间**：
```
始终报告 CI
不确定性量化
```

**5. 数据质量检查**：
```
检查支持覆盖
识别分布偏移
数据清洗
```

**方法选择决策框架**：

**1. 评估序列长度**：
```
短序列（<10 步）：IS
长序列：WIS 或 DR
```

**2. 评估数据量**：
```
大数据：IS（方差可降低）
小数据：DR（方差低）
```

**3. 评估 Q 函数质量**：
```
Q 准：DR
Q 不准：IS 或 WIS
```

**4. 评估支持覆盖**：
```
覆盖好：任何方法
覆盖差：需要谨慎
```

**选择指南**：

| 场景 | 推荐方法 |
|------|----------|
| 短序列，大数据 | IS |
| 长序列，小数据 | DR |
| Q 函数准确 | DR |
| Q 函数不准 | WIS |
| 需要保守估计 | WIS 下界 |

**实践流程**：
```
1. 数据质量检查
2. 选择多种方法
3. 计算估计和 CI
4. 对比结果
5. 敏感性分析
6. 决策建议
```

**报告模板**：
```
策略评估报告：
- 点估计：X.XX
- 95% CI: [X.XX, X.XX]
- 方法：DR
- 样本量：N
- 建议：部署/不部署
```

**评分要点：** 说出多方法对比得基本分，详细决策框架得高分
