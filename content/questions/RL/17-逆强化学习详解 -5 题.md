# 逆强化学习详解面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是逆强化学习？它与模仿学习有什么本质区别？

**参考答案：**
**逆强化学习（IRL）**：
从专家演示中推断奖励函数的问题，试图理解专家行为背后的目标和偏好。

**本质区别**：

| 方面 | 模仿学习 | 逆强化学习 |
|------|----------|------------|
| **输出** | 策略π(s)→a | 奖励函数 R(s,a) |
| **目标** | 复制行为 | 理解意图 |
| **方法** | 监督/对抗学习 | 优化/概率推断 |
| **泛化** | 有限（相似情境） | 更好（理解目标） |
| **可解释** | 低（黑盒策略） | 高（奖励可解释） |

**直观理解**：
- 模仿学习："专家怎么做，我就怎么做"
- 逆强化学习："专家为什么这么做"

**示例**：
```
自动驾驶场景：
- 模仿学习：看到红灯就刹车
- 逆强化学习：推断"避免碰撞"的奖励
  → 红灯刹车，行人也刹车，障碍物也刹车
```

**IRL 优势**：
1. **泛化能力**：理解目标后可泛化到新情境
2. **可解释性**：奖励函数揭示偏好
3. **组合性**：奖励可组合、迁移
4. **安全性**：理解约束和偏好

**IRL 挑战**：
1. **不适定问题**：多个奖励函数可解释同一行为
2. **计算复杂**：需要反复求解 MDP
3. **奖励歧义**：R=0 解释所有行为

**评分要点：** 说出推断奖励得基本分，详细对比得高分

## 题目 2：最大熵 IRL
**问题：** 最大熵 IRL 的核心思想是什么？如何形式化？

**参考答案：**
**最大熵 IRL 核心思想**：
在满足专家演示约束的前提下，选择最大熵的轨迹分布，避免做出不必要的假设。

**形式化**：

**轨迹概率**：
```
P(τ|w) = (1/Z(w)) · exp(Σ_t R(s_t; w))
```
其中：
- `R(s; w)`：参数化奖励函数
- `Z(w)`：配分函数（归一化）

**优化目标**：
```
max_w E_τ∼D_E[log P(τ|w)]
```

**展开**：
```
L(w) = E_τ∼D_E[Σ_t R(s_t; w)] - log Z(w)
```

**梯度**：
```
∇_w L(w) = E_τ∼D_E[Σ_t ∇_w R(s_t; w)] - E_τ∼P(τ|w)[Σ_t ∇_w R(s_t; w)]
```

**含义**：
- 第一项：专家演示的特征期望
- 第二项：当前模型的特征期望
- 梯度下降使两者匹配

**配分函数问题**：
```
Z(w) = Σ_τ exp(Σ_t R(s_t; w))
```
- 指数级数量
- 需要近似（采样、变分）

**实践方法**：
- 用 MCMC 采样近似
- 或用对抗方法（AIRL）

**评分要点：** 写出概率公式得基本分，解释梯度含义得高分

## 题目 3：奖励歧义
**问题：** 逆强化学习中奖励歧义问题是什么？如何解决？

**参考答案：**
**奖励歧义问题**：
多个不同的奖励函数可以解释相同的专家行为，导致解不唯一。

**极端情况**：
```
R(s) = 0  ∀s
```
- 所有策略都是最优的
- 可以解释任何专家行为
- 但无意义

**其他歧义**：
```
R'(s) = R(s) + c（常数偏移）
R''(s) = k·R(s)（缩放）
```
- 最优策略不变
- 但奖励函数不同

**解决方法**：

**1. 最大熵原则**：
- 选择最"不确定"的奖励
- 避免过度拟合
- Ziebart et al. 最大熵 IRL

**2. 正则化**：
```
max_w E[奖励] - λ||w||²
```
- 偏好简单奖励
- 避免复杂过拟合

**3. 多专家数据**：
- 多个专家的行为约束
- 减少歧义空间

**4. 主动学习**：
- 查询专家偏好
- 区分歧义奖励

**5. 结构约束**：
- 奖励函数形式约束
- 如稀疏奖励、线性奖励

**6. 层次奖励**：
- 内在奖励 + 外在奖励
- 分解不同动机

**实践建议**：
- 用最大熵框架
- 添加正则化
- 收集多样数据
- 结合先验知识

**评分要点：** 说出 R=0 问题得基本分，提出多种解决得高分

## 题目 4：AIRL 算法
**问题：** AIRL 是什么？它如何结合 IRL 和对抗学习？

**参考答案：**
**AIRL（Adversarial Inverse Reinforcement Learning）**：
用对抗框架学习可解释的奖励函数，结合 IRL 和 GAN 思想。

**核心思想**：
- 判别器学习奖励函数
- 生成器（策略）在奖励下优化
- 对抗训练使策略匹配专家

**算法框架**：
```
判别器 D(s,a) = exp(R(s,a)) / (exp(R(s,a)) + π(a|s))
```

**训练**：
```
min_R max_π E_E[log D(s,a)] + E_π[log(1-D(s,a))]
```

**与 GAIL 的区别**：

| 方面 | GAIL | AIRL |
|------|------|------|
| **判别器** | D(s,a)∈[0,1] | 与策略相关 |
| **输出** | 隐式奖励 | 显式奖励 R(s,a) |
| **可解释** | 低 | 高 |
| **奖励形式** | 黑盒 | 可分解 |

**AIRL 优势**：
1. **可解释奖励**：显式学习 R(s,a)
2. **奖励分解**：可分离状态和动作
3. **迁移能力**：奖励可迁移到新环境
4. **理论保证**：收敛到专家奖励

**实现要点**：
```python
# 判别器（奖励）
D(s,a) = exp(R(s,a)) / (exp(R(s,a)) + π(a|s))

# 奖励更新
loss = -E_E[log D] - E_π[log(1-D)]

# 策略更新
loss = -E_π[log D]  # 最大化判别器分数
```

**应用**：
- 机器人技能学习
- 人类偏好推断
- 奖励迁移

**评分要点：** 说出对抗框架得基本分，对比 GAIL 得高分

## 题目 5：应用场景
**问题：** 逆强化学习适合哪些应用场景？与直接设计奖励相比有什么优势？

**参考答案：**
**适合的应用场景**：

**1. 人类偏好学习**：
- 个性化推荐
- 用户行为建模
- 难以明确表达偏好

**2. 机器人学习**：
- 从演示学习目标
- 复杂任务（如烹饪）
- 安全约束学习

**3. 自动驾驶**：
- 学习驾驶风格
- 理解交通规则
- 道德决策

**4. 医疗决策**：
- 从医生决策学习
- 治疗方案偏好
- 风险 - 收益权衡

**5. 游戏 AI**：
- 理解玩家目标
- NPC 行为生成
- 平衡游戏设计

**与手动设计奖励相比的优势**：

| 方面 | 手动设计 | 逆强化学习 |
|------|----------|------------|
| **工作量** | 高（反复调试） | 低（从数据学习） |
| **准确性** | 可能偏差 | 反映真实偏好 |
| **复杂性** | 难以处理复杂 | 自动学习复杂 |
| **可解释** | 高（人工设计） | 中（学习得到） |
| **泛化** | 有限 | 更好 |

**具体优势**：

**1. 避免奖励工程**：
- 手动设计奖励耗时
- IRL 自动从数据学习
- 减少人工干预

**2. 捕捉隐含偏好**：
- 人类难以明确表达
- IRL 从行为推断
- 发现隐含约束

**3. 处理复杂任务**：
- 多目标权衡
- 时序依赖
- 情境依赖

**4. 个性化**：
- 不同用户不同偏好
- IRL 学习个体差异
- 定制化系统

**局限性**：
- 需要高质量演示
- 计算成本高
- 奖励歧义问题

**最佳实践**：
- 结合手动和自动
- 用 IRL 初始化
- 人工微调

**评分要点：** 说出 3 个以上场景得基本分，详细对比得高分
