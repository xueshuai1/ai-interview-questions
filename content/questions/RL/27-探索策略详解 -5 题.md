# 探索策略详解面试题（5 道）

## 题目 1：探索 - 利用权衡
**问题：** 什么是探索 - 利用权衡？为什么它是 RL 的核心问题？

**参考答案：**
**探索 - 利用权衡（Exploration-Exploitation Trade-off）**：
智能体需要在尝试新动作（探索）和选择已知最优动作（利用）之间做出权衡。

**探索（Exploration）**：
```
尝试新动作
目的：发现更好的策略
风险：可能得到更差回报
```

**利用（Exploitation）**：
```
选择已知最优动作
目的：最大化当前回报
风险：可能错过更好策略
```

**为什么是核心问题**：

**1. 局部最优风险**：
```
只利用不探索：
→ 陷入局部最优
→ 错过全局最优
```

**2. 效率问题**：
```
只探索不利用：
→ 学习速度慢
→ 浪费已知知识
```

**3. 长期影响**：
```
短期：利用可能更好
长期：探索可能发现更好策略
```

**4. 信息价值**：
```
探索获得信息
信息有长期价值
需要量化信息价值
```

**经典示例（多臂老虎机）**：
```
10 个老虎机，每个有不同胜率
策略 1（只利用）：
- 试每个一次
- 选择最好的，永远玩它
- 可能错过真正最好的

策略 2（只探索）：
- 随机选择
- 永远不知道哪个最好
- 回报低

策略 3（平衡）：
- 初期多探索
- 后期多利用
- 最优
```

**量化权衡**：
```
Regret = 最优回报 - 实际回报
目标：最小化累积 Regret
```

**评分要点：** 说出探索和利用定义得基本分，解释局部最优得高分

## 题目 2：ε-贪婪
**问题：** ε-贪婪策略的工作原理是什么？如何调度ε？

**参考答案：**
**ε-贪婪（Epsilon-Greedy）**：
最简单的探索策略，以ε概率随机选择动作。

**工作原理**：
```python
if random() < ε:
    action = random_action()  # 探索
else:
    action = argmax Q(s,a)    # 利用
```

**概率分布**：
```
π(a|s) = {
    1-ε + ε/|A|  如果 a = argmax Q(s,a)
    ε/|A|        其他动作
}
```

**ε调度策略**：

**1. 固定ε**：
```
ε = 0.1  # 始终 10% 探索
简单但非最优
```

**2. 线性衰减**：
```
ε_t = max(ε_min, ε_start - t · decay)
```

**3. 指数衰减**：
```
ε_t = max(ε_min, ε_start · decay^t)
常用：decay = 0.995
```

**4. 自适应ε**：
```
根据学习进度调整
性能好→减少ε
性能差→增加ε
```

**参数选择**：
```
ε_start = 1.0      # 初期完全探索
ε_end = 0.01       # 后期 1% 探索
decay = 0.995      # 每步衰减
```

**优缺点**：

| 优点 | 缺点 |
|------|------|
| 简单实现 | 随机探索效率低 |
| 理论保证 | 不区分动作 |
| 广泛使用 | 固定概率非最优 |

**改进方法**：
1. **Softmax 探索**：按 Q 值概率选择
2. **UCB**：考虑不确定性
3. **NoisyNet**：参数噪声

**实践建议**：
- 从ε=1.0 开始
- 指数衰减到 0.01
- 根据任务调整

**评分要点：** 写出概率公式得基本分，解释调度策略得高分

## 题目 3：内在动机
**问题：** 内在动机探索的核心思想是什么？有哪些实现方法？

**参考答案：**
**内在动机（Intrinsic Motivation）**：
添加内在奖励鼓励探索，不依赖外部奖励。

**核心思想**：
```
总奖励 = 外在奖励 + λ·内在奖励
```

**实现方法**：

**1. 好奇心驱动（Curiosity）**：
```
内在奖励 = 预测误差
预测下一状态
误差大→新奇→奖励高
```

**实现**：
```python
# 正向模型
predicted_next_state = model(state, action)
intrinsic_reward = ||predicted - actual||²
```

**2. 计数-Based**：
```
内在奖励 = 1 / √N(s,a)
访问少→奖励高
```

**3. 熵最大化**：
```
max E[回报 + α·H(π)]
策略熵高→探索多
```

**4. 状态覆盖**：
```
鼓励访问新状态
最大化状态覆盖
```

**5. 信息增益**：
```
选择信息增益最大的动作
减少不确定性
```

**好奇心驱动详解**：

**正向模型**：
```
输入：(s, a)
输出：ŝ'（预测的下一状态）
```

**内在奖励**：
```
r_intrinsic = ||ŝ' - s'||²
```

**训练**：
```
最小化预测误差
但用预测误差作为奖励
```

**效果**：
- 自动探索新奇状态
- 无需外部奖励
- 适合稀疏奖励任务

**挑战**：
1. **噪声问题**：
   - 随机噪声也产生大误差
   - 可能探索无意义区域

2. **可预测环境**：
   - 环境完全可预测后
   - 内在奖励消失

3. **计算成本**：
   - 需要训练额外模型
   - 增加复杂度

**解决方案**：
- 特征空间预测（而非原始状态）
- 结合外在奖励
- 自适应内在奖励权重

**成功案例**：
- Montezuma's Revenge（Atari）
- 3D 导航任务
- 稀疏奖励机器人任务

**评分要点：** 说出内在奖励公式得基本分，列举多种方法得高分

## 题目 4：NoisyNet
**问题：** NoisyNet 如何实现探索？它与ε-贪婪有什么区别？

**参考答案：**
**NoisyNet（Noisy Network）**：
在神经网络参数中添加噪声，实现参数空间探索。

**实现原理**：

**参数化噪声**：
```
W = μ_W + σ_W ⊙ ε_W
b = μ_b + σ_b ⊙ ε_b
```
其中ε∼N(0,1)是随机噪声。

**前向传播**：
```python
def forward(self, x):
    # 采样噪声
    weight_noise = torch.randn_like(self.weight_sigma)
    bias_noise = torch.randn_like(self.bias_sigma)
    
    # 参数化噪声
    weight = self.weight_mu + self.weight_sigma * weight_noise
    bias = self.bias_mu + self.bias_sigma * bias_noise
    
    return linear(x, weight, bias)
```

**训练**：
```
学习μ和σ（噪声参数）
不是学习 W 本身
```

**与ε-贪婪的区别**：

| 方面 | ε-贪婪 | NoisyNet |
|------|--------|----------|
| **探索空间** | 动作空间 | 参数空间 |
| **探索方式** | 随机动作 | 参数噪声 |
| **状态依赖** | 否 | 是 |
| **训练** | 不需要 | 联合训练 |
| **超参数** | ε | 噪声初始值 |

**NoisyNet 优势**：

**1. 状态依赖探索**：
```
不同状态有不同噪声效果
更智能的探索
```

**2. 无需调度**：
```
ε需要衰减调度
NoisyNet 自动学习噪声大小
```

**3. 端到端训练**：
```
探索策略与 Q 函数联合优化
```

**4. 连续动作友好**：
```
ε-贪婪不适合连续动作
NoisyNet 天然支持
```

**实现要点**：

**1. 噪声参数初始化**：
```
σ_init = 0.5  # 初始噪声标准差
```

**2. 独立噪声**：
```
每个 episode 采样新噪声
episode 内固定
```

**3. 评估模式**：
```
测试时用μ（无噪声）
或平均多个噪声样本
```

**效果**：
```
Atari：优于ε-贪婪
连续控制：天然支持
```

**评分要点：** 说出参数噪声得基本分，对比ε-贪婪得高分

## 题目 5：实际应用
**问题：** 在实际应用中如何选择合适的探索策略？有什么最佳实践？

**参考答案：**
**选择探索策略的决策框架**：

**1. 评估任务类型**：
```
离散动作→ε-贪婪、UCB
连续动作→噪声、NoisyNet
```

**2. 评估奖励稀疏度**：
```
稠密奖励→简单探索
稀疏奖励→内在动机
```

**3. 评估环境复杂度**：
```
简单环境→ε-贪婪
复杂环境→高级探索
```

**4. 评估计算资源**：
```
资源有限→简单方法
资源充足→复杂方法
```

**方法选择指南**：

| 场景 | 推荐方法 |
|------|----------|
| Atari 游戏 | ε-贪婪衰减 |
| 连续控制 | 高斯噪声、NoisyNet |
| 稀疏奖励 | 内在动机、好奇心 |
| 多臂老虎机 | UCB、Thompson |
| 大规模状态 | 计数-Based、哈希 |

**最佳实践**：

**1. 从简单开始**：
```
先试ε-贪婪
不够再试高级方法
```

**2. 组合方法**：
```
ε-贪婪 + 内在奖励
NoisyNet + 熵正则化
```

**3. 自适应调整**：
```
根据学习进度调整
探索不足→增加
探索过度→减少
```

**4. 监控指标**：
```
状态覆盖率
动作多样性
Q 值分布
```

**5. 环境特定**：
```
考虑环境特性
定制探索策略
```

**实践流程**：
```
1. 分析任务特性
2. 选择基线方法
3. 训练并监控
4. 分析探索行为
5. 调整或升级方法
```

**成功案例**：
```
DQN（Atari）:
- ε-贪婪
- ε从 1.0 衰减到 0.01
- 1M 帧内收敛

SAC（连续控制）:
- 熵正则化
- 自动温度调节
- 高效探索

ICM（稀疏奖励）:
- 好奇心驱动
- Montezuma's Revenge 突破
```

**常见陷阱**：
1. ε衰减太快→探索不足
2. 内在奖励权重太大→忽略外在奖励
3. 噪声太大→训练不稳定
4. 不考虑任务特性→效果差

**评分要点：** 说出决策框架得基本分，给出最佳实践得高分
