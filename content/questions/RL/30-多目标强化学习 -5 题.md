# 多目标强化学习面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是多目标强化学习？什么是帕累托最优？

**参考答案：**
**多目标强化学习（Multi-Objective RL）**：
同时优化多个可能冲突的目标的强化学习方法。

**核心特点**：
```
奖励是向量：R = (r_1, r_2, ..., r_d)
目标：最大化向量值回报
```

**帕累托最优（Pareto Optimality）**：
一个解如果不被任何其他解支配，则是帕累托最优。

**支配关系**：
```
J_1 支配 J_2 当且仅当：
1. ∀i: J_1[i] ≥ J_2[i]（所有目标不差）
2. ∃i: J_1[i] > J_2[i]（至少一个目标更好）
```

**帕累托前沿**：
```
所有帕累托最优解的集合
在目标空间中形成前沿
```

**示例**：
```
两个目标：安全、效率

策略 A: (安全=0.9, 效率=0.5)
策略 B: (安全=0.8, 效率=0.8)
策略 C: (安全=0.7, 效率=0.9)

A 不支配 B（安全高但效率低）
B 不支配 C（安全低但效率高）
A、B、C 都是帕累托最优
```

**为什么需要多目标**：
```
真实世界问题：
- 自动驾驶：安全 vs 效率
- 机器人：精度 vs 速度
- 投资：收益 vs 风险

单目标无法捕捉权衡
```

**求解方法**：
1. 标量化（权重和）
2. 帕累托方法
3. 条件优化

**评分要点：** 说出支配关系得基本分，解释帕累托前沿得高分

## 题目 2：标量化方法
**问题：** 如何用标量化方法求解多目标 RL？有什么局限？

**参考答案：**
**标量化（Scalarization）**：
将多目标问题转化为单目标问题。

**线性标量化**：
```
R_scalar = Σ w_i · R_i
其中 w_i ≥ 0, Σ w_i = 1
```

**求解**：
```
max_π E[Σ γ^t Σ w_i · R_i(s_t, a_t)]
= max_π Σ w_i · E[Σ γ^t R_i(s_t, a_t)]
```

**权重选择**：
```
等权重：w = (1/d, ..., 1/d)
偏好权重：根据重要性
随机权重：探索帕累托前沿
```

**Chebyshev 标量化**：
```
R_scalar = min_i (w_i · R_i)
```
- 可找到非凸帕累托点
- 更复杂

**局限性**：

**1. 只能找到凸点**：
```
线性标量化
只能找到凸帕累托前沿
非凸部分找不到
```

**2. 权重敏感**：
```
权重微小变化
可能导致解大幅变化
不稳定
```

**3. 权重指定困难**：
```
需要事先知道偏好
实际中可能未知
```

**4. 单次运行一个解**：
```
一次权重→一个策略
要多个解需多次运行
```

**改进方法**：

**1. 多权重训练**：
```
同时训练多个权重
共享表示
```

**2. 条件策略**：
```
π(a|s, w)
输入权重输出策略
```

**3. 超网络**：
```
权重→策略参数
连续权重空间
```

**实践建议**：
- 简单问题：线性标量化
- 复杂问题：帕累托方法
- 未知偏好：探索前沿

**评分要点：** 写出线性标量化公式得基本分，解释局限得高分

## 题目 3：帕累托方法
**问题：** 帕累托方法如何求解多目标 RL？与标量化有什么区别？

**参考答案：**
**帕累托方法**：
直接维护帕累托前沿，不标量化。

**核心思想**：
```
维护非支配解集
更新时保持帕累托性质
```

**多目标 Q-Learning**：
```
Q(s,a) 是向量
更新：Q ← Q + α(R + γQ' - Q)
选择：帕累托支配
```

**算法流程**：
```
1. 初始化 Q 向量
2. 对于每个转移 (s,a,r,s')：
   a. 更新 Q(s,a) 向量
   b. 更新帕累托前沿
3. 选择动作：
   - 从帕累托前沿随机选
   - 或用标量化
```

**与标量化的区别**：

| 方面 | 标量化 | 帕累托方法 |
|------|--------|------------|
| **输出** | 单个策略 | 帕累托前沿 |
| **运行次数** | 多次（不同权重） | 一次 |
| **前沿形状** | 只能凸 | 任意形状 |
| **计算** | 简单 | 复杂 |
| **存储** | 少 | 多（多个解） |

**优势**：
1. **一次运行得前沿**：
   - 无需多次训练
   - 效率高

2. **非凸前沿**：
   - 可找到所有帕累托点
   - 更完整

3. **后期决策**：
   - 先学习前沿
   - 后选偏好

**挑战**：
1. **计算复杂度**：
   - 支配检查 O(n²)
   - 前沿可能很大

2. **动作选择**：
   - 多个帕累托动作
   - 如何选择？

3. **收敛性**：
   - 理论复杂
   - 实践有效

**实践建议**：
- 小问题：帕累托方法
- 大问题：标量化 + 条件策略
- 未知偏好：帕累托方法

**评分要点：** 说出维护前沿得基本分，对比标量化得高分

## 题目 4：条件策略
**问题：** 条件策略如何实现多目标 RL？有什么优势？

**参考答案：**
**条件策略（Conditional Policy）**：
策略以目标权重为条件：π(a|s, w)。

**核心思想**：
```
一个网络处理所有权重
输入：(s, w)
输出：a
```

**网络架构**：

**1. 权重拼接**：
```
x = concat(s, w)
π(x) → a
```

**2. 超网络**：
```
超网络输入 w
输出策略网络参数
```

**3. 注意力机制**：
```
权重作为注意力
聚焦相关目标
```

**实现示例**：
```python
class ConditionalPolicy(nn.Module):
    def __init__(self, state_dim, action_dim, n_objectives):
        super().__init__()
        self.weight_encoder = nn.Linear(n_objectives, 64)
        self.policy = nn.Sequential(
            nn.Linear(state_dim + 64, 128),
            nn.ReLU(),
            nn.Linear(128, action_dim)
        )
    
    def forward(self, state, weights):
        w_enc = self.weight_encoder(weights)
        x = torch.cat([state, w_enc], dim=1)
        return self.policy(x)
```

**优势**：

**1. 连续权重空间**：
```
任意权重都可处理
无需离散化
```

**2. 泛化能力**：
```
未见过的权重
也能合理应对
```

**3. 参数共享**：
```
所有权重共享网络
参数效率高
```

**4. 一次训练**：
```
训练时采样权重
一次训练覆盖所有
```

**训练方法**：
```
for iteration in range(N):
    # 随机采样权重
    w = sample_weights()
    
    # 用该权重收集数据
    data = collect_data(w)
    
    # 更新条件策略
    update_policy(data, w)
```

**应用**：
- 自动驾驶（安全 - 效率权衡）
- 机器人（精度 - 速度权衡）
- 推荐（多样 - 准确权衡）

**评分要点：** 说出权重条件得基本分，解释优势得高分

## 题目 5：实际应用
**问题：** 多目标 RL 在实际应用中如何指定目标权重？有什么最佳实践？

**参考答案：**
**权重指定方法**：

**1. 专家指定**：
```
领域专家根据经验
设定固定权重
```

**2. 偏好学习**：
```
从人类偏好学习
比较轨迹选权重
```

**3. 交互式优化**：
```
用户调整权重
实时查看结果
```

**4. 自动调整**：
```
根据性能自动调
满足约束
```

**最佳实践**：

**1. 分层目标**：
```
硬约束：必须满足
软目标：尽量优化
优先级处理
```

**2. 动态权重**：
```
初期：探索权重
后期：固定权重
或根据上下文调整
```

**3. 敏感性分析**：
```
测试不同权重
了解影响
选择鲁棒权重
```

**4. 可视化前沿**：
```
绘制帕累托前沿
帮助理解权衡
辅助决策
```

**5. 约束优先**：
```
先满足安全约束
再优化其他目标
```

**实践流程**：
```
1. 识别所有目标
2. 分类（硬/软）
3. 初始权重设定
4. 训练并评估
5. 根据反馈调整
6. 部署并监控
```

**应用案例**：

**自动驾驶**：
```
目标：安全、效率、舒适、能耗

权重：
- 城市：安全 0.5, 效率 0.2, 舒适 0.2, 能耗 0.1
- 高速：安全 0.3, 效率 0.4, 舒适 0.2, 能耗 0.1
```

**机器人操作**：
```
目标：精度、速度、能耗

权重：
- 精密操作：精度 0.7, 速度 0.2, 能耗 0.1
- 快速搬运：精度 0.2, 速度 0.6, 能耗 0.2
```

**监控指标**：
- 各目标性能
- 约束满足率
- 用户满意度
- 权衡合理性

**评分要点：** 说出多种指定方法得基本分，详细最佳实践得高分
