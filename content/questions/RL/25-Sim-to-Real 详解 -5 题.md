# Sim-to-Real 详解面试题（5 道）

## 题目 1：核心挑战
**问题：** Sim-to-Real 迁移的核心挑战是什么？Reality Gap 有哪些来源？

**参考答案：**
**核心挑战**：
仿真与真实世界存在差异（Reality Gap），导致仿真中学习的策略在真实世界失效。

**Reality Gap 来源**：

**1. 外观差异（Appearance Gap）**：
```
仿真：简单纹理、统一光照、干净背景
真实：复杂纹理、变化光照、杂乱背景
```

**2. 动力学差异（Dynamics Gap）**：
```
仿真：理想物理引擎、精确参数
真实：摩擦变化、弹性、空气阻力、模型误差
```

**3. 传感器差异（Sensor Gap）**：
```
仿真：完美观测、无噪声、无延迟
真实：传感器噪声、延迟、校准误差、失效
```

**4. 执行器差异（Actuator Gap）**：
```
仿真：理想执行、即时响应
真实：响应延迟、饱和、 backlash、磨损
```

**影响**：

**策略失效**：
- 仿真成功率 95%
- 真实成功率<50%

**安全问题**：
- 未预见行为
- 可能损坏设备

**解决方案方向**：
1. 域随机化
2. 系统识别
3. 域适应
4. 渐进迁移

**评分要点：** 说出 4 种差异得基本分，详细分析影响得高分

## 题目 2：域随机化
**问题：** 域随机化的核心思想是什么？如何设计随机化策略？

**参考答案：**
**域随机化（Domain Randomization）**：
训练时随机化仿真环境参数，学习鲁棒策略以适应真实变化。

**核心思想**：
```
如果策略能在各种随机化的仿真环境中工作
→ 也能在真实环境中工作
```

**随机化内容**：

**1. 外观随机化**：
```
- 物体纹理、颜色
- 光照位置、强度、颜色
- 背景图像
- 相机位置、角度
```

**2. 物理随机化**：
```
- 摩擦系数
- 质量、惯性
- 阻尼、弹性
- 重力
```

**3. 环境随机化**：
```
- 物体位置、方向
- 障碍物数量、位置
- 目标位置
```

**4. 动力学随机化**：
```
- 执行器延迟
- 传感器噪声
- 控制频率
```

**设计原则**：

**1. 覆盖真实范围**：
```
随机化范围应包含真实值
如：摩擦系数 [0.5, 2.0] 应包含真实摩擦
```

**2. 足够多样性**：
```
每次 episode 随机化
策略看到各种情况
```

**3. 不过度随机化**：
```
太大会导致学习困难
平衡多样性和可学习性
```

**4. 逐步增加**：
```
初期：小范围随机化
后期：大范围随机化
```

**实现要点**：
```python
# 每 episode 随机化
for episode in range(n_episodes):
    env.randomize_parameters()  # 随机化
    train_episode(env)
```

**成功案例**：
- OpenAI Rubik's Cube
- 无人机飞行
- 机械臂抓取

**评分要点：** 说出随机化内容得基本分，解释设计原则得高分

## 题目 3：系统识别
**问题：** 系统识别如何帮助 Sim-to-Real？实现流程是什么？

**参考答案：**
**系统识别（System Identification）**：
从真实数据识别动力学参数，更新仿真模型以缩小 Reality Gap。

**核心思想**：
```
真实数据 → 识别参数 → 更新仿真 → 缩小差距
```

**实现流程**：

**1. 收集真实数据**：
```
执行动作序列
记录 (state, action, next_state)
```

**2. 参数化仿真模型**：
```
可识别参数：
- 摩擦系数 μ
- 质量 m
- 惯性 I
- 阻尼 b
```

**3. 优化目标**：
```
min_θ ||仿真轨迹 (θ) - 真实轨迹||²
```

**4. 优化方法**：
```
梯度下降
贝叶斯优化
进化算法
```

**5. 更新仿真**：
```
用识别的参数更新仿真
重新训练或微调策略
```

**迭代流程**：
```
1. 初始仿真训练
2. 真实部署收集数据
3. 系统识别更新参数
4. 更新仿真重新训练
5. 重复直到满意
```

**挑战**：

**1. 数据需求**：
- 需要足够真实数据
- 收集成本高

**2. 参数可识别性**：
- 某些参数难以区分
- 需要设计激励轨迹

**3. 计算成本**：
- 优化需要多次仿真
- 时间长

**4. 时变参数**：
- 真实参数可能变化
- 需要在线识别

**实践建议**：
- 从关键参数开始
- 设计激励轨迹
- 结合域随机化
- 持续更新

**评分要点：** 说出识别流程得基本分，解释挑战得高分

## 题目 4：渐进式迁移
**问题：** 渐进式迁移的策略是什么？如何设计迁移阶段？

**参考答案：**
**渐进式迁移（Progressive Transfer）**：
从仿真逐步过渡到真实，降低迁移风险。

**核心思想**：
```
仿真 → 简化真实 → 完整真实
逐步增加真实度
```

**阶段设计**：

**阶段 1：纯仿真**：
```
- 域随机化训练
- 高成功率
- 无真实风险
```

**阶段 2：简化真实**：
```
- 受控环境
- 简化任务
- 安全约束
```

**阶段 3：混合环境**：
```
- 部分仿真 + 部分真实
- 逐步增加真实比例
```

**阶段 4：完整真实**：
```
- 真实环境
- 完整任务
- 监控部署
```

**阶段切换条件**：
```
性能阈值：成功率>80%
安全指标：无碰撞
稳定性：收敛
```

**实现方式**：

**1. 环境混合**：
```python
alpha = current_stage / total_stages
env = alpha * real_env + (1-alpha) * sim_env
```

**2. 任务简化**：
```
完整任务 → 子任务 → 原子动作
反向迁移：原子→子任务→完整
```

**3. 约束放松**：
```
初期：严格安全约束
后期：逐步放松
```

**优势**：
- 降低风险
- 早期发现问题
- 建立信心

**实践建议**：
- 定义清晰阶段
- 设置切换标准
- 监控每个阶段
- 准备回退方案

**评分要点：** 说出阶段设计得基本分，解释切换条件得高分

## 题目 5：实际应用
**问题：** Sim-to-Real 在实际应用中有什么最佳实践？如何评估迁移效果？

**参考答案：**
**最佳实践**：

**1. 高保真仿真**：
```
- 精确物理引擎
- 真实传感器模型
- 准确执行器模型
```

**2. 域随机化**：
```
- 广泛随机化
- 覆盖真实范围
- 每 episode 随机化
```

**3. 课程学习**：
```
- 简单任务开始
- 逐步增加难度
- 技能组合
```

**4. 系统识别**：
```
- 收集真实数据
- 更新仿真参数
- 迭代改进
```

**5. 安全监控**：
```
- 紧急停止
- 安全约束
- 逐步部署
```

**评估指标**：

**1. 成功率**：
```
仿真成功率 vs 真实成功率
差距越小越好
```

**2. 性能下降**：
```
(仿真性能 - 真实性能) / 仿真性能
目标：<30%
```

**3. 泛化能力**：
```
未见场景的表现
新物体、新环境
```

**4. 鲁棒性**：
```
扰动下的表现
噪声、延迟、故障
```

**5. 样本效率**：
```
真实世界需要多少数据
越少越好
```

**评估流程**：
```
1. 仿真内评估（多随机种子）
2. 简化真实评估（受控环境）
3. 完整真实评估（实际场景）
4. 长期部署评估（稳定性）
```

**成功案例指标**：
```
OpenAI Rubik's Cube:
- 仿真：100% 成功
- 真实：~90% 成功

无人机飞行:
- 仿真训练 10000 小时
- 真实部署 0 事故
```

**实践建议**：
- 量化评估指标
- 对比基线方法
- 报告失败案例
- 持续改进

**评分要点：** 说出 3 个以上最佳实践得基本分，详细评估指标得高分
