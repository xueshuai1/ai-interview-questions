# 强化学习最佳实践面试题（5 道）

## 题目 1：算法选择
**问题：** 如何为不同问题选择合适的 RL 算法？有什么决策框架？

**参考答案：**
**算法选择决策框架**：

**1. 动作空间类型**：
```
离散动作：
- DQN：简单有效
- PPO：稳定通用
- Rainbow：SOTA

连续动作：
- SAC：样本高效
- TD3：稳定
- PPO：通用
```

**2. 样本效率需求**：
```
样本充足：
- Model-Free (DQN, PPO)

样本有限：
- 模型-Based
- 离线 RL
- 迁移学习
```

**3. 安全要求**：
```
安全关键：
- 安全 RL (CPO, Safe RL)
- 约束 RL

一般应用：
- 标准算法
```

**4. 环境特性**：
```
episodic：
- 大多数算法

continuous：
- Actor-Critic 类

partial observable：
- DRQN, POMDP 方法
```

**选择指南**：

| 场景 | 首选 | 备选 |
|------|------|------|
| Atari | DQN/Rainbow | PPO |
| 机器人控制 | SAC | TD3/PPO |
| 样本有限 | 模型-Based | 离线 RL |
| 安全关键 | CPO | 约束 RL |
| 多智能体 | MAPPO | QMIX |

**实践建议**：
```
1. 从基线算法开始
2. 验证环境
3. 调优超参数
4. 对比多个算法
5. 选择最佳
```

**评分要点：** 说出动作空间分类得基本分，详细决策框架得高分

## 题目 2：超参数调优
**问题：** RL 中有哪些关键超参数？如何系统调优？

**参考答案：**
**关键超参数**：

**1. 学习率**：
```
策略网络：3e-4 到 1e-3
价值网络：1e-3 到 3e-3
太大：不稳定
太小：学习慢
```

**2. 折扣因子γ**：
```
短任务：0.9-0.95
长任务：0.99-0.999
影响长期规划
```

**3. 探索率ε**：
```
初始：1.0
最终：0.01-0.1
衰减：0.995 每步
```

**4. 批次大小**：
```
DQN: 32-64
PPO: 64-256
SAC: 256
大批次：稳定但慢
```

**5. 目标网络更新**：
```
硬更新：每 C 步
软更新：τ=0.005
```

**调优流程**：

**1. 默认值开始**：
```
使用论文/库默认值
验证能工作
```

**2. 单参数扫描**：
```
一次改变一个参数
观察影响
```

**3. 网格/随机搜索**：
```
参数网格
随机采样
```

**4. 贝叶斯优化**：
```
更高效
自动搜索
```

**实践建议**：
```
- 先调学习率
- 再调探索率
- 最后微调
- 记录所有实验
```

**常见陷阱**：
- 同时调多个参数
- 不调随机种子
- 训练不充分
- 过拟合验证集

**评分要点：** 说出 5 个超参数得基本分，详细调优流程得高分

## 题目 3：训练监控
**问题：** RL 训练中应该监控哪些指标？如何设置告警？

**参考答案：**
**核心监控指标**：

**1. 奖励指标**：
```
- 平均奖励
- 最佳奖励
- 奖励标准差
- 移动平均
```

**2. 损失指标**：
```
- 策略损失
- 价值损失
- TD 误差
- 是否下降
```

**3. 探索指标**：
```
- 动作熵
- 唯一动作数
- 状态覆盖率
```

**4. 价值指标**：
```
- Q 值分布
- Q 值均值/方差
- 是否爆炸
```

**5. 梯度指标**：
```
- 梯度范数
- 梯度分布
- 是否消失/爆炸
```

**告警阈值**：

**奖励告警**：
```
- 连续 100 episode 下降>20%
- 奖励<随机策略
- 奖励 NaN/Inf
```

**Q 值告警**：
```
- |Q| > 1000
- Q 值 NaN/Inf
- Q 值方差过大
```

**梯度告警**：
```
- 梯度范数>10
- 梯度范数<1e-6
- 梯度 NaN
```

**探索告警**：
```
- 熵<0.1（探索不足）
- 熵>2.0（探索过度）
```

**监控工具**：
```
- TensorBoard
- Weights & Biases
- 自定义脚本
```

**实践建议**：
```
- 实时可视化
- 自动告警
- 定期人工检查
- 保存所有日志
```

**评分要点：** 说出 5 类指标得基本分，详细告警阈值得高分

## 题目 4：可复现性
**问题：** 如何确保 RL 实验的可复现性？有什么最佳实践？

**参考答案：**
**可复现性最佳实践**：

**1. 随机种子**：
```
设置所有随机源：
- numpy.random.seed()
- torch.manual_seed()
- random.seed()
- 环境种子
```

**2. 版本控制**：
```
- 代码版本（git）
- 依赖版本（requirements.txt）
- 环境版本（Docker）
```

**3. 配置管理**：
```
- 所有超参数记录
- 配置文件保存
- 实验 ID 唯一
```

**4. 实验日志**：
```
- 完整训练日志
- 指标记录
- 检查点保存
```

**5. 多随机种子**：
```
- 至少 3-5 个种子
- 报告平均值±标准差
- 显示方差
```

**实验记录模板**：
```
实验 ID: 20240101_120000
算法：PPO
环境：HalfCheetah-v3
种子：42
超参数:
  - lr: 3e-4
  - gamma: 0.99
  - ...
结果:
  - 平均奖励：3500 ± 200
  - 最佳：3800
训练时间：2 小时
```

**报告要求**：
```
- 实验配置完整
- 性能指标详细
- 曲线图清晰
- 对比公平
```

**常见错误**：
- 忘记设置种子
- 不记录超参数
- 只跑一次
- 选择性报告

**评分要点：** 说出随机种子得基本分，详细最佳实践得高分

## 题目 5：实验报告
**问题：** 一个完整的 RL 实验报告应包含哪些内容？如何呈现结果？

**参考答案：**
**实验报告必要内容**：

**1. 问题描述**：
```
- 任务定义
- 环境说明
- 评估指标
```

**2. 方法描述**：
```
- 算法及变体
- 网络架构
- 超参数
```

**3. 实验设置**：
```
- 硬件配置
- 软件版本
- 随机种子
- 训练细节
```

**4. 结果呈现**：
```
- 奖励曲线
- 对比表格
- 统计分析
- 置信区间
```

**5. 分析讨论**：
```
- 成功/失败分析
- 消融实验
- 局限性
- 未来工作
```

**结果呈现最佳实践**：

**奖励曲线**：
```
- 移动平均平滑
- 置信区间显示
- 多种子平均
- x 轴统一（步数/episode）
```

**对比表格**：
```
| 方法 | 平均 | 标准差 | 最佳 |
|------|------|--------|------|
| Ours | 3500 | 200    | 3800 |
| Baseline | 3000 | 300 | 3400 |
```

**可视化**：
```
- 清晰标签
- 合理颜色
- 统一风格
- 高分辨率
```

**报告结构**：
```
1. 摘要
2. 引言
3. 方法
4. 实验
5. 结果
6. 讨论
7. 结论
8. 附录
```

**评审标准**：
```
- 方法新颖性
- 实验充分性
- 结果可信度
- 可复现性
```

**评分要点：** 说出 5 个必要内容得基本分，详细呈现方法得高分
