# 强化学习调试技巧面试题（5 道）

## 题目 1：常见问题
**问题：** RL 训练中有哪些常见问题？如何识别和解决？

**参考答案：**
**常见问题**：

**1. 不收敛**：
```
症状：奖励不提升
原因：
- 学习率太大
- 探索不足
- 奖励稀疏
解决：
- 降低学习率
- 增加探索
- 奖励塑造
```

**2. 收敛到次优**：
```
症状：奖励停滞
原因：
- 局部最优
- 探索过早停止
解决：
- 增加探索
- 重启训练
- 改变初始条件
```

**3. 训练不稳定**：
```
症状：奖励大幅波动
原因：
- 方差大
- 批次太小
- 学习率太高
解决：
- 增大批次
- 降低学习率
- 梯度裁剪
```

**4. Q 值爆炸**：
```
症状：Q 值趋向无穷
原因：
- 目标网络更新慢
- 奖励缩放问题
解决：
- 奖励缩放
- 更快更新目标
- 梯度裁剪
```

**5. 策略崩溃**：
```
症状：性能突然下降
原因：
- 灾难性遗忘
- 分布偏移
解决：
- 经验回放
- 正则化
- 约束更新
```

**识别方法**：
```
1. 监控奖励曲线
2. 检查损失曲线
3. 查看 Q 值分布
4. 监控梯度范数
5. 检查探索率
```

**调试流程**：
```
1. 识别症状
2. 查看相关指标
3. 假设原因
4. 针对性修复
5. 验证修复
```

**评分要点：** 说出 3 个以上问题得基本分，详细解决得高分

## 题目 2：过拟合测试
**问题：** 什么是过拟合测试？为什么它对 RL 调试很重要？

**参考答案：**
**过拟合测试**：
在极小环境中训练，验证算法能否快速学会最优策略。

**测试方法**：
```
1. 创建小环境
   - 状态空间小（如 4-10 个状态）
   - 动作空间小（如 2-4 个动作）
   - 奖励稠密

2. 训练算法
   - 应该快速收敛
   - 达到已知最优

3. 验证
   - 性能接近最优
   - 策略正确
```

**为什么重要**：

**1. 验证实现正确性**：
```
小问题应该能解决
如果不行→实现有 bug
```

**2. 快速迭代**：
```
训练快
快速验证修改
```

**3. 隔离问题**：
```
排除环境问题
聚焦算法问题
```

**4. 建立信心**：
```
小问题能解决
大问题才可能解决
```

**测试环境示例**：
```
GridWorld:
- 4x4 网格
- 起点、终点
- 无障碍

应该：
- <100 episode 收敛
- 找到最短路径
```

**预期结果**：
```
正确实现：
- 快速收敛
- 达到最优

有 bug：
- 不收敛
- 次优策略
- 震荡
```

**实践建议**：
- 先过小拟合测试
- 再上复杂环境
- 保留测试脚本

**评分要点：** 说出小环境测试得基本分，解释重要性得高分

## 题目 3：梯度检查
**问题：** 如何进行梯度检查？为什么它对 RL 重要？

**参考答案：**
**梯度检查**：
比较数值梯度和解析梯度，验证反向传播正确性。

**方法**：
```
数值梯度：
∂L/∂θ ≈ (L(θ+ε) - L(θ-ε)) / (2ε)

解析梯度：
∂L/∂θ（反向传播计算）

比较：
相对误差 = ||数值 - 解析|| / ||数值 + 解析||
< 1e-5：通过
> 1e-3：有问题
```

**实现代码**：
```python
def gradient_check(network, input, epsilon=1e-5):
    # 解析梯度
    input.requires_grad_(True)
    output = network(input)
    loss = output.sum()
    loss.backward()
    analytic_grad = input.grad.clone()
    
    # 数值梯度
    numerical_grad = torch.zeros_like(input)
    for i in range(len(input)):
        input_plus = input.clone()
        input_plus[i] += epsilon
        loss_plus = network(input_plus).sum()
        
        input_minus = input.clone()
        input_minus[i] -= epsilon
        loss_minus = network(input_minus).sum()
        
        numerical_grad[i] = (loss_plus - loss_minus) / (2*epsilon)
    
    # 比较
    diff = torch.norm(analytic_grad - numerical_grad)
    norm = torch.norm(analytic_grad + numerical_grad)
    return (diff / (norm + 1e-8)).item()
```

**为什么对 RL 重要**：

**1. 复杂计算图**：
```
RL 有多个网络
策略、价值、目标
容易出错
```

**2. 自定义损失**：
```
策略梯度损失
TD 损失
可能实现错误
```

**3. 调试困难**：
```
RL 反馈延迟
梯度错误难发现
早期检查重要
```

**4. 常见错误**：
```
- 忘记 detach
- 梯度累积错误
- 损失计算错误
```

**实践建议**：
- 训练前检查
- 修改后检查
- 自定义组件检查

**评分要点：** 说出数值 vs 解析得基本分，解释 RL 重要性得高分

## 题目 4：监控指标
**问题：** RL 训练中应该监控哪些指标？如何解读？

**参考答案：**
**核心指标**：

**1. 奖励曲线**：
```
监控：平均奖励、最佳奖励
解读：
- 上升：学习正常
- 平稳：可能收敛
- 下降：有问题
```

**2. 损失曲线**：
```
监控：策略损失、价值损失
解读：
- 下降：学习正常
- 震荡：可能不稳定
- 上升：有问题
```

**诊断指标**：

**3. 探索指标**：
```
监控：动作熵、唯一动作数
解读：
- 熵太低：探索不足
- 熵太高：探索过度
- 适中：正常
```

**4. 价值指标**：
```
监控：Q 值分布、TD 误差
解读：
- Q 值爆炸：学习率大
- TD 误差大：学习初期
- TD 误差小：收敛
```

**训练指标**：

**5. 梯度指标**：
```
监控：梯度范数、梯度分布
解读：
- 梯度消失：网络太深
- 梯度爆炸：学习率大
- 适中：正常
```

**6. 参数指标**：
```
监控：参数范数、更新幅度
解读：
- 参数爆炸：不稳定
- 更新太小：学习慢
```

**告警阈值**：
```
奖励下降>20%→告警
Q 值>1000→告警
梯度>10→告警
熵<0.1→告警
```

**可视化工具**：
- TensorBoard
- Weights & Biases
- 自定义绘图

**评分要点：** 说出 5 个以上指标得基本分，详细解读得高分

## 题目 5：调试流程
**问题：** 描述一个系统的 RL 调试流程。有什么最佳实践？

**参考答案：**
**系统调试流程**：

**阶段 1：实现验证**
```
1. 梯度检查
2. 过拟合测试（小环境）
3. 单元测试组件
```

**阶段 2：初步训练**
```
1. 小规模训练
2. 监控核心指标
3. 检查是否收敛
```

**阶段 3：问题诊断**
```
1. 识别症状
2. 查看相关指标
3. 假设原因
```

**阶段 4：针对性修复**
```
1. 修改假设原因
2. 重新训练
3. 验证修复
```

**阶段 5：大规模训练**
```
1. 完整环境
2. 长时间训练
3. 最终评估
```

**最佳实践**：

**1. 从简单开始**：
```
小环境→大环境
简单任务→复杂任务
```

**2. 基线对比**：
```
与已知结果对比
与开源实现对比
```

**3. 超参数搜索**：
```
学习率网格搜索
其他参数调优
```

**4. 多次随机种子**：
```
至少 3-5 个种子
报告平均值和方差
```

**5. 详细日志**：
```
记录所有实验
便于复现和回溯
```

**6. 可视化分析**：
```
奖励曲线
Q 值分布
策略可视化
```

**调试清单**：
```
[ ] 梯度检查通过
[ ] 过拟合测试通过
[ ] 奖励曲线正常
[ ] 损失曲线正常
[ ] Q 值正常
[ ] 梯度正常
[ ] 探索正常
[ ] 多次种子一致
```

**常见陷阱**：
- 过早下结论
- 只试一次
- 不看指标
- 不调超参

**评分要点：** 说出系统流程得基本分，详细最佳实践得高分
