# 分层强化学习详解面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是分层强化学习？它与扁平 RL 相比有什么优势？

**参考答案：**
**分层强化学习（HRL）**：
通过将任务分解为层次结构来实现时间抽象和技能复用的强化学习方法。

**核心思想**：
- 高层策略：长期规划，选择子目标
- 低层策略：短期执行，选择动作
- 时间抽象：高层每 H 步决策一次

**与扁平 RL 对比**：

| 方面 | 扁平 RL | 分层 RL |
|------|---------|---------|
| **时间抽象** | 无（每步决策） | 有（高层稀疏决策） |
| **长期规划** | 困难（信用分配） | 容易（高层负责） |
| **技能复用** | 困难 | 容易（低层技能） |
| **样本效率** | 低 | 高 |
| **可解释** | 低 | 高（层次清晰） |
| **长时域** | 困难 | 适合 |

**优势详解**：

**1. 时间抽象**：
```
扁平 RL：每步选择动作
分层 RL：高层每 10 步选子目标，低层每步选动作
→ 有效视野扩大 10 倍
```

**2. 技能复用**：
- 低层学习通用技能（如"走到某位置"）
- 高层组合技能完成复杂任务
- 新任务只需学习高层策略

**3. 信用分配**：
- 高层：长期奖励
- 低层：短期奖励（到达子目标）
- 减少信用分配难度

**4. 探索效率**：
- 高层探索子目标空间
- 低层探索动作空间
- 结构化探索更高效

**示例**：
```
导航任务：
- 扁平 RL：学习每步转向
- 分层 RL：
  - 高层：选择"去厨房"、"去卧室"
  - 低层：学习走到指定位置
```

**评分要点：** 说出时间抽象得基本分，详细对比优势得高分

## 题目 2：Options 框架
**问题：** 什么是 Options 框架？它如何形式化分层 RL？

**参考答案：**
**Options 框架**：
Sutton 等人提出的分层 RL 形式化框架，将技能定义为 temporally extended actions。

**Option 定义**：
```
Option = (I, π, β)
```

**三个组件**：

**1. Initiation set (I)**：
- 可以启动该 option 的状态集合
- I ⊆ S
- 如："开门"option 只能在门附近启动

**2. Internal policy (π)**：
- option 内部的策略
- π: S × A → [0,1]
- 执行到终止

**3. Termination condition (β)**：
- 终止概率函数
- β: S → [0,1]
- β(s) = P(option 在 s 终止)

**执行流程**：
```
在状态 s：
1. 如果当前无 option 或 option 终止：
   选择新 option o（从可用 options 中）
2. 执行 option o 的内部策略π_o
3. 以概率β_o(s) 终止
4. 重复
```

**与扁平 RL 对比**：
```
扁平 RL：π: S → A
Options：π: S → {options}
         每个 option: S → A（内部）
```

**优势**：
1. **时间抽象**：option 持续多步
2. **复用**：option 可在不同任务中复用
3. **组合**：options 可组合完成复杂任务
4. **理论**：有收敛性保证

**学习 Options**：
- 手工设计
- 自动发现（聚类、瓶颈状态）
- 端到端学习（如 HIRO）

**评分要点：** 写出三元组定义得基本分，解释执行流程得高分

## 题目 3：HIRO 算法
**问题：** HIRO 算法的核心思想是什么？它如何解决分层 RL 的挑战？

**参考答案：**
**HIRO（Hierarchical Reinforcement Learning with Off-policy correction）**：
端到端学习分层策略的算法，通过离线校正解决非平稳性问题。

**核心思想**：

**1. 两层架构**：
```
高层：选择子目标 g
低层：执行动作到达 g
```

**2. 低层目标**：
```
max E[Σ γ^t r_low(s_t, g)]
r_low = -||s_t - g||²（到达子目标）
```

**3. 高层目标**：
```
max E[Σ γ^{kH} r_high(s_{kH})]
```

**解决的挑战**：

**1. 非平稳性**：
- 问题：低层策略变化→高层目标变化
- HIRO 解决：离线校正子目标
```
g_corrected = g + (s_actual - s_predicted)
```

**2. 端到端学习**：
- 问题：两层联合训练困难
- HIRO 解决：交替更新，离线校正

**3. 样本效率**：
- 问题：分层需要更多数据
- HIRO 解决：off-policy 学习，经验回放

**算法流程**：
```
1. 低层更新：
   - 目标：到达高层设定的子目标
   - 损失：||s - g||²

2. 高层更新：
   - 目标：最大化环境奖励
   - 校正：g_corrected = g + 低层误差

3. 交替迭代
```

**优势**：
- 端到端可微
- 无需手工设计
- 样本效率较高

**评分要点：** 说出两层架构得基本分，解释离线校正得高分

## 题目 4：代码实现
**问题：** 请实现分层 RL 的高层和低层策略前向传播。

**参考答案：**
```python
import torch
import torch.nn as nn

class HighLevelPolicy(nn.Module):
    """高层策略：选择子目标"""
    
    def __init__(self, state_dim, goal_dim, hidden_dim=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, goal_dim)
        )
    
    def forward(self, state):
        """
        输入：state (batch, state_dim)
        输出：goal (batch, goal_dim)
        """
        return self.net(state)

class LowLevelPolicy(nn.Module):
    """低层策略：执行动作"""
    
    def __init__(self, state_dim, goal_dim, action_dim, hidden_dim=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(state_dim + goal_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim)
        )
    
    def forward(self, state, goal):
        """
        输入：
            state (batch, state_dim)
            goal (batch, goal_dim)
        输出：action (batch, action_dim)
        """
        x = torch.cat([state, goal], dim=1)
        return self.net(x)

class HierarchicalAgent:
    """分层智能体"""
    
    def __init__(self, state_dim, action_dim, goal_dim):
        self.high_policy = HighLevelPolicy(state_dim, goal_dim)
        self.low_policy = LowLevelPolicy(state_dim, goal_dim, action_dim)
        
        self.high_horizon = 10  # 高层每 10 步决策一次
    
    def act(self, state, goal=None):
        """
        执行一步
        
        如果 goal 为 None 或到达 horizon，调用高层
        否则调用低层
        """
        state = torch.FloatTensor(state).unsqueeze(0)
        
        if goal is None:
            # 高层选择子目标
            with torch.no_grad():
                goal = self.high_policy(state)
            return goal.cpu().numpy()[0], None
        
        else:
            # 低层选择动作
            goal = torch.FloatTensor(goal).unsqueeze(0)
            with torch.no_grad():
                action = self.low_policy(state, goal)
            return None, action.cpu().numpy()[0]
    
    def select_high_action(self, state):
        """高层选择子目标"""
        state = torch.FloatTensor(state).unsqueeze(0)
        with torch.no_grad():
            goal = self.high_policy(state)
        return goal.cpu().numpy()[0]
    
    def select_low_action(self, state, goal):
        """低层选择动作"""
        state = torch.FloatTensor(state).unsqueeze(0)
        goal = torch.FloatTensor(goal).unsqueeze(0)
        with torch.no_grad():
            action = self.low_policy(state, goal)
        return action.cpu().numpy()[0]

# 使用示例
if __name__ == "__main__":
    agent = HierarchicalAgent(state_dim=10, action_dim=4, goal_dim=3)
    
    # 高层决策（每 10 步）
    state = np.random.randn(10)
    goal, _ = agent.act(state)  # 返回子目标
    
    # 低层执行（每步）
    for _ in range(10):
        _, action = agent.act(state, goal)  # 返回动作
        # 执行 action，观察新状态
```

**关键点**：
1. 高层输出子目标
2. 低层输入状态 + 子目标
3. 高层稀疏调用（每 H 步）
4. 低层每步调用

**评分要点：** 代码正确得基本分，解释层次调用得高分

## 题目 5：应用场景
**问题：** 分层 RL 适合哪些应用场景？设计层次结构时需要考虑什么？

**参考答案：**
**适合的应用场景**：

**1. 长时域任务**：
- 导航到远处目标
- 多房间探索
- 需要长期规划

**2. 复杂操作**：
- 机器人抓取 + 放置
- 工具使用
- 多步骤任务

**3. 技能组合**：
- 学习基础技能库
- 组合完成新任务
- 如：走 + 抓取 + 放置

**4. 多任务学习**：
- 共享低层技能
- 高层任务特定
- 快速适应新任务

**5. 可解释需求**：
- 需要理解决策
- 调试策略
- 安全关键应用

**设计层次结构的考虑**：

**1. 时间尺度**：
```
高层频率：每 H 步
- H 太大：低层任务太难
- H 太小：失去抽象优势
- 经验值：H=10-100
```

**2. 子目标空间**：
```
子目标表示：
- 状态空间子集（如目标位置）
- 抽象表示（如"去厨房"）
- 技能索引
```

**3. 层次数量**：
```
两层：最常用
三层：更复杂任务
更多：难以训练
```

**4. 信息流**：
```
高层→低层：子目标
低层→高层：成功/失败信号
是否需要双向通信？
```

**5. 训练方式**：
```
- 端到端：联合训练
- 分阶段：先低层后高层
- 交替：轮流更新
```

**设计原则**：

**1. 任务分解**：
- 自然分解点
- 子任务独立
- 清晰接口

**2. 复用性**：
- 低层技能通用
- 高层任务特定
- 便于迁移

**3. 可学习性**：
- 每层难度适中
- 奖励信号清晰
- 避免稀疏奖励

**4. 可扩展**：
- 新增技能容易
- 层次可调整
- 支持迁移

**实践建议**：
- 从两层开始
- 子目标用状态表示
- 低层用到达奖励
- 高层用环境奖励

**评分要点：** 说出 3 个以上场景得基本分，详细设计考虑得高分
