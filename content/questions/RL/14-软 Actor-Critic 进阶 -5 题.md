# 软 Actor-Critic 进阶面试题（5 道）

## 题目 1：最大熵
**问题：** 什么是最大熵强化学习？熵正则化项的作用是什么？

**参考答案：**
**最大熵 RL**：
在标准 RL 目标基础上增加策略熵作为正则化项：
```
max_π E[Σ γ^t (r_t + α H(π(·|s_t)))]
```

其中熵：
```
H(π(·|s)) = E_a∼π[-log π(a|s)]
```

**熵正则化的作用**：

**1. 鼓励探索**：
- 高熵=随机策略
- 自动探索状态空间
- 无需额外探索策略

**2. 防止早熟收敛**：
- 避免过早收敛到次优确定性策略
- 保持策略多样性
- 更可能找到全局最优

**3. 鲁棒性**：
- 对扰动更鲁棒
- 策略不会过于"尖锐"
- 适应环境变化

**4. 多模态策略**：
- 可以学习多模态分布
- 多个最优动作都有概率
- 更灵活的行为

**温度参数α的作用**：
- 权衡回报和熵
- α大：重视探索
- α小：重视利用
- 自动调节最佳

**与ε-贪婪对比**：
- ε-贪婪：均匀探索
- 最大熵：智能探索（高价值区域高熵）

**评分要点：** 说出鼓励探索得基本分，解释多模态得高分

## 题目 2：软 Q 函数
**问题：** 什么是软 Q 函数？它与标准 Q 函数有什么区别？

**参考答案：**
**软 Q 函数**：
在最大熵框架下的动作价值函数：
```
Q^π_soft(s,a) = E[Σ γ^t (r_t + α H(π(·|s_t))) | s_0=s, a_0=a]
```

**软 Bellman 方程**：
```
Q^π(s,a) = r + γ E_s'[V^π(s')]
```

**软价值函数**：
```
V^π(s) = E_a∼π[Q^π(s,a) - α log π(a|s)]
```

**与标准 Q 函数的区别**：

| 方面 | 标准 Q | 软 Q |
|------|--------|------|
| **目标** | 最大化回报 | 回报 + 熵 |
| **Bellman** | V(s)=max_a Q(s,a) | V(s)=E[Q-αlogπ] |
| **最优策略** | 确定性 | 随机（玻尔兹曼） |
| **更新** | Q ← r+γmax Q' | Q ← r+γE[Q'-αlogπ'] |

**最优软 Q 函数**：
```
Q*(s,a) = r + γ E_s'[log Σ_a' exp(Q*(s',a')/α)]
```

这类似于 softmax 操作。

**策略形式**：
```
π*(a|s) ∝ exp(Q*(s,a)/α)
```
玻尔兹曼分布。

**优势**：
- 自动平衡探索利用
- 更平滑的价值函数
- 对扰动鲁棒

**评分要点：** 写出软 Bellman 方程得基本分，解释玻尔兹曼分布得高分

## 题目 3：重参数化技巧
**问题：** SAC 中为什么用重参数化技巧？如何实现？

**参考答案：**
**为什么用重参数化技巧**：

**问题**：策略采样不可导
```
a ∼ π_θ(·|s)
```
- 采样操作阻断梯度
- 无法直接用梯度下降优化θ

**解决方案**：重参数化
```
a = f_θ(ε; s),  ε ∼ p(ε)
```
- 噪声ε与参数θ分离
- f_θ可导
- 梯度可反向传播

**SAC 中的实现**：

**高斯策略**：
```
a = tanh(μ_θ(s) + σ_θ(s) ⊙ ε)
ε ∼ N(0, I)
```

**步骤**：
1. 策略输出μ和σ
2. 采样ε∼N(0,I)
3. 计算 a = tanh(μ + σ⊙ε)
4. 梯度可通过μ,σ反向传播

**对数概率修正**：
```
log π(a|s) = log N(z; μ, σ) - Σ log(1 - tanh²(z_i))
```
其中 z = μ + σ⊙ε。

**优势**：
- 低方差梯度估计
- 可直接用 SGD 优化
- 适合连续动作

**对比 REINFORCE**：
- REINFORCE：高方差
- 重参数化：低方差

**评分要点：** 说出采样不可导得基本分，写出实现得高分

## 题目 4：自动温度调节
**问题：** SAC 如何自动调节温度参数α？为什么需要自动调节？

**参考答案：**
**为什么需要自动调节**：

**问题**：手动调α困难
- α太大：过度探索，学习慢
- α太小：探索不足，次优
- 不同任务最佳α不同
- 同一任务不同阶段需要不同α

**自动调节原理**：

**目标熵**：
```
H_target = -dim(action)
```
启发式：动作空间维度的负值。

**优化问题**：
```
min_α E_a∼π[-α log π(a|s) - α H_target]
```

**含义**：
- 如果 H(π) < H_target：增加α（鼓励探索）
- 如果 H(π) > H_target：减少α（鼓励利用）
- 平衡到目标熵

**梯度更新**：
```
∇_α loss = -log π(a|s) - H_target
α ← α - λ ∇_α loss
```

**实现**：
```python
# 对数参数化（保证α>0）
log_alpha = torch.zeros(1, requires_grad=True)
alpha = log_alpha.exp()

# 损失
alpha_loss = -(alpha * (log_prob + target_entropy).detach()).mean()

# 更新
alpha_optimizer.zero_grad()
alpha_loss.backward()
alpha_optimizer.step()
```

**优势**：
- 无需手动调α
- 自适应不同任务
- 训练过程自动调整
- 更鲁棒

**实践效果**：
- 通常比固定α好
- 减少调参工作
- 成为 SAC 标准配置

**评分要点：** 说出手动调参困难得基本分，写出更新公式得高分

## 题目 5：SAC vs TD3
**问题：** 比较 SAC 和 TD3 两种连续控制算法的异同。

**参考答案：**
**相同点**：
- 都是 off-policy 算法
- 都用于连续控制
- 都用 Actor-Critic 架构
- 都有高样本效率
- 都是当前 SOTA

**不同点**：

| 方面 | SAC | TD3 |
|------|-----|-----|
| **目标** | 最大熵 | 标准 RL |
| **策略** | 随机（高斯） | 确定性 + 噪声 |
| **Q 网络** | 双 Q，取最小 | 双 Q，取最小 |
| **探索** | 熵正则化 | 动作噪声 |
| **温度** | 自动调节 | 无 |
| **延迟更新** | 无 | 有（策略延迟） |
| **目标平滑** | 无 | 有（目标平滑） |

**SAC 优势**：
1. **自动探索**：熵鼓励探索
2. **多模态**：可学习多模态策略
3. **鲁棒性**：对扰动更鲁棒
4. **自动调参**：温度自调节

**TD3 优势**：
1. **简单**：无熵项
2. **稳定**：延迟更新 + 目标平滑
3. **确定性**：某些任务更好
4. **计算略低**：无熵计算

**TD3 关键技术**：
- **延迟策略更新**：Critic 更新 2 次，Actor 更新 1 次
- **目标平滑**：目标动作加噪声
- **双 Q 截断**：减少过估计

**选择建议**：
- 默认：SAC（自动调参）
- 需要确定性：TD3
- 多模态任务：SAC
- 资源有限：TD3（略简单）

**实践**：
- 两者性能接近
- SAC 更易用
- TD3 更稳定（某些任务）
- 都优于 DDPG

**评分要点：** 说出熵 vs 确定性得基本分，详细对比得高分
