# 迁移学习详解面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是迁移学习？它与多任务学习有什么区别？

**参考答案：**
**迁移学习（Transfer Learning）**：
将在源任务中学到的知识迁移到目标任务，减少目标任务的学习成本。

**核心思想**：
- 源任务：已学习的任务
- 目标任务：新任务
- 迁移：知识从源到目标

**与多任务学习的区别**：

| 方面 | 迁移学习 | 多任务学习 |
|------|----------|------------|
| **任务关系** | 顺序（源→目标） | 并行（同时学习） |
| **目标** | 提升目标任务 | 提升所有任务 |
| **知识流** | 单向 | 双向/多向 |
| **训练** | 先源后目标 | 同时训练 |
| **应用** | 冷启动、少数据 | 相关任务共享 |

**直观理解**：
- 迁移学习：学会骑自行车→学摩托车（顺序）
- 多任务学习：同时学骑车、摩托、开车（并行）

**迁移学习内容**：
1. **实例**：源任务数据
2. **特征**：共享表示
3. **参数**：模型权重
4. **知识**：规则、技能

**迁移学习场景**：
```
1. 源任务数据多，目标任务数据少
2. 目标任务冷启动
3. 源任务与目标任务相关
4. 需要快速适应新任务
```

**评分要点：** 说出知识迁移得基本分，对比多任务学习得高分

## 题目 2：迁移类型
**问题：** 迁移学习有哪些类型？什么是负向迁移？如何避免？

**参考答案：**
**迁移类型**：

**1. 同构迁移（Homogeneous Transfer）**：
```
源任务：S, A
目标任务：S, A（相同空间）
```
- 直接迁移参数
- 最简单

**2. 异构迁移（Heterogeneous Transfer）**：
```
源任务：S₁, A₁
目标任务：S₂, A₂（不同空间）
```
- 需要映射或抽象
- 更复杂

**3. 正向迁移（Positive Transfer）**：
- 源任务帮助目标任务
- 性能提升
- 理想情况

**4. 负向迁移（Negative Transfer）**：
- 源任务干扰目标任务
- 性能下降
- 需要避免

**负向迁移原因**：

**1. 任务不相关**：
```
源任务：下棋
目标任务：开车
→ 知识不相关，可能干扰
```

**2. 冲突知识**：
```
源任务：靠右行驶
目标任务：靠左行驶
→ 规则冲突
```

**3. 过度拟合源任务**：
```
源任务特定特征
→ 不适应目标任务
```

**避免负向迁移的方法**：

**1. 任务相似性评估**：
```
在迁移前评估任务相似度
相似度低→不迁移或少迁移
```

**2. 选择性迁移**：
```
只迁移相关部分
- 部分层迁移
- 注意力机制选择
```

**3. 渐进式迁移**：
```
逐步增加迁移量
监控性能变化
发现负迁移及时停止
```

**4. 多源融合**：
```
多个源任务
加权组合
降低单源影响
```

**5. 元学习**：
```
学习何时迁移
学习迁移多少
自适应决策
```

**检测负向迁移**：
```python
# 基线：不迁移，从头学习
baseline_perf = train_from_scratch()

# 迁移学习
transfer_perf = train_with_transfer()

if transfer_perf < baseline_perf:
    # 负向迁移
    reduce_transfer_amount()
```

**评分要点：** 说出负向迁移得基本分，提出多种避免方法得高分

## 题目 3：参数迁移
**问题：** 参数迁移如何实现？微调时需要注意什么？

**参考答案：**
**参数迁移实现**：

**1. 复制参数**：
```python
# 源模型参数
source_dict = source_model.state_dict()

# 目标模型参数
target_dict = target_model.state_dict()

# 复制（形状匹配的层）
for key in source_dict:
    if key in target_dict and source_dict[key].shape == target_dict[key].shape:
        target_dict[key] = source_dict[key]

target_model.load_state_dict(target_dict)
```

**2. 部分迁移**：
```python
# 只迁移底层（特征提取）
layers_to_transfer = ['conv1', 'conv2', 'conv3']

for key in layers_to_transfer:
    if key in source_dict and key in target_dict:
        target_dict[key] = source_dict[key]
```

**微调注意事项**：

**1. 学习率**：
```
微调学习率 < 从头学习学习率
通常 1e-4 到 1e-5
```
- 太大：破坏预训练知识
- 太小：学习太慢

**2. 层冻结**：
```
冻结底层（通用特征）
微调高层（任务特定）
```
```python
# 冻结底层
for name, param in model.named_parameters():
    if 'conv' in name:  # 卷积层
        param.requires_grad = False
```

**3. 训练数据**：
```
目标任务数据量少→更多 epoch
目标任务数据量大→正常训练
```

**4. 早停**：
```
监控验证集性能
性能下降时停止
避免过拟合
```

**5. 正则化**：
```
L2 正则化
Dropout
防止过拟合小数据集
```

**微调策略**：

| 数据量 | 策略 |
|--------|------|
| 很少 | 冻结大部分，只微调顶层 |
| 中等 | 冻结底层，微调高层 |
| 充足 | 全部微调 |

**实践建议**：
- 从预训练模型开始
- 小学习率微调
- 监控验证性能
- 准备回退方案

**评分要点：** 写出参数复制得基本分，详细说明微调注意得高分

## 题目 4：Sim-to-Real
**问题：** Sim-to-Real 迁移的挑战是什么？有哪些解决方法？

**参考答案：**
**Sim-to-Real 挑战**：

**1. 域差异（Domain Gap）**：
```
仿真：理想传感器、完美物理
真实：噪声、延迟、模型误差
```

**2. 外观差异**：
```
仿真：简单纹理、统一光照
真实：复杂纹理、变化光照
```

**3. 动力学差异**：
```
仿真：精确物理引擎
真实：摩擦、弹性、空气阻力
```

**4. 传感器差异**：
```
仿真：完美观测
真实：噪声、延迟、校准误差
```

**解决方法**：

**1. 域随机化（Domain Randomization）**：
```
仿真中随机化：
- 纹理、颜色
- 光照条件
- 物体位置
- 物理参数（摩擦、质量）
```
- 训练鲁棒策略
- 适应真实变化

**2. 域适应（Domain Adaptation）**：
```
学习域不变特征
- 对抗训练
- 特征对齐
- MMD/CORAL损失
```

**3. 系统识别**：
```
从真实数据识别动力学参数
更新仿真模型
缩小差距
```

**4. 渐进式迁移**：
```
仿真 → 简化真实 → 完整真实
逐步增加真实度
```

**5. 数据增强**：
```
仿真数据增强：
- 添加噪声
- 改变外观
- 模拟传感器误差
```

**6. 元学习**：
```
学习快速适应
少量真实数据微调
```

**实践流程**：
```
1. 在仿真中训练
2. 域随机化增加多样性
3. 收集少量真实数据
4. 域适应或微调
5. 真实部署
6. 持续学习
```

**成功案例**：
- OpenAI Rubik's Cube
- 无人机飞行
- 机械臂抓取

**评分要点：** 说出域差异得基本分，提出多种解决得高分

## 题目 5：应用场景
**问题：** 迁移学习适合哪些应用场景？如何选择合适的迁移方法？

**参考答案：**
**适合的应用场景**：

**1. 数据稀缺场景**：
- 医疗诊断（患者数据少）
- 工业检测（缺陷样本少）
- 罕见事件预测

**2. 冷启动问题**：
- 新用户推荐
- 新产品上线
- 新环境部署

**3. Sim-to-Real**：
- 机器人部署
- 自动驾驶
- 无人机控制

**4. 多任务场景**：
- 多语言处理
- 多游戏 AI
- 多机器人协作

**5. 持续学习**：
- 顺序学习任务
- 知识积累
- 避免遗忘

**选择迁移方法的决策框架**：

**1. 评估任务关系**：
```
任务是否相关？
├─ 高度相关 → 参数迁移
├─ 部分相关 → 特征迁移
└─ 低相关 → 谨慎迁移或不迁移
```

**2. 评估数据情况**：
```
目标任务数据量？
├─ 很少 → 冻结 + 微调
├─ 中等 → 部分微调
└─ 充足 → 全部微调
```

**3. 评估空间差异**：
```
状态 - 动作空间是否相同？
├─ 相同 → 同构迁移
└─ 不同 → 异构迁移（需要映射）
```

**4. 评估计算资源**：
```
资源是否充足？
├─ 充足 → 复杂方法（域适应）
└─ 有限 → 简单方法（参数复制）
```

**方法选择指南**：

| 场景 | 推荐方法 |
|------|----------|
| 相同任务不同域 | 域适应 |
| 相似任务 | 参数迁移 + 微调 |
| 不同任务但相关 | 特征迁移 |
| 技能可复用 | 技能迁移 |
| 多源可用 | 多源融合 |

**实践建议**：

**1. 从简单开始**：
- 先试参数复制
- 再试复杂方法

**2. 监控负迁移**：
- 对比基线（从头学习）
- 发现负迁移及时调整

**3. 迭代改进**：
- 分析失败案例
- 调整迁移策略

**4. 记录经验**：
- 哪些任务可迁移
- 哪些方法有效
- 建立知识库

**评分要点：** 说出 3 个以上场景得基本分，给出决策框架得高分
