# 多智能体强化学习面试题（5 道）

## 题目 1：核心挑战
**问题：** 多智能体 RL 与单智能体 RL 相比有什么核心挑战？

**参考答案：**
**核心挑战**：

**1. 环境非平稳性**：
- 单智能体：环境平稳（MDP 假设）
- 多智能体：其他智能体也在学习
- 转移概率 P(s'|s,a) 随时间变化
- 导致收敛困难

**2. 信用分配问题**：
```
团队奖励 R_team
→ 如何分配给每个智能体？
```
- 全局奖励无法区分个体贡献
- 搭便车问题（free-rider）
- 需要分解或分配机制

**3. 可扩展性**：
- 联合动作空间：|A|^N
- 指数级增长
- 需要因子化或参数共享

**4. 协调与通信**：
- 智能体需要协调行动
- 显式或隐式通信
- 学习通信协议

**5. 均衡收敛**：
- 竞争场景：纳什均衡
- 合作场景：协调均衡
- 可能收敛到次优均衡

**6. 部分可观测**：
- 每个智能体局部观测
- 需要信念或通信
- Dec-POMDP 框架

**解决方案方向**：

| 挑战 | 解决方法 |
|------|----------|
| 非平稳性 | CTDE 框架 |
| 信用分配 | VDN, QMIX |
| 可扩展性 | 参数共享 |
| 协调 | 通信学习 |
| 均衡 | 对手建模 |

**评分要点：** 说出 3 个以上挑战得基本分，提出解决方向得高分

## 题目 2：CTDE 框架
**问题：** 什么是 CTDE 框架？为什么它在 MARL 中如此重要？

**参考答案：**
**CTDE（Centralized Training with Decentralized Execution）**：
集中训练分散执行框架，是 MARL 的主流范式。

**核心思想**：
- **训练时**：使用全局信息（状态、其他智能体动作）
- **执行时**：只用局部观测

**为什么重要**：

**1. 解决非平稳性**：
- 训练时 Critic 有全局信息
- 环境对 Critic 是平稳的
- 稳定学习

**2. 保持分散执行**：
- 执行时无需全局通信
- 可扩展到多智能体
- 实际部署可行

**3. 信用分配**：
- Critic 看到全局奖励
- 学习个体贡献
- 如 QMIX 的单调混合

**4. 协调学习**：
- 训练时学习协调策略
- 执行时隐式协调
- 无需显式通信

**代表算法**：

| 算法 | Critic | Actor |
|------|--------|-------|
| **VDN** | Σ Q_i | 独立 |
| **QMIX** | 单调混合 | 独立 |
| **MAPPO** | 全局 V(s) | 独立π_i |
| **MADDPG** | 全局 Q | 独立μ_i |

**实现要点**：
```python
# 训练时
critic_input = (global_state, all_actions)
actor_input = (local_obs_i)

# 执行时
actor_input = (local_obs_i)  # 只需局部
```

**优势**：
- 稳定训练
- 分散执行
- 可扩展

**局限**：
- 需要全局信息训练
- 通信假设
- 某些场景不适用

**评分要点：** 说出集中训练分散执行得基本分，解释重要性得高分

## 题目 3：QMIX 算法
**问题：** QMIX 的核心思想是什么？它如何保证个体 greedy 与全局一致？

**参考答案：**
**QMIX 核心思想**：
用单调混合网络将个体 Q 值混合为全局 Q 值，保证个体最优选择与全局最优一致。

**混合函数**：
```
Q_tot = f(Q_1, ..., Q_N; s)
```

**单调性约束**：
```
∂Q_tot / ∂Q_i ≥ 0  ∀i
```

**为什么保证一致**：

**数学证明**：
```
argmax_{a_1} Q_tot = argmax_{a_1} f(Q_1, ..., Q_N)

由于∂Q_tot/∂Q_1 ≥ 0：
Q_1 最大 → Q_tot 最大

所以：
argmax_{a_1} Q_1 = argmax_{a_1} Q_tot
```

**实现方式**：

**超网络**：
```
w_1 = |HyperNet_1(s)|  # 绝对值保证非负
w_2 = |HyperNet_2(s)|
b_1, b_2 = HyperNet(s)

Q_tot = w_2 · ReLU(w_1 · Q + b_1) + b_2
```

**优势**：
1. **分散 greedy**：每个智能体独立选择 argmax Q_i
2. **全局最优**：联合动作是全局最优
3. **灵活混合**：超网络学习复杂混合
4. **可扩展**：智能体数量可变

**与 VDN 对比**：

| 方面 | VDN | QMIX |
|------|-----|------|
| **混合** | 简单求和 | 非线性混合 |
| **表达力** | 低 | 高 |
| **单调** | 是 | 是 |
| **计算** | 简单 | 较复杂 |

**评分要点：** 说出单调约束得基本分，解释一致性保证得高分

## 题目 4：代码实现
**问题：** 请实现 QMIX 的混合网络部分。

**参考答案：**
```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class HyperNetwork(nn.Module):
    """超网络生成混合权重"""
    
    def __init__(self, state_dim, n_agents, hidden_dim=64, mixing_dim=32):
        super().__init__()
        self.n_agents = n_agents
        self.hidden_dim = hidden_dim
        self.mixing_dim = mixing_dim
        
        # 生成 w1, b1
        self.hyper_w1 = nn.Linear(state_dim, hidden_dim * mixing_dim)
        self.hyper_b1 = nn.Linear(state_dim, hidden_dim)
        
        # 生成 w2, b2
        self.hyper_w2 = nn.Linear(state_dim, mixing_dim * 1)
        self.hyper_b2 = nn.Linear(state_dim, 1)
        
        # 中间混合层
        self.mixing_net = nn.Sequential(
            nn.Linear(hidden_dim, mixing_dim),
            nn.ReLU()
        )
    
    def forward(self, q_values, state):
        """
        混合个体 Q 值为全局 Q
        
        q_values: (batch, n_agents)
        state: (batch, state_dim)
        """
        batch_size = q_values.shape[0]
        
        # 生成权重（绝对值保证非负）
        w1 = torch.abs(self.hyper_w1(state))  # (batch, hidden*mixing)
        w1 = w1.view(-1, self.hidden_dim, self.mixing_dim)
        
        b1 = self.hyper_b1(state)  # (batch, hidden)
        
        # 第一层混合
        # q_values: (batch, 1, n_agents)
        # w1: (batch, n_agents, mixing)
        hidden = torch.bmm(q_values.unsqueeze(1), w1).squeeze(1)
        hidden = hidden + b1
        hidden = F.relu(hidden)
        
        # 通过中间层
        hidden = self.mixing_net(hidden)  # (batch, mixing)
        
        # 第二层混合
        w2 = torch.abs(self.hyper_w2(state))  # (batch, mixing)
        w2 = w2.unsqueeze(-1)  # (batch, mixing, 1)
        
        b2 = self.hyper_b2(state)  # (batch, 1)
        
        # 输出 Q_tot
        q_tot = torch.bmm(hidden.unsqueeze(1), w2).squeeze(-1) + b2
        
        return q_tot

class QMIX:
    def __init__(self, n_agents, obs_dim, action_dim, state_dim):
        self.n_agents = n_agents
        
        # 个体 Q 网络
        self.agent_nets = nn.ModuleList([
            nn.Sequential(
                nn.Linear(obs_dim, 64),
                nn.ReLU(),
                nn.Linear(64, action_dim)
            ) for _ in range(n_agents)
        ])
        
        # 混合网络
        self.mixer = HyperNetwork(state_dim, n_agents)
    
    def forward(self, obs, state, actions=None):
        """前向传播"""
        batch_size = obs.shape[0]
        
        # 计算个体 Q 值
        q_values = []
        for i in range(self.n_agents):
            q = self.agent_nets[i](obs[:, i, :])  # (batch, action_dim)
            
            if actions is not None:
                # 选择执行的动作 Q 值
                q = q.gather(1, actions[:, i:i+1]).squeeze(-1)
            
            q_values.append(q)
        
        q_values = torch.stack(q_values, dim=1)  # (batch, n_agents)
        
        # 混合
        q_tot = self.mixer(q_values, state)
        
        return q_tot, q_values
```

**关键点**：
1. 超网络生成权重
2. 绝对值保证单调性
3. 两层混合结构
4. 个体 Q 值 gather 选择

**评分要点：** 代码正确实现混合得基本分，解释单调性得高分

## 题目 5：应用场景
**问题：** 多智能体 RL 适合哪些应用场景？选择 MARL 而非单智能体 RL 的条件是什么？

**参考答案：**
**适合的应用场景**：

**1. 多机器人系统**：
- 协同搬运重物
- 编队飞行
- 搜索救援
- 仓库自动化

**2. 智能交通**：
- 交通信号协调
- 自动驾驶车队
- 路径规划
- 拼车系统

**3. 游戏 AI**：
- MOBA 游戏（Dota 2, LoL）
- RTS 游戏（StarCraft）
- 足球游戏
- 多人射击

**4. 经济系统**：
- 市场模拟
- 拍卖机制
- 资源分配
- 供应链优化

**5. 通信网络**：
- 频谱分配
- 路由优化
- 负载均衡

**选择 MARL 的条件**：

**需要 MARL 当**：
1. **多个决策者**：
   - 多个独立智能体
   - 每个有自己的动作空间

2. **交互重要**：
   - 智能体之间相互影响
   - 需要协调或竞争

3. **分散信息**：
   - 每个智能体局部观测
   - 无法集中决策

4. **可扩展需求**：
   - 智能体数量可变
   - 需要泛化

**不需要 MARL 当**：
1. **单一决策者**：
   - 中央控制器
   - 单智能体 RL 足够

2. **无交互**：
   - 智能体独立
   - 可分别训练

3. **完全可观测**：
   - 全局状态可用
   - 集中控制可行

**决策框架**：
```
多个智能体？
├─ 否 → 单智能体 RL
└─ 是 → 智能体交互？
    ├─ 否 → 独立 RL
    └─ 是 → MARL
        ├─ 合作 → QMIX, MAPPO
        ├─ 竞争 → 对手建模
        └─ 混合 → 一般和博弈
```

**实践建议**：
- 从简单开始（独立学习）
- 逐步增加协调
- 评估通信需求
- 考虑部署约束

**评分要点：** 说出 3 个以上场景得基本分，给出决策框架得高分
