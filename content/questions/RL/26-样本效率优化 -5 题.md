# 样本效率优化面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是样本效率？为什么它对强化学习如此重要？

**参考答案：**
**样本效率（Sample Efficiency）**：
算法从有限环境交互中学习有效策略的能力。样本效率高意味着用更少的交互达到相同性能。

**度量方式**：
```
样本效率 = 性能 / 样本数量
```
或
```
达到目标性能所需的样本数（越少越好）
```

**为什么重要**：

**1. 真实交互成本高**：
```
机器人：硬件磨损、时间成本
自动驾驶：安全风险、法规限制
医疗：患者安全、伦理约束
```

**2. 时间限制**：
```
真实世界时间宝贵
无法像仿真那样加速
```

**3. 安全考虑**：
```
探索可能危险
需要最小化风险尝试
```

**4. 可扩展性**：
```
样本效率高→可处理复杂任务
样本效率低→只能简单任务
```

**对比**：
```
Atari 游戏（样本效率低）：
- DQN：数千万帧
- 人类：几分钟

机器人控制（需要高效率）：
- 真实交互：每小时成本$
- 必须高效学习
```

**提高样本效率的方法**：
1. 模型-Based RL
2. 离线 RL
3. 迁移学习
4. 数据增强
5. 优先级回放

**评分要点：** 说出成本和安全得基本分，详细对比得高分

## 题目 2：模型-Based RL
**问题：** 模型-Based RL 如何提高样本效率？有什么挑战？

**参考答案：**
**模型-Based RL**：
学习环境的动力学模型 P(s'|s,a)，用模型进行规划或数据增强。

**提高样本效率的机制**：

**1. 规划（Planning）**：
```
用模型预测未来
在"想象"中试错
无需真实交互
```

**2. 数据增强**：
```
用模型生成合成数据
扩充训练集
增加样本多样性
```

**3. 价值估计**：
```
用模型预测长期回报
减少 Monte Carlo 方差
```

**样本效率对比**：
```
Model-Free：10000+ 真实交互
Model-Based：100-1000 真实交互
```

**挑战**：

**1. 模型误差**：
```
模型不完美→规划错误
误差累积→策略失效
```

**2. 计算成本**：
```
训练模型需要数据
规划需要计算
可能抵消样本效率优势
```

**3. 模型偏差**：
```
模型学到的可能是虚假相关
策略利用模型漏洞
```

**4. 复杂环境**：
```
高维状态难建模
随机环境难预测
```

**解决方案**：

**1. 不确定性估计**：
```
集成模型
贝叶斯神经网络
避免过度信任模型
```

**2. 保守规划**：
```
在模型可靠区域规划
不确定时查询真实
```

**3. 混合方法**：
```
模型-Based + Model-Free
初期用模型，后期用真实
```

**成功案例**：
- MBPO（Model-Based Policy Optimization）
- Dreamer
- MuZero

**评分要点：** 说出规划机制得基本分，解释挑战及解决得高分

## 题目 3：优先级回放
**问题：** 优先级经验回放如何提高样本效率？实现要点是什么？

**参考答案：**
**优先级经验回放（Prioritized Experience Replay）**：
根据 TD 误差大小优先级采样，优先学习"意外"的经验。

**提高样本效率的机制**：

**1. 聚焦重要样本**：
```
TD 误差大→预测不准→需要学习
TD 误差小→预测准→学习价值低
```

**2. 减少冗余**：
```
均匀采样：重复学习已知
优先级：聚焦未知
```

**3. 加速收敛**：
```
重要样本优先
更快学习关键知识
```

**优先级定义**：
```
P(i) = (|δ_i| + ε)^α
```
- δ_i：TD 误差
- ε：小常数（保证所有样本有概率）
- α：优先级程度（0=均匀，1=完全优先）

**实现要点**：

**1. 数据结构**：
```
Sum-Tree：O(log N) 更新和采样
```

**2. 重要性采样权重**：
```
w_i = (1/N · 1/P(i))^β
```
- 纠正采样偏差
- β从 0 增加到 1

**3. 优先级更新**：
```
每次更新后更新优先级
新样本最大优先级
```

**代码实现**：
```python
def sample(self, batch_size):
    priorities = self.priorities[:len(self.buffer)]
    probs = priorities / priorities.sum()
    
    indices = np.random.choice(len(self.buffer), batch_size, p=probs)
    
    # 重要性采样权重
    weights = (len(self.buffer) * probs[indices]) ** (-self.beta)
    weights /= weights.max()
    
    return samples, indices, weights
```

**参数选择**：
```
α = 0.6  # 优先级程度
β = 0.4 → 1.0  # 重要性采样
ε = 0.01  # 防止零概率
```

**效果**：
- 样本效率提升 20-50%
- 收敛更快
- 最终性能更好

**评分要点：** 说出 TD 误差优先级得基本分，解释重要性采样得高分

## 题目 4：数据增强
**问题：** 数据增强如何提高 RL 的样本效率？有哪些增强方法？

**参考答案：**
**数据增强（Data Augmentation）**：
对观测数据进行变换，增加训练样本多样性，提高泛化能力。

**提高样本效率的机制**：

**1. 增加样本量**：
```
一个样本→多个增强版本
有效样本量增加
```

**2. 提高泛化**：
```
学习不变特征
适应新情况
```

**3. 正则化**：
```
防止过拟合
提高鲁棒性
```

**增强方法**：

**1. 观测增强**：
```
- 随机裁剪
- 颜色抖动
- 随机旋转
- 翻转
- 模糊
```

**2. 数据混合**：
```
- Mixup：线性插值样本
- CutMix：拼接样本
```

**3. 域随机化**：
```
- 随机化环境参数
- 学习域不变策略
```

**4. 时间增强**：
```
- 帧堆叠
- 时间裁剪
- 速度变化
```

**实现要点**：

**1. 在线增强**：
```python
def train_step(self, batch):
    obs = batch['obs']
    obs_aug = self.augment(obs)  # 在线增强
    loss = self.compute_loss(obs_aug, ...)
```

**2. 一致性约束**：
```
增强前后 Q 值应一致
添加一致性损失
```

**3. 适度增强**：
```
太弱：效果有限
太强：破坏信息
平衡是关键
```

**效果**：
```
Atari：样本效率提升 2-3 倍
机器人：泛化能力提升
```

**注意事项**：
- 保持任务相关信息
- 不破坏关键特征
- 考虑动作不变性

**评分要点：** 说出增加样本量得基本分，列举多种方法得高分

## 题目 5：实际应用
**问题：** 在实际应用中如何选择合适的样本效率优化方法？有什么最佳实践？

**参考答案：**
**选择方法的决策框架**：

**1. 评估交互成本**：
```
成本高→模型-Based 或离线 RL
成本低→Model-Free 也可
```

**2. 评估数据可用性**：
```
有历史数据→离线 RL
无历史数据→在线学习
```

**3. 评估安全要求**：
```
安全关键→保守方法
可接受风险→探索性方法
```

**4. 评估计算资源**：
```
资源充足→复杂方法
资源有限→简单方法
```

**方法选择指南**：

| 场景 | 推荐方法 |
|------|----------|
| 机器人控制 | 模型-Based + 域随机化 |
| 自动驾驶 | 仿真 + 真实混合 |
| 推荐系统 | 离线 RL |
| 游戏 AI | 优先级回放 + 数据增强 |
| 医疗决策 | 离线 RL + 保守学习 |

**最佳实践**：

**1. 从简单开始**：
```
先试 Model-Free
不够再试 Model-Based
```

**2. 组合方法**：
```
优先级回放 + 数据增强
模型-Based + 离线微调
```

**3. 监控学习曲线**：
```
样本效率 = 性能/样本数
对比基线
```

**4. 早停机制**：
```
性能不再提升→停止
避免浪费样本
```

**5. 迁移学习**：
```
相关任务预训练
目标任务微调
```

**实践流程**：
```
1. 定义样本预算
2. 选择基线方法
3. 训练并监控
4. 未达标→升级方法
5. 记录最佳配置
```

**成功案例**：
```
OpenAI Dactyl（机械手）:
- 域随机化
- PPO
- 样本效率：~50 小时真实

Waymo 自动驾驶:
- 仿真为主
- 真实验证
- 样本效率：百万公里仿真/千公里真实
```

**评分要点：** 说出决策框架得基本分，给出最佳实践得高分
