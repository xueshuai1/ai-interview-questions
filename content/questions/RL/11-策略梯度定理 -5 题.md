# 策略梯度定理面试题（5 道）

## 题目 1：定理内容
**问题：** 写出策略梯度定理的数学表达式，并解释各项含义。

**参考答案：**
**策略梯度定理**：
```
∇_θ J(θ) = E_τ∼π_θ[Σ_t ∇_θ log π_θ(a_t|s_t) Q^π(s_t, a_t)]
```

**各项含义**：
- `J(θ)`：策略目标函数（期望回报）
- `θ`：策略参数
- `π_θ(a_t|s_t)`：策略在 s_t 选择 a_t 的概率
- `∇_θ log π_θ(a_t|s_t)`：分数函数（score function）
- `Q^π(s_t, a_t)`：动作价值函数
- `τ`：轨迹 (s_0, a_0, s_1, a_1, ...)

**直观理解**：
- `∇logπ` 指向增加π(a|s) 概率的方向
- `Q(s,a)` 衡量动作好坏
- 好动作（Q 大）：增加概率
- 坏动作（Q 小）：降低概率

**等价形式**（用回报 G_t）：
```
∇_θ J(θ) = E_τ[Σ_t ∇_θ log π_θ(a_t|s_t) G_t]
```

**评分要点：** 写出公式得基本分，解释直观含义得高分

## 题目 2：证明思路
**问题：** 简述策略梯度定理的证明思路，关键步骤是什么？

**参考答案：**
**证明思路**：

**1. 从目标函数开始**：
```
J(θ) = E_τ∼π_θ[R(τ)] = Σ_τ P(τ|θ) R(τ)
```

**2. 梯度运算**：
```
∇_θ J(θ) = Σ_τ ∇_θ P(τ|θ) R(τ)
```

**3. 对数导数技巧（关键步骤）**：
```
∇_θ P(τ|θ) = P(τ|θ) ∇_θ log P(τ|θ)
```
所以：
```
∇_θ J(θ) = Σ_τ P(τ|θ) ∇_θ log P(τ|θ) R(τ)
         = E_τ[∇_θ log P(τ|θ) R(τ)]
```

**4. 轨迹概率分解**：
```
P(τ|θ) = ρ_0(s_0) Π_t π_θ(a_t|s_t) P(s_{t+1}|s_t,a_t)
```

取对数：
```
log P(τ|θ) = log ρ_0(s_0) + Σ_t [log π_θ(a_t|s_t) + log P(s_{t+1}|s_t,a_t)]
```

**5. 对θ求梯度**：
只有π依赖θ：
```
∇_θ log P(τ|θ) = Σ_t ∇_θ log π_θ(a_t|s_t)
```

**6. 代入得定理**：
```
∇_θ J(θ) = E_τ[Σ_t ∇_θ log π_θ(a_t|s_t) R(τ)]
```

用 Q 函数代替完整回报（等价）：
```
∇_θ J(θ) = E_τ[Σ_t ∇_θ log π_θ(a_t|s_t) Q^π(s_t, a_t)]
```

**关键步骤**：
- 对数导数技巧（log derivative trick）
- 轨迹概率分解
- 只有策略依赖θ

**评分要点：** 说出对数导数技巧得基本分，完整证明得高分

## 题目 3：基线
**问题：** 为什么可以在策略梯度中引入基线？基线的作用是什么？

**参考答案：**
**为什么可以引入基线**：

基线项的期望为 0：
```
E_a∼π[∇_θ log π_θ(a|s) · b(s)] 
= b(s) · E_a∼π[∇_θ log π_θ(a|s)]
= b(s) · ∇_θ E_a∼π[1]
= b(s) · ∇_θ 1
= 0
```

所以：
```
E[∇logπ · Q] = E[∇logπ · (Q - b)]
```

**基线的作用**：

**1. 减少方差**：
- 不减去基线：梯度方差大
- 减去合适基线：方差显著降低
- 不引入偏差（期望仍为 0）

**2. 常用基线**：
- 状态价值 V(s)
- 平均奖励
- 可学习基线网络

**3. 优势函数**：
```
A(s,a) = Q(s,a) - V(s)
```
用优势代替 Q：
```
∇_θ J(θ) = E[∇logπ · A(s,a)]
```

**4. 方差分析**：
```
Var(∇logπ · G) 大（G 变化大）
Var(∇logπ · (G-b)) 小（减去共同部分）
```

**最优基线**：
```
b* = E[∇logπ·G²] / E[∇logπ]
```

**实践**：
- 用 Critic 网络学习 V(s)
- Actor-Critic 天然提供基线

**评分要点：** 证明期望为 0 得基本分，解释方差减少得高分

## 题目 4：代码实现
**问题：** 请实现 REINFORCE 算法的更新函数。

**参考答案：**
```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.distributions import Categorical

class REINFORCE:
    def __init__(self, state_dim, action_dim, lr=1e-3, gamma=0.99):
        self.gamma = gamma
        self.policy = PolicyNetwork(state_dim, action_dim)
        self.optimizer = optim.Adam(self.policy.parameters(), lr=lr)
    
    def select_action(self, state):
        """选择动作并返回 log 概率"""
        logits = self.policy(torch.FloatTensor(state).unsqueeze(0))
        dist = Categorical(logits=logits)
        action = dist.sample()
        log_prob = dist.log_prob(action)
        return action.item(), log_prob
    
    def update(self, log_probs, rewards):
        """
        REINFORCE 更新
        
        log_probs: [log_prob_0, log_prob_1, ...]
        rewards: [r_0, r_1, ...]
        """
        # 1. 计算折扣回报 G_t
        returns = []
        G = 0
        for r in reversed(rewards):
            G = r + self.gamma * G
            returns.insert(0, G)
        
        returns = torch.FloatTensor(returns)
        
        # 2. 标准化（减少方差）
        returns = (returns - returns.mean()) / (returns.std() + 1e-8)
        
        # 3. 策略梯度损失
        # J(θ) = E[logπ(a|s) · G]
        # 梯度上升 → 最小化 -logπ(a|s) · G
        policy_loss = -(torch.stack(log_probs) * returns).mean()
        
        # 4. 反向传播和更新
        self.optimizer.zero_grad()
        policy_loss.backward()
        self.optimizer.step()
        
        return policy_loss.item()
    
    def train_episode(self, env):
        """训练一个 episode"""
        state = env.reset()
        log_probs = []
        rewards = []
        
        while True:
            action, log_prob = self.select_action(state)
            next_state, reward, done, _ = env.step(action)
            
            log_probs.append(log_prob)
            rewards.append(reward)
            
            if done:
                break
            
            state = next_state
        
        return self.update(log_probs, rewards)
```

**关键点**：
1. 从后向前计算回报
2. 标准化减少方差
3. 负对数概率乘回报（梯度上升）

**评分要点：** 代码正确得基本分，解释标准化得高分

## 题目 5：与价值方法对比
**问题：** 比较策略梯度方法和价值方法（如 Q-Learning）的优缺点。

**参考答案：**

**策略梯度方法**：

**优点**：
1. **连续动作空间**：天然支持
2. **随机策略**：可以学习多模态策略
3. **收敛性**：通常收敛到局部最优
4. **高维动作**：扩展性好

**缺点**：
1. **高方差**：梯度估计方差大
2. **样本效率低**：需要大量样本
3. **局部最优**：可能陷入次优
4. **训练慢**：收敛速度慢

**价值方法（Q-Learning）**：

**优点**：
1. **样本效率**：通常更高
2. **全局最优**：理论上收敛到全局
3. **低方差**：Bootstrap 减少方差
4. **成熟算法**：DQN 等成功应用

**缺点**：
1. **离散动作**：难以处理连续空间
2. **确定性策略**：难以学习随机策略
3. **最大化偏差**：可能高估
4. **动作选择**：需要 argmax 操作

**对比表格**：

| 方面 | 策略梯度 | 价值方法 |
|------|----------|----------|
| 动作空间 | 连续/离散 | 主要离散 |
| 策略类型 | 随机 | 确定性 |
| 方差 | 高 | 低 |
| 样本效率 | 低 | 高 |
| 收敛 | 局部最优 | 全局最优 |
| 代表算法 | PPO, SAC | DQN, DDQN |

**混合方法（Actor-Critic）**：
- 结合两者优点
- Actor：策略梯度
- Critic：价值估计（低方差基线）
- 最佳实践

**评分要点：** 说出 3 个以上优缺点得基本分，提到 Actor-Critic 得高分
