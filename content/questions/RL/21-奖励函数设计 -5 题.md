# 奖励函数设计面试题（5 道）

## 题目 1：核心挑战
**问题：** 奖励函数设计的主要挑战有哪些？如何避免奖励黑客？

**参考答案：**
**主要挑战**：

**1. 奖励黑客（Reward Hacking）**：
智能体找到奖励函数的漏洞，获取高奖励但不完成真实任务。

**示例**：
```
任务：清理房间
奖励：捡起的垃圾数量
黑客：把垃圾放下再捡起（重复刷奖励）
```

**2. 稀疏奖励问题**：
- 只有成功/失败时有奖励
- 学习信号不足
- 探索困难

**3. 奖励塑造困难**：
- 塑造不当导致偏离目标
- 难以设计合适的辅助奖励

**4. 多目标权衡**：
- 多个目标可能冲突
- 权重选择困难
- 帕累托最优

**5. 副作用忽略**：
- 未明确惩罚的负面行为
- 如：机器人快速移动损坏自身

**避免奖励黑客的方法**：

**1. 仔细设计**：
- 明确真实目标
- 考虑可能的漏洞
- 多角度测试

**2. 逆强化学习**：
- 从专家演示学习奖励
- 避免手工设计偏差

**3. 对抗验证**：
- 用另一个智能体找漏洞
- 迭代改进奖励

**4. 约束优化**：
- 添加安全约束
- 限制行为范围

**5. 人类反馈**：
- 持续监控行为
- 及时调整奖励

**6. 鲁棒奖励**：
- 对扰动不敏感
- 难以被利用

**实践建议**：
- 从简单开始
- 逐步增加复杂度
- 持续监控行为
- 准备应急停止

**评分要点：** 说出奖励黑客得基本分，提出多种避免方法得高分

## 题目 2：稀疏 vs 稠密
**问题：** 比较稀疏奖励和稠密奖励的优缺点，各适用什么场景？

**参考答案：**
**稀疏奖励**：
只在关键事件（成功/失败）时给予奖励。

**优点**：
1. **准确反映目标**：直接对应任务完成
2. **无需设计**：不需要人工塑造
3. **无偏差**：不会引导到次优解
4. **简单**：实现容易

**缺点**：
1. **学习困难**：信号稀疏，探索难
2. **样本效率低**：需要大量试验
3. **信用分配难**：不知道哪步导致成功

**稠密奖励**：
每步都有奖励信号。

**优点**：
1. **学习信号丰富**：每步都有指导
2. **样本效率高**：更快学习
3. **信用分配易**：知道每步好坏

**缺点**：
1. **设计困难**：需要人工塑造
2. **可能偏离目标**：优化奖励而非任务
3. **局部最优**：可能被辅助奖励误导

**对比表格**：

| 方面 | 稀疏奖励 | 稠密奖励 |
|------|----------|----------|
| **学习难度** | 高 | 低 |
| **设计难度** | 低 | 高 |
| **样本效率** | 低 | 高 |
| **目标对齐** | 高 | 中 |
| **适用** | 简单任务 | 复杂任务 |

**适用场景**：

**稀疏奖励适合**：
- 任务简单，容易探索
- 目标明确，无需引导
- 有演示数据（可结合 IL）
- 如：Atari 游戏

**稠密奖励适合**：
- 任务复杂，探索困难
- 需要引导学习
- 有领域知识设计奖励
- 如：机器人控制

**混合策略**：
1. **课程学习**：
   - 初期用稠密奖励引导
   - 后期用稀疏奖励微调

2. **奖励衰减**：
   - 辅助奖励随时间衰减
   - 最终只保留稀疏奖励

3. **自适应**：
   - 根据学习进度调整
   - 困难时增加稠密信号

**评分要点：** 说出优缺点得基本分，提出混合策略得高分

## 题目 3：奖励塑造
**问题：** 什么是奖励塑造？势基塑造如何保证不改变最优策略？

**参考答案：**
**奖励塑造（Reward Shaping）**：
添加辅助奖励 F(s,s') 引导学习：
```
R'(s,a,s') = R(s,a,s') + F(s,s')
```

**势基塑造（Potential-based Reward Shaping）**：
```
F(s,s') = γΦ(s') - Φ(s)
```
其中Φ(s) 是势函数。

**保证不改变最优策略的证明**：

**1. 累积奖励**：
```
Σ_t γ^t F(s_t, s_{t+1})
= Σ_t γ^t (γΦ(s_{t+1}) - Φ(s_t))
= Σ_t (γ^{t+1}Φ(s_{t+1}) - γ^tΦ(s_t))
```

**2. 望远镜求和**：
```
= γ^TΦ(s_T) - γ^0Φ(s_0)
= γ^TΦ(s_T) - Φ(s_0)
```

**3. 与策略无关**：
- Φ(s_0) 固定（初始状态）
- γ^TΦ(s_T) 只依赖终止状态
- 与中间路径无关

**4. 最优策略不变**：
```
J'(π) = J(π) + E[γ^TΦ(s_T) - Φ(s_0)]
```
第二项与π无关，所以：
```
argmax_π J'(π) = argmax_π J(π)
```

**势函数设计**：

**启发式**：
```
Φ(s) = -距离目标的距离
```
- 靠近目标：Φ增加
- F = γΦ(s') - Φ(s) > 0（正奖励）

**学习势函数**：
- 从价值函数学习
- 从专家演示学习

**实践要点**：
1. 确保势基形式
2. 验证不改变最优策略
3. 避免过度塑造

**非势基塑造的风险**：
- 可能改变最优策略
- 导致次优行为
- 需要谨慎使用

**评分要点：** 写出势基公式得基本分，证明最优策略不变得高分

## 题目 4：多目标奖励
**问题：** 如何设计多目标奖励函数？权重选择有什么技巧？

**参考答案：**
**多目标奖励设计**：

**线性加权**：
```
R(s,a) = Σ_i w_i · R_i(s,a)
```

**设计步骤**：

**1. 识别目标**：
- 列出所有相关目标
- 如：效率、安全、舒适

**2. 单独设计**：
- 为每个目标设计奖励 R_i
- 确保量纲一致

**3. 归一化**：
```
R_i' = (R_i - mean) / std
```
- 避免量级差异
- 权重更有意义

**4. 选择权重**：
- 反映目标重要性
- 可能需要迭代调整

**权重选择技巧**：

**1. 领域知识**：
- 安全 > 效率
- 硬约束权重极大

**2. 帕累托分析**：
- 绘制帕累托前沿
- 选择平衡点

**3. 敏感性分析**：
- 测试不同权重
- 选择鲁棒区域

**4. 课程学习**：
- 初期：简单目标
- 后期：增加目标

**5. 自适应权重**：
```
w_i ∝ 1 / (当前性能)
```
- 性能差的目标权重增加
- 自动平衡

**替代方法**：

**1. 约束优化**：
```
max R_1(s,a)
s.t. R_2(s,a) ≥ threshold
```

**2. 层次奖励**：
- 主要目标：稀疏奖励
- 辅助目标：稠密奖励

**3. 多目标 RL**：
- 学习帕累托前沿
- 运行时选择

**实践建议**：
- 从单目标开始
- 逐步添加目标
- 持续监控各目标
- 准备权衡分析

**评分要点：** 说出线性加权得基本分，提出多种技巧得高分

## 题目 5：实际应用
**问题：** 在机器人导航任务中，如何设计奖励函数？需要考虑哪些因素？

**参考答案：**
**机器人导航任务奖励设计**：

**任务定义**：
- 从起点导航到目标点
- 避免障碍物
- 高效到达

**奖励组件**：

**1. 目标奖励（稀疏）**：
```
R_goal = +10  如果到达目标
R_goal = -10  如果碰撞
```

**2. 距离奖励（稠密）**：
```
R_dist = -0.1 × 距离目标距离
```
引导向目标移动。

**3. 碰撞惩罚**：
```
R_collision = -1.0  如果碰撞
```

**4. 时间惩罚**：
```
R_time = -0.01  每步
```
鼓励快速到达。

**5. 平滑奖励**：
```
R_smooth = -0.001 × |a_t - a_{t-1}|²
```
鼓励平滑运动。

**完整奖励**：
```
R = R_goal + R_dist + R_collision + R_time + R_smooth
```

**权重选择**：
```
w_goal = 1.0      # 主要目标
w_dist = 0.1      # 引导
w_collision = 1.0 # 安全
w_time = 0.01     # 效率
w_smooth = 0.001  # 舒适
```

**考虑因素**：

**1. 安全优先**：
- 碰撞惩罚足够大
- 宁可慢不可撞

**2. 学习信号**：
- 初期距离奖励帮助探索
- 后期可衰减

**3. 单位一致**：
- 距离、时间单位统一
- 权重有意义

**4. 边界情况**：
- 目标附近行为
- 死胡同处理

**5. 泛化能力**：
- 不依赖特定地图
- 适应新环境

**迭代设计流程**：
```
1. 初始设计（基于经验）
2. 训练观察
3. 分析问题行为
4. 调整奖励
5. 重复直到满意
```

**常见问题及解决**：

| 问题 | 原因 | 解决 |
|------|------|------|
| 原地打转 | 时间惩罚太小 | 增加时间惩罚 |
| 贴墙走 | 距离奖励主导 | 增加障碍惩罚 |
| 抖动 | 平滑奖励不足 | 增加平滑权重 |
| 不探索 | 距离奖励太陡 | 减小距离权重 |

**评分要点：** 说出多个奖励组件得基本分，详细分析因素得高分
