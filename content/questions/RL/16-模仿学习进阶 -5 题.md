# 模仿学习进阶面试题（5 道）

## 题目 1：行为克隆
**问题：** 什么是行为克隆？它有什么问题？如何解决？

**参考答案：**
**行为克隆（Behavioral Cloning, BC）**：
将模仿学习视为监督学习问题，直接从专家演示中学习状态到动作的映射。

**方法**：
```
min_θ E_(s,a)∼D_E [-log π_θ(a|s)]
```

**问题**：

**1. 分布偏移（Covariate Shift）**：
- 训练数据分布：专家访问的状态
- 测试时分布：策略访问的状态
- 策略犯错→访问新状态→更可能犯错

**2. 误差累积**：
```
单步错误率ε
T 步后累积错误率 ≈ 1 - (1-ε)^T ≈ T·ε
```
- 长序列任务问题严重
- 无法从错误中恢复

**3. 因果混淆**：
- 可能学习虚假相关
- 如：看到刹车灯就刹车（而不是因为前车减速）

**解决方案**：

**1. DAgger**：
- 迭代收集策略访问的状态
- 专家标注这些状态
- 减少分布偏移

**2. 数据增强**：
- 增加演示多样性
- 覆盖更多状态

**3. 多模态策略**：
- 处理专家行为的多样性
- 条件策略

**4. 结合 RL**：
- 先用 BC 初始化
- 再用 RL 微调

**评分要点：** 说出分布偏移得基本分，提出解决方案得高分

## 题目 2：DAgger 算法
**问题：** DAgger 算法的核心思想是什么？它如何解决行为克隆的问题？

**参考答案：**
**DAgger（Dataset Aggregation）**：
迭代收集策略访问的状态，请专家标注，聚合数据重新训练。

**核心思想**：
- 行为克隆的问题：训练和测试分布不同
- DAgger 的解决：让训练分布接近测试分布

**算法流程**：
```
1. 初始化策略π_1（如 BC）
2. 对于 i=1,2,...,N:
   a. 用π_i 执行，收集访问的状态 D_i
   b. 专家标注 D_i 中的最优动作
   c. 聚合数据：D = D ∪ D_i
   d. 在 D 上训练新策略π_{i+1}
3. 返回最终策略
```

**如何解决分布偏移**：

**1. 覆盖策略访问的状态**：
- 行为克隆：只覆盖专家状态
- DAgger：覆盖策略可能访问的所有状态

**2. 迭代改进**：
- 每轮策略改进
- 收集更难的状态
- 逐步覆盖状态空间

**3. 理论保证**：
- 在聚合数据上训练
- 测试分布≈训练分布
- 误差有上界

**优点**：
- 解决分布偏移
- 理论保证
- 实践有效

**缺点**：
- 需要专家在线标注
- 多次迭代成本高
- 专家负担重

**变体**：
- No-regret DAgger
- 近似 DAgger（减少标注）

**评分要点：** 说出迭代收集得基本分，解释分布偏移解决得高分

## 题目 3：GAIL
**问题：** GAIL 的核心思想是什么？它与行为克隆有什么区别？

**参考答案：**
**GAIL（Generative Adversarial Imitation Learning）**：
用 GAN 框架学习策略，策略作为生成器，判别器区分专家和策略。

**核心思想**：
```
min_π max_D E_(s,a)∼E[log D(s,a)] + E_(s,a)∼π[log(1-D(s,a))]
```

**对抗训练**：
- 判别器 D：区分专家演示和策略行为
- 策略π：生成判别器无法区分的行为

**与行为克隆的区别**：

| 方面 | 行为克隆 | GAIL |
|------|----------|------|
| **框架** | 监督学习 | 对抗学习 |
| **目标** | 最小化动作误差 | 最大化判别器混淆 |
| **数据** | 只需专家 | 专家 + 策略交互 |
| **分布偏移** | 有问题 | 较好处理 |
| **在线交互** | 无需 | 需要 |
| **多模态** | 困难 | 较好 |

**GAIL 优势**：
1. **处理分布偏移**：在线交互收集数据
2. **学习隐式奖励**：判别器学习区分标准
3. **多模态行为**：可以学习多种策略
4. **无需动作标注**：只需状态 - 动作对

**GAIL 劣势**：
1. **需要交互**：不能纯离线
2. **训练不稳定**：GAN 固有问题
3. **计算成本高**：对抗训练
4. **超参数敏感**：需要仔细调参

**实践建议**：
- 有交互：GAIL 可能更好
- 纯离线：BC 或 DAgger
- 结合：BC 初始化 + GAIL 微调

**评分要点：** 说出对抗训练得基本分，详细对比得高分

## 题目 4：代码实现
**问题：** 请实现行为克隆的核心训练函数。

**参考答案：**
```python
import torch
import torch.nn as nn
import torch.optim as optim

class BehaviorCloning:
    def __init__(self, state_dim, action_dim, hidden_dim=256, lr=1e-3):
        self.policy = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, action_dim)
        )
        self.optimizer = optim.Adam(self.policy.parameters(), lr=lr)
        self.loss_fn = nn.MSELoss()  # 连续动作
    
    def train(self, states, actions, epochs=100, batch_size=32):
        """
        训练行为克隆
        
        states: numpy array (N, state_dim)
        actions: numpy array (N, action_dim)
        """
        states = torch.FloatTensor(states)
        actions = torch.FloatTensor(actions)
        
        dataset = torch.utils.data.TensorDataset(states, actions)
        dataloader = torch.utils.data.DataLoader(
            dataset, batch_size=batch_size, shuffle=True
        )
        
        losses = []
        for epoch in range(epochs):
            epoch_loss = 0
            num_batches = 0
            
            for batch_states, batch_actions in dataloader:
                # 前向传播
                pred_actions = self.policy(batch_states)
                
                # 计算损失
                loss = self.loss_fn(pred_actions, batch_actions)
                
                # 反向传播
                self.optimizer.zero_grad()
                loss.backward()
                self.optimizer.step()
                
                epoch_loss += loss.item()
                num_batches += 1
            
            avg_loss = epoch_loss / num_batches
            losses.append(avg_loss)
            
            if (epoch + 1) % 10 == 0:
                print(f"Epoch {epoch+1}, Loss: {avg_loss:.4f}")
        
        return losses
    
    def select_action(self, state):
        """选择动作"""
        with torch.no_grad():
            state = torch.FloatTensor(state).unsqueeze(0)
            action = self.policy(state)
            return action.cpu().numpy()[0]
    
    def evaluate(self, states, actions):
        """评估"""
        states = torch.FloatTensor(states)
        actions = torch.FloatTensor(actions)
        
        with torch.no_grad():
            pred_actions = self.policy(states)
            mse = self.loss_fn(pred_actions, actions).item()
        
        return mse

    # 加载专家数据

    # 创建并训练

    losses = bc.train(expert_states, expert_actions, epochs=100)
    
    # 评估
    mse = bc.evaluate(expert_states, expert_actions)
    print(f"评估 MSE: {mse:.4f}")
```

**关键点**：
1. 监督学习框架
2. 批量训练
3. MSE 损失（连续动作）

## 题目 5：应用场景
**问题：** 模仿学习适合哪些应用场景？与强化学习相比有什么优势？

**参考答案：**
**适合的应用场景**：

**1. 机器人操作**：
- 从人类演示学习抓取、放置
- 复杂操作难以设计奖励
- 安全考虑（避免危险探索）

**2. 自动驾驶**：
- 从人类驾驶学习行为
- 奖励函数复杂（安全、舒适、效率）
- 大量驾驶数据可用

**3. 游戏 AI**：
- 从人类玩家学习
- 快速达到人类水平
- 结合 RL 超越人类

**4. 医疗决策**：
- 从医生决策学习
- 奖励难以定义（患者健康）
- 伦理约束

**5. 对话系统**：
- 从人类对话学习
- 奖励稀疏（用户满意度）
- 自然语言复杂

**与 RL 相比的优势**：

| 方面 | 模仿学习 | 强化学习 |
|------|----------|----------|
| **奖励设计** | 无需 | 困难 |
| **探索** | 无需 | 需要 |
| **安全性** | 高 | 低（探索危险） |
| **数据效率** | 高（直接学习） | 低（试错） |
| **收敛速度** | 快 | 慢 |
| **最优性** | 专家水平 | 可能超专家 |

**具体优势**：

**1. 无需奖励设计**：
- RL 最大挑战之一
- 模仿学习绕过此问题
- 直接从演示学习

**2. 安全**：
- 无危险探索
- 适合安全关键应用
- 如医疗、自动驾驶

**3. 数据效率**：
- 直接学习正确行为
- 无需试错
- 样本效率高

**4. 快速部署**：
- 训练快
- 快速达到可用水平
- 适合产品化

**局限性**：
- 无法超越专家
- 需要高质量演示
- 分布偏移问题

**最佳实践**：
- 先用模仿学习初始化
- 再用 RL 微调提升
- 结合两者优势

**评分要点：** 说出 3 个以上场景得基本分，详细对比 RL 得高分
