# 信任域方法详解面试题（5 道）

## 题目 1：核心思想
**问题：** 信任域方法的核心思想是什么？为什么它能保证单调改进？

**参考答案：**
**核心思想**：
在策略变化不太大的"信任域"内优化，保证近似的有效性，从而保证每次迭代都能改进策略性能。

**为什么保证单调改进**：

**1. 性能下界定理**：
```
η(π̃) ≥ η(π) + E[A_π(s,a)] - C · KL_max(π || π̃)
```

**2. 约束优化**：
```
max_θ E[重要性比率 · 优势]
s.t. KL(π_old || π_new) ≤ δ
```

**3. 保证机制**：
- 约束 KL 散度≤δ
- 确保近似在信任域内有效
- 性能下界为正→实际性能改进

**4. 线搜索**：
- 如果更新不满足 KL 约束
- 缩小步长直到满足
- 保证每次更新都"安全"

**直观理解**：
- 信任域：近似模型可靠的区域
- 在域内优化：近似准确
- 域外未知：可能性能下降
- 约束在域内：保证改进

**与梯度下降对比**：
- 梯度下降：固定步长，可能过大
- TRPO：自适应步长，保证安全

**评分要点：** 说出约束 KL 得基本分，解释性能下界得高分

## 题目 2：优化问题
**问题：** 写出 TRPO 的优化问题公式，并解释如何求解。

**参考答案：**
**TRPO 优化问题**：
```
max_θ E_s∼ρ, a∼π_θ [π_θ(a|s)/π_θ_old(a|s) · A(s,a)]

s.t. E_s[KL(π_θ_old(·|s) || π_θ(·|s))] ≤ δ
```

**近似（一阶 + 二阶）**：
```
令 Δθ = θ - θ_old

max_Δθ g^T Δθ
s.t. (1/2) Δθ^T F Δθ ≤ δ
```

其中：
- g = ∇_θ J(θ_old)（梯度）
- F = Fisher 信息矩阵

**求解步骤**：

**1. Lagrange 函数**：
```
L(Δθ, λ) = g^T Δθ - λ((1/2)Δθ^T F Δθ - δ)
```

**2. 最优条件**：
```
∂L/∂Δθ = g - λ F Δθ = 0
→ Δθ = (1/λ) F^{-1} g
```

**3. 确定λ**：
代入约束：
```
(1/2)((1/λ)F^{-1}g)^T F ((1/λ)F^{-1}g) = δ
→ (1/λ)² (1/2) g^T F^{-1} g = δ
→ 1/λ = √(2δ / (g^T F^{-1} g))
```

**4. 最终解**：
```
Δθ = √(2δ / (g^T F^{-1} g)) · F^{-1} g
```

**5. 实际求解**：
- 用共轭梯度法求解 F x = g
- 得到 x = F^{-1} g
- 计算步长：step_size = √(2δ / (x^T g))
- 更新：Δθ = step_size · x

**6. 线搜索**：
- 回溯确保 KL≤δ
- 接受满足约束的最大步长

**评分要点：** 写出优化问题得基本分，详细求解步骤得高分

## 题目 3：共轭梯度法
**问题：** TRPO 中为什么用共轭梯度法？如何实现？

**参考答案：**
**为什么用共轭梯度法**：

**问题**：需要求解 F^{-1} g
- F 是 d×d 矩阵（d=参数量）
- 直接求逆 O(d³)，不可行
- 大规模网络参数百万级

**共轭梯度优势**：
- 只需计算 F·v（矩阵向量积）
- 无需显式构造 F
- 复杂度 O(k·d)，k≈10-20
- 适合大规模问题

**实现步骤**：

**1. Fisher 向量积**（自动微分）：
```python
def fvp(v):
    # g = ∇logp
    g = grad(log_probs)
    
    # F·v = ∇(g^T · v)
    Fv = grad((g * v).sum())
    
    # 加阻尼
    return Fv + damping * v
```

**2. 共轭迭代**：
```python
x = 0
r = g - F(x) = g
p = r

for k in range(n_steps):
    Ap = F(p)
    alpha = (r^T r) / (p^T Ap)
    x = x + alpha * p
    r = r - alpha * Ap
    beta = (r^T r) / (r_old^T r_old)
    p = r + beta * p
```

**3. 阻尼**：
```
F_damped = F + λI
```
- 提高数值稳定性
- 确保正定

**4. 迭代次数**：
- 通常 10-20 次足够
- 太多计算成本高
- 太少精度不够

**实践技巧**：
- 监控残差 ||r||
- 自适应迭代次数
- 预热（warm start）

**评分要点：** 说出避免求逆得基本分，写出实现得高分

## 题目 4：线搜索
**问题：** TRPO 中线搜索的作用是什么？如何实现？

**参考答案：**
**线搜索的作用**：

**1. 处理近似误差**：
- 二阶近似有误差
- 理论步长可能过大
- 线搜索确保实际满足约束

**2. 保证单调改进**：
- 检查实际性能是否改进
- 不改进则缩小步长
- 确保每次更新有效

**3. 自适应步长**：
- 无需手动调学习率
- 自动找到合适步长
- 更鲁棒

**实现（回溯线搜索）**：

```python
def line_search(theta_old, full_step, states, actions, delta=0.01):
    alpha = 1.0
    max_backtrack = 10
    
    for _ in range(max_backtrack):
        # 尝试更新
        theta_new = theta_old + alpha * full_step
        
        # 更新参数
        set_params(theta_new)
        
        # 计算实际 KL
        kl = compute_kl(states, actions)
        
        # 检查约束
        if kl < delta:
            # 可选：检查性能是否改进
            if performance_improved():
                return theta_new, alpha
        
        # 回溯
        alpha *= 0.5
    
    # 失败：返回原参数
    set_params(theta_old)
    return theta_old, 0
```

**参数选择**：
- 初始α=1（全步长）
- 回溯因子=0.5
- 最大回溯 10 次

**优化**：
- 先检查 KL（便宜）
- 再检查性能（昂贵）
- 早期停止

**评分要点：** 说出处理近似误差得基本分，写出实现得高分

## 题目 5：TRPO vs PPO
**问题：** 比较 TRPO 和 PPO 的异同，各适用什么场景？

**参考答案：**
**相同点**：
- 都基于信任域思想
- 都约束策略更新幅度
- 都保证稳定学习
- 都适用于连续控制

**不同点**：

| 方面 | TRPO | PPO |
|------|------|-----|
| **约束方式** | 硬约束（KL≤δ） | 软约束（Clip） |
| **优化方法** | 共轭梯度 + 线搜索 | 标准 SGD |
| **实现复杂度** | 高 | 低 |
| **计算成本** | 高（多次反向传播） | 低 |
| **理论保证** | 单调改进 | 经验有效 |
| **超参数** | δ（KL 约束） | ε（Clip 范围） |
| **样本效率** | 单次更新用大量样本 | 可多 epoch |

**TRPO 优势**：
- 理论保证更强
- 约束更精确
- 适合安全关键应用

**PPO 优势**：
- 实现简单
- 计算高效
- 易于调参
- 实践效果好

**适用场景**：

**TRPO 适合**：
- 需要理论保证
- 安全关键应用
- 研究/教学
- 计算资源充足

**PPO 适合**：
- 大多数 RL 应用
- 快速原型
- 资源有限
- 生产环境

**实践建议**：
- 默认选择：PPO
- 需要保证：TRPO
- 资源紧张：PPO
- 研究对比：两者都试

**代码对比**：
```
TRPO: 共轭梯度 + 线搜索（复杂）
PPO:  Clip 损失 + SGD（简单）
```

**评分要点：** 说出约束方式区别得基本分，详细对比得高分
