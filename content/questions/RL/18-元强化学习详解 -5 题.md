# 元强化学习详解面试题（5 道）

## 题目 1：核心概念
**问题：** 什么是元强化学习？它与传统 RL 有什么本质区别？

**参考答案：**
**元强化学习（Meta-RL）**：
学习如何快速适应新任务的强化学习方法，实现"学会学习"（learning to learn）。

**本质区别**：

| 方面 | 传统 RL | 元 RL |
|------|---------|-------|
| **目标** | 解决单一任务 | 学会快速适应 |
| **训练** | 单一 MDP | 任务分布 p(T) |
| **测试** | 同一任务 | 新任务 |
| **数据效率** | 需要大量交互 | 少样本适应 |
| **泛化** | 无 | 泛化到新任务 |

**直观理解**：
- 传统 RL：学习解决特定任务
- 元 RL：学习如何学习新任务

**示例**：
```
机器人抓取不同物体：
- 传统 RL：为每个物体单独训练
- 元 RL：训练后，新物体只需少量试验

游戏 AI：
- 传统 RL：每个关卡单独训练
- 元 RL：新关卡快速适应
```

**元 RL 优势**：
1. **快速适应**：新任务只需少量经验
2. **数据高效**：减少总交互次数
3. **泛化能力**：适应未见任务
4. **通用智能**：迈向 AGI 的一步

**元 RL 挑战**：
1. **任务分布**：需要定义合理的任务分布
2. **过拟合**：可能过拟合训练任务
3. **计算成本**：元训练复杂
4. **理论理解**：仍在发展中

**评分要点：** 说出"学会学习"得基本分，详细对比得高分

## 题目 2：MAML 算法
**问题：** MAML 的核心思想是什么？如何应用于 RL？

**参考答案：**
**MAML（Model-Agnostic Meta-Learning）**：
学习一个良好的初始参数，使得从该初始点出发，只需少量梯度步就能适应新任务。

**核心思想**：
```
学习θ，使得 θ - α∇L_T(θ) 在任务 T 上表现好
```

**两层优化**：

**内层（任务特定）**：
```
φ_i = θ - α ∇_θ L_T_i(θ)
```
- 在任务 T_i 上梯度更新
- 得到适应后的参数φ_i

**外层（元学习）**：
```
θ ← θ - β ∇_θ Σ_i L_T_i(φ_i)
```
- 在适应后参数上计算损失
- 更新元参数θ

**应用于 RL**：

**1. 策略梯度 MAML**：
```
内层：用策略梯度更新策略
外层：在适应后策略上评估
```

**2. 损失函数**：
```
L_T(θ) = -E_τ∼π_θ[Σ γ^t r_t]
```

**3. 实现要点**：
- 内层：1-5 步梯度更新
- 外层：在测试轨迹上评估
- 二阶梯度（计算 Hessian）

**优点**：
- 模型无关（可用于任何网络）
- 简单有效
- 理论保证

**缺点**：
- 二阶梯度计算成本高
- 内层步数需要调
- 可能不稳定

**变体**：
- First-order MAML（忽略二阶）
- Reptile（更简单近似）

**评分要点：** 说出两层优化得基本分，写出公式得高分

## 题目 3：任务嵌入
**问题：** PEARL 算法如何学习任务嵌入？任务嵌入的作用是什么？

**参考答案：**
**PEARL（Probabilistic Embeddings for Actor-Critic RL）**：
学习任务的概率嵌入，用于条件策略的快速适应。

**任务嵌入学习**：

**1. 编码器**：
```
h = Encoder(s, a, r)  # 从转移编码
z ∼ N(μ(h), σ²(h))   # 概率嵌入
```

**2. 聚合**：
```
z_context = Aggregate({h_1, ..., h_k})
```
- 从多个转移聚合
- 通常用平均或注意力

**3. 变分推断**：
```
max E[log π(a|s,z)] - KL(q(z|context) || p(z))
```

**任务嵌入的作用**：

**1. 任务表示**：
- 紧凑的任务编码
- 捕获任务关键信息
- 如奖励函数、动力学

**2. 条件策略**：
```
π(a|s, z)  # 策略以 z 为条件
```
- 同一网络处理不同任务
- z 指定当前任务

**3. 快速适应**：
- 新任务：推断 z
- 无需更新参数
- 前向传播即可

**4. 不确定性**：
- 概率嵌入捕获不确定性
- 探索任务空间
- 主动信息收集

**优势**：
- 无需梯度更新适应
- 概率框架
- 可扩展到多任务

**应用**：
- 多任务 RL
- 迁移学习
- 上下文 bandit

**评分要点：** 说出编码器得基本分，解释条件策略得高分

## 题目 4：代码实现
**问题：** 请实现 MAML 的内层更新逻辑。

**参考答案：**
```python
import torch
import torch.nn as nn
import torch.optim as optim

class MAML:
    def __init__(self, policy, inner_lr=0.01, inner_steps=1):
        self.policy = policy
        self.inner_lr = inner_lr
        self.inner_steps = inner_steps
    
    def inner_update(self, states, actions, advantages):
        """
        内层更新（任务特定适应）
        
        返回：适应后的策略（参数独立）
        """
        # 克隆当前策略（创建独立副本）
        import copy
        task_policy = copy.deepcopy(self.policy)
        
        # 创建任务特定优化器
        task_optimizer = optim.SGD(
            task_policy.parameters(), 
            lr=self.inner_lr
        )
        
        # 转换为 tensor
        states = torch.FloatTensor(states)
        actions = torch.LongTensor(actions)
        advantages = torch.FloatTensor(advantages)
        
        # 内层梯度更新
        for step in range(self.inner_steps):
            # 前向传播
            logits = task_policy(states)
            dist = torch.distributions.Categorical(logits=logits)
            log_probs = dist.log_prob(actions)
            
            # 策略梯度损失
            loss = -(log_probs * advantages).mean()
            
            # 梯度更新
            task_optimizer.zero_grad()
            loss.backward()
            task_optimizer.step()
        
        return task_policy
    
    def meta_update(self, tasks, meta_optimizer):
        """
        元更新
        
        tasks: 列表，每个任务包含训练和测试数据
        """
        meta_loss = 0
        
        for task in tasks:
            # 内层适应
            adapted_policy = self.inner_update(
                task['train_states'],
                task['train_actions'],
                task['train_advantages']
            )
            
            # 外层评估（在测试数据上）
            test_states = torch.FloatTensor(task['test_states'])
            test_actions = torch.LongTensor(task['test_actions'])
            test_advantages = torch.FloatTensor(task['test_advantages'])
            
            with torch.no_grad():
                logits = adapted_policy(test_states)
                dist = torch.distributions.Categorical(logits=logits)
                log_probs = dist.log_prob(test_actions)
            
            task_loss = -(log_probs * test_advantages).mean()
            meta_loss = meta_loss + task_loss
        
        # 元更新
        meta_optimizer.zero_grad()
        meta_loss.backward()
        meta_optimizer.step()
        
        return meta_loss.item()
```

**关键点**：
1. 克隆策略（独立参数）
2. 内层梯度更新
3. 外层在适应后评估
4. 元更新原始策略

## 题目 5：应用场景
**问题：** 元强化学习适合哪些应用场景？实现元 RL 需要什么条件？

**参考答案：**
**适合的应用场景**：

**1. 机器人学习**：
- 新环境快速适应
- 不同物体抓取
- 损伤恢复
- 工具使用学习

**2. 个性化系统**：
- 新用户快速建模
- 个性化推荐
- 自适应界面
- 用户偏好学习

**3. 游戏 AI**：
- 新关卡适应
- 新对手策略
- 快速上手新游戏
- 程序生成内容适应

**4. 自动驾驶**：
- 新路线适应
- 不同天气条件
- 新交通规则
- 不同驾驶风格

**5. 金融交易**：
- 市场状态适应
- 新资产类别
- 监管变化适应

**实现元 RL 的条件**：

**1. 任务分布**：
```
p(T)：可采样的任务分布
```
- 任务之间有关联
- 足够多样性
- 可定义训练/测试任务

**2. 任务标识或上下文**：
- 显式：任务 ID
- 隐式：从交互推断
- 上下文：历史 (s,a,r)

**3. 评估协议**：
- 训练任务集
- 测试任务集（不相交）
- 少样本适应评估

**4. 计算资源**：
- 元训练计算密集
- 需要并行采样
- GPU/TPU 加速

**5. 算法选择**：
- 基于优化：MAML
- 基于模型：RL²
- 基于嵌入：PEARL

**实践建议**：

**任务设计**：
- 从简单开始
- 逐步增加多样性
- 确保可泛化

**评估**：
- 报告适应曲线
- 对比基线（从头训练）
- 统计显著性

**调参**：
- 内层学习率
- 内层步数
- 元学习率

**挑战**：
- 任务分布定义
- 过拟合训练任务
- 计算成本

**评分要点：** 说出 3 个以上场景得基本分，详细说明条件得高分
