### 1. Tokenization（5 道题）

**Q1.1** 请解释什么是 Tokenization，为什么它是 NLP 预处理中最重要的步骤之一？

**参考答案**：
Tokenization（分词）是将连续文本切分成离散单元（tokens）的过程。它是 NLP 最重要的预处理步骤，因为：
1. **模型输入基础**：所有 NLP 模型都需要离散的 token 序列作为输入
2. **影响模型性能**：分词质量直接影响后续任务效果
3. **处理语言复杂性**：需要正确处理标点、缩写、复合词等
4. **词汇表大小**：分词策略决定词汇表大小，影响模型复杂度
5. **跨语言适配**：不同语言需要不同的分词策略

**Q1.2** 比较 Word-level、Subword-level 和 Character-level 分词的优缺点。

**参考答案**：
| 粒度 | 优点 | 缺点 |
|------|------|------|
| Word-level | 语义完整、序列短 | OOV 问题严重、词汇表大 |
| Subword-level | 平衡 OOV 和效率、可处理新词 | 可能破坏语义完整性 |
| Character-level | 无 OOV 问题、词汇表小 | 序列极长、计算成本高 |

**Q1.3** 请解释 BPE（Byte Pair Encoding）算法的工作原理。

**参考答案**：
BPE 是一种子词分词算法：
1. 初始化：将文本拆分为字符级别
2. 统计：计算所有相邻字符对的频率
3. 合并：将最频繁的字符对合并为一个新 token
4. 迭代：重复步骤 2-3，直到达到目标词汇量
5. 应用：使用学到的合并规则对新文本分词

```python
# 示例：
# 初始：l, o, w, e, r, n, e, w, e, s, t
# 迭代 1：合并 (e, s) → es
# 迭代 2：合并 (e, s, t) → est
# 最终词汇表包含：l, o, w, e, r, n, est, ...
```

**Q1.4** WordPiece 和 BPE 有什么区别？

**参考答案**：
| 特性 | BPE | WordPiece |
|------|-----|-----------|
| 合并标准 | 频率最高 | 语言模型似然度最高 |
| 标记方式 | 无特殊标记 | 使用 ## 标记子词 |
| 实现 | 开源多种 | Google/BERT 专用 |
| 效果 | 相似 | 略好（某些任务） |

**Q1.5** 如何处理中文分词的挑战？

**参考答案**：
中文分词挑战及解决方案：
1. **无空格分隔**：使用基于词典或统计的方法（如 jieba、HanLP）
2. **歧义问题**：使用上下文信息（HMM、CRF、BERT）
3. **未登录词**：结合规则和深度学习识别
4. **领域适应**：使用领域特定词典和微调模型
5. **多粒度**：支持字、词、短语等多级粒度

### 2. Word Embedding（5 道题）

**Q2.1** 为什么 One-Hot 编码不适合表示词语？词嵌入解决了什么问题？

**参考答案**：
One-Hot 编码的问题：
1. **维度灾难**：词汇表大小=维度，导致高维稀疏
2. **语义鸿沟**：任意两词相似度为 0，无法捕捉语义
3. **无法泛化**：每个词独立，无法迁移知识

词嵌入的解决方案：
1. **低维稠密**：通常 50-1000 维，远小于词汇表
2. **语义相似**：相似词向量距离近
3. **线性关系**：支持向量运算（king - man + woman ≈ queen）

**Q2.2** 解释分布式假设（Distributional Hypothesis）及其在词嵌入中的作用。

**参考答案**：
分布式假设："Words that occur in similar contexts tend to have similar meanings."

作用：
1. **理论基础**：为词嵌入提供语言学依据
2. **训练目标**：通过上下文预测学习词向量
3. **语义捕捉**：共现模式编码语义关系

**Q2.3** 请解释词向量中的类比推理现象（如 king - man + woman = queen）。

**参考答案**：
类比推理是词向量的神奇特性：
1. **数学形式**：vec(king) - vec(man) + vec(woman) ≈ vec(queen)
2. **原因**：训练过程中，具有相同关系的词对学习到了相似的向量偏移
3. **应用**：可用于词义推理、知识图谱补全等
4. **局限**：并非所有关系都能完美捕捉

**Q2.4** 如何评估词嵌入的质量？

**参考答案**：
评估方法：
1. **内在评估**：
   - 词相似度（WordSim-353、SimLex-999）
   - 词类比（Google Analogy Test）
2. **外在评估**：
   - 下游任务性能（分类、NER 等）
   - 与基线模型对比
3. **可视化**：t-SNE 降维观察聚类

**Q2.5** 词嵌入如何处理一词多义问题？有什么局限性？

**参考答案**：
传统词嵌入的局限：
1. **单一向量**：每个词只有一个向量，无法区分多义
2. **上下文无关**：bank 在"river bank"和"bank account"中向量相同

解决方案：
1. **多原型嵌入**：为每个词学习多个向量
2. **上下文相关嵌入**：ELMo、BERT 等动态生成
3. **词义消歧**：结合外部知识库

### 3. Word2Vec（5 道题）

**Q3.1** 比较 Word2Vec 中 CBOW 和 Skip-gram 的区别。

**参考答案**：
| 特性 | CBOW | Skip-gram |
|------|------|-----------|
| 预测方向 | 上下文→中心词 | 中心词→上下文 |
| 训练速度 | 快（2-3 倍） | 慢 |
| 频繁词 | 效果好 | 一般 |
| 稀有词 | 一般 | 效果好 |
| 词向量质量 | 好 | 略好 |

**Q3.2** Word2Vec 中的负采样（Negative Sampling）是什么？为什么需要它？

**参考答案**：
负采样是一种优化技术：
1. **问题**：完整 softmax 需要计算所有词的概率，O(V) 复杂度
2. **方案**：每次只更新正样本和少量负样本
3. **实现**：从噪声分布中采样 k 个负词（k=5-20）
4. **效果**：将复杂度降至 O(k)，大幅加速训练

**Q3.3** Word2Vec 训练中的子采样（Subsampling）策略是什么？

**参考答案**：
子采样用于处理高频词：
1. **公式**：P(keep) = (√(freq/t) + 1) × (t/freq)，t 通常取 1e-5
2. **效果**：高频词（如"the"）被大量丢弃，稀有词保留
3. **好处**：提高训练效率，改善稀有词向量质量

**Q3.4** 如何使用 Gensim 训练 Word2Vec 模型？

**参考答案**：
```python
from gensim.models import Word2Vec

sentences = [["cat", "sits", "on", "mat"], ["dog", "runs", "in", "park"]]

model = Word2Vec(
    sentences=sentences,
    vector_size=300,
    window=5,
    min_count=1,
    sg=1,        # 1=Skip-gram
    negative=5,
    epochs=10
)

# 使用
vector = model.wv["cat"]
similar = model.wv.most_similar("cat", topn=5)
```

**Q3.5** Word2Vec 的主要局限性是什么？

**参考答案**：
局限性：
1. **一词多义**：每个词单一向量
2. **OOV 问题**：无法处理训练时未出现的词
3. **上下文无关**：忽略句子结构信息
4. **领域迁移**：通用语料训练的词向量在特定领域可能不适用

### 4. GloVe（5 道题）

**Q4.1** GloVe 与 Word2Vec 的核心区别是什么？

**参考答案**：
| 特性 | Word2Vec | GloVe |
|------|----------|-------|
| 方法类型 | 预测式（局部窗口） | 计数式（全局共现） |
| 训练数据 | 滑动窗口样本 | 共现矩阵 |
| 理论基础 | 神经网络语言模型 | 矩阵分解 + 最小二乘 |
| 训练速度 | 较快 | 较慢（需构建共现矩阵） |

**Q4.2** 解释 GloVe 的目标函数及其各部分的含义。

**参考答案**：
GloVe 目标函数：
$$J = \sum_{i,j} f(X_{ij}) (w_i^T \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij})^2$$

各部分含义：
- $X_{ij}$：词 i 和 j 的共现次数
- $w_i, \tilde{w}_j$：主向量和上下文向量
- $b_i, \tilde{b}_j$：偏置项
- $f(X_{ij})$：权重函数，避免极端值主导

**Q4.3** GloVe 中的权重函数 f(x) 有什么作用？

**参考答案**：
权重函数：$f(x) = \min((x/x_{max})^\alpha, 1)$，通常 $x_{max}=100, \alpha=0.75$

作用：
1. **抑制高频共现**：避免极高频词对主导损失
2. **保留低频信息**：低频共现仍有适当权重
3. **平滑过渡**：在 $x_{max}$ 处平滑连接

**Q4.4** 如何加载和使用预训练的 GloVe 词向量？

**参考答案**：
```python
import numpy as np

def load_glove(path):
    embeddings = {}
    with open(path, 'r') as f:
        for line in f:
            values = line.split()
            word = values[0]
            vector = np.array(values[1:], dtype='float32')
            embeddings[word] = vector
    return embeddings

glove = load_glove('glove.6B.300d.txt')
vector = glove['king']
```

**Q4.5** GloVe 在哪些任务上表现特别好？

**参考答案**：
GloVe 优势任务：
1. **词相似度**：在 WordSim-353 等基准上表现优异
2. **词类比**：捕捉线性关系能力强
3. **文本分类**：作为特征初始化效果好
4. **小数据集**：预训练向量在小数据上迁移效果好

### 5. FastText（5 道题）

**Q5.1** FastText 与 Word2Vec 的主要区别是什么？

**参考答案**：
| 特性 | Word2Vec | FastText |
|------|----------|----------|
| 表示单元 | 完整词 | 词 + 字符 n-gram |
| OOV 处理 | 无法处理 | 可通过子词组合 |
| 形态学 | 不捕捉 | 捕捉词缀信息 |
| 词汇表 | 较小 | 较大（含 n-gram） |

**Q5.2** FastText 如何生成未登录词（OOV）的向量？

**参考答案**：
FastText 处理 OOV：
1. **分解**：将 OOV 词分解为字符 n-gram
2. **查找**：获取每个 n-gram 的向量
3. **组合**：将所有 n-gram 向量求和/平均
4. **结果**：即使词不在词汇表中，也能生成合理向量

```python
# "unbelievable" 分解为：<un, unb, nbe, ..., ble, le>, able>, ...
# 即使该词未见过，子词向量可能已学习
```

**Q5.3** FastText 的文本分类模型架构是怎样的？

**参考答案**：
FastText 分类器架构：
1. **输入层**：词/词袋表示
2. **嵌入层**：查找词向量
3. **平均池化**：所有词向量平均
4. **输出层**：线性分类 + Softmax

特点：简单高效，适合大规模文本分类

**Q5.4** FastText 在哪些场景下特别有用？

**参考答案**：
适用场景：
1. **多语言应用**：支持 157 种语言预训练
2. **OOV 频繁**：社交媒体、新词频繁出现
3. **形态丰富语言**：土耳其语、芬兰语等
4. **资源受限**：训练和推理速度快
5. **文本分类**：情感分析、主题分类等

**Q5.5** 如何使用 FastText 进行语言识别？

**参考答案**：
```python
import fasttext

# 加载预训练语言识别模型
model = fasttext.load_model('lid.176.bin')

# 预测
text = "Bonjour, comment allez-vous?"
prediction = model.predict(text)
print(prediction)  # ('__label__fr', 0.99)
```

### 11. 文本分类（5 道题）

**Q11.1** 请解释文本分类的基本流程和常用方法。

**参考答案**：
基本流程：
1. **数据收集**：标注好的文本 - 标签对
2. **预处理**：分词、清洗、标准化
3. **特征提取**：TF-IDF、词嵌入等
4. **模型训练**：朴素贝叶斯、SVM、深度学习
5. **评估**：准确率、精确率、召回率、F1

常用方法：
- 传统：朴素贝叶斯、SVM、逻辑回归
- 深度学习：TextCNN、LSTM、BERT

**Q11.2** 如何处理文本分类中的类别不平衡问题？

**参考答案**：
解决方案：
1. **重采样**：过采样少数类或欠采样多数类
2. **类别权重**：在损失函数中给少数类更高权重
3. **Focal Loss**：降低易分类样本权重
4. **数据增强**：生成少数类样本
5. **阈值调整**：调整分类阈值

**Q11.3** 比较 TextCNN、TextRNN 和 BERT 在文本分类上的优缺点。

**参考答案**：
| 模型 | 优点 | 缺点 |
|------|------|------|
| TextCNN | 并行、捕捉局部特征 | 无法捕捉长距离依赖 |
| TextRNN | 捕捉序列信息 | 训练慢、难并行 |
| BERT | 上下文相关、效果好 | 计算成本高、推理慢 |

**Q11.4** 什么是多标签文本分类？如何处理？

**参考答案**：
多标签分类：每个样本可以有多个标签

处理方法：
1. **独立分类器**：为每个标签训练一个二分类器
2. **共享编码器**：共享底层表示，多个输出头
3. **标签相关性**：建模标签间依赖关系
4. **损失函数**：使用 BCEWithLogitsLoss

**Q11.5** 如何评估文本分类模型的性能？

**参考答案**：
评估指标：
1. **准确率**：正确预测比例
2. **精确率**：预测为正的样本中实际为正的比例
3. **召回率**：实际为正的样本中被正确预测的比例
4. **F1 分数**：精确率和召回率的调和平均
5. **混淆矩阵**：详细展示各类别预测情况
6. **ROC-AUC**：适用于二分类

### 12. NER（命名实体识别）（5 道题）

**Q12.1** 什么是命名实体识别（NER）？列举常见的实体类型。

**参考答案**：
NER 是识别文本中具有特定意义的实体的任务。

常见实体类型：
- **人名（PER）**：John Smith
- **地名（LOC）**：New York, China
- **组织名（ORG）**：Google, United Nations
- **时间（TIME）**：Monday, 2024
- **数字（NUM）**：100, fifty
- **其他**：产品、事件、法律等

**Q12.2** 解释 BIO 标注方案。

**参考答案**：
BIO 标注：
- **B-**：实体开始（Begin）
- **I-**：实体内部（Inside）
- **O**：非实体（Outside）

示例：
```
John  B-PER
Smith I-PER
works O
at    O
Google B-ORG
```

**Q12.3** 如何使用 BERT 进行 NER 任务？

**参考答案**：
```python
from transformers import BertForTokenClassification

model = BertForTokenClassification.from_pretrained(
    'bert-base-cased',
    num_labels=9  # BIO 标签数
)

# 输入需要 token-level 标签
# 注意：WordPiece 分词可能导致标签对齐问题
```

**Q12.4** NER 中的嵌套实体问题如何解决？

**参考答案**：
嵌套实体：实体内部包含另一个实体
如："[New [York City]]"

解决方案：
1. **多层 BIO**：使用不同层标注不同嵌套级别
2. **跨度预测**：直接预测实体跨度
3. **序列标注 + 分类**：先检测跨度再分类
4. **图模型**：使用图结构建模嵌套关系

**Q12.5** 低资源场景下如何进行 NER？

**参考答案**：
低资源 NER 方法：
1. **迁移学习**：使用预训练模型（BERT、RoBERTa）
2. **少样本学习**：Prototypical Networks、Meta-Learning
3. **数据增强**：回译、实体替换
4. **远程监督**：利用知识库自动标注
5. **主动学习**：选择性标注最有价值的样本
